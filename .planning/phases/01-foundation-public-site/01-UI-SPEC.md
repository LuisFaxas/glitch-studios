> Visual and interaction contract for the Foundation + Public Site phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

| Property | Value |

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight element margins |
| sm | 8px | Compact element spacing, input padding-y |
| md | 16px | Default element spacing, card padding, input padding-x |
| lg | 24px | Section internal padding, card content gaps |
| xl | 32px | Layout gaps between sibling sections |
| 2xl | 48px | Major section breaks on mobile |
| 3xl | 64px | Page-level section spacing on desktop, sidebar collapsed width |

Exceptions:
- Bottom tab bar height: 64px (3xl) — matches native app feel (D-11)
- Sidebar collapsed width: 64px (3xl) — icon-only mode (D-10)
- Sidebar expanded width: 256px — full labels + icons (D-10)
- Audio player placeholder height: 72px — reserved for Phase 2 persistent player (D-13)
- Touch targets: minimum 44px on mobile (WCAG 2.5.8)

| Role | Font | Size | Weight | Line Height | Tracking |
|------|------|------|--------|-------------|----------|
| Body | Inter | 16px | 400 (regular) | 1.5 | 0 |

- Monospace numerals on pricing elements (`font-variant-numeric: tabular-nums`)

## Color

This is a flat black-and-white design. There is NO traditional accent color. The "accent" is white glow/luminance against black — brightness IS the hierarchy signal. Source: D-07.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (pure black) | Page backgrounds, root surface |
| Secondary (30%) | `#111111` (gray-900) | Cards, sidebar background, nav surfaces, elevated panels |
| Tertiary text surface | `#222222` (gray-800) | Hover states on cards, input backgrounds, code blocks |
| Primary text | `#f5f5f0` (off-white) | Body text, headings, primary content — NOT pure white (D-07, specifics) |
| Secondary text | `#888888` (gray-400) | Captions, labels, placeholder text, metadata |
| Muted text | `#555555` (gray-600) | Disabled states, tertiary information |
| Border | `#222222` (gray-800) | Card borders, dividers, input borders |
| Border hover | `#555555` (gray-600) | Input focus borders, interactive element hover borders |
| Glow | `rgba(255, 255, 255, 0.15)` | Box-shadow glow on hero CTAs, active nav item, focused inputs |
| Pure white | `#ffffff` | Reserved ONLY for: GLITCH logo text, primary CTA button text on hover, active states |
| Destructive | `#dc2626` (red-600) | Destructive actions only (delete confirmations, error states) |

**Accent reserved for:** There is no color accent. White glow (`box-shadow: 0 0 20px rgba(255,255,255,0.15)`) serves as the visual emphasis on: primary CTA buttons on hover, active navigation item, focused form inputs, GLITCH logo continuous glow.

**60/30/10 in this design:**
- 60% black backgrounds
- 30% gray-900 elevated surfaces (cards, panels, nav)
- 10% off-white text and white glow effects (the "light" IS the accent)

## Component Patterns

### GLITCH Logo (D-06)
- Base text: "GLITCH" in JetBrains Mono Bold, uppercase
- Technique: CSS `::before` and `::after` pseudo-elements using `data-text` attribute
- Load animation: 1.5s dramatic glitch (rapid `clip-path: inset()` changes, RGB-split offsets of 2-4px, horizontal slice displacement)
- Settled state: Continuous subtle glitch (slow clip changes every 3-5s, max 1px offset)
- Continuous subtle white glow: `text-shadow: 0 0 10px rgba(255,255,255,0.3)`
- Reference: Capture.PNG — fragmented/distorted text with horizontal scan-line displacement

### Navigation — Desktop (D-10)
- shadcn/ui Sidebar component in collapsible mode
- Collapsed: 64px wide, icon-only, tooltips on hover
- Expanded: 256px wide, icon + label, smooth width transition
- Toggle control: user-activated button (top of sidebar)
- State persisted via cookie (shadcn/ui Sidebar built-in)
- Background: gray-900 (`#111111`), border-right: gray-800 (`#222222`)
- Active nav item: off-white text + left border accent (2px solid off-white) + subtle glow

### Navigation — Mobile (D-11)
- Fixed bottom tab bar, 64px height
- 4 items: Beats, Services, Portfolio, About (D-12)
- Icon + small label (12px Inter medium) below each icon
- Active state: off-white icon + label, inactive: gray-400
- Background: gray-900 with top border gray-800
- Safe area padding for notched devices (`env(safe-area-inset-bottom)`)

### Hero Section (D-01, D-02)
- Viewport height: 90vh (Claude's discretion per D-02 — leaves 10vh visible of next section to signal scrollability)
- Video showreel background: auto-playing, muted, looped, `playsInline`
- Dark overlay gradient: `linear-gradient(to bottom, rgba(0,0,0,0.4), rgba(0,0,0,0.8))`
- Split text overlay: GLITCH logo top, tagline below, dual CTAs side-by-side
- CTAs equally prominent (D-01): "Book a Session" + "Browse Beats"
- CTA style: off-white text, gray-800 background, 1px gray-600 border, glow on hover

### Card Panels (D-04)
- Max-width container: 1280px centered
- Background: gray-900 (`#111111`)
- Border: 1px solid gray-800 (`#222222`)
- Border-radius: 8px
- Padding: 24px (lg)
- No shadow by default; subtle glow on hover for interactive cards

### Homepage Sections Order (D-03)
1. Hero (video showreel + CTAs)
2. Services overview (grid of service cards)
3. Featured beats carousel (Embla — placeholder cards for Phase 2)
4. Video portfolio carousel (Embla — YouTube lazy embeds)
5. Testimonials carousel (Embla — client quote cards)

### Scroll Animations (D-05)
- Technique: Motion `whileInView` (NOT full page transitions)
- Default reveal: `opacity: 0, y: 40` to `opacity: 1, y: 0` over 600ms ease-out
- Viewport trigger: `once: true, margin: "-100px"` (fires 100px before element enters viewport)
- Parallax: `useScroll` + `useTransform` on background layers only — subtle, max 20% speed differential
- Glitch section transitions: Brief 200ms CSS `clip-path` flicker on section borders as they enter view

### Footer
- Background: gray-900 (`#111111`), top border gray-800
- Content: GLITCH logo (small, no animation), nav links, social icons (Lucide), newsletter signup form, copyright
- Newsletter form: inline email input + "Subscribe" button
- Max-width container matching card panels (1280px)

## Animation Budget

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Logo load glitch | 1500ms | steps(2, start) | Page load (once) |
| Logo subtle glitch | 3000-5000ms loop | steps(2, start) | Continuous after load settle |
| Section scroll-in | 600ms | ease-out | whileInView (once) |
| Parallax background | Continuous | linear | useScroll |
| Sidebar expand/collapse | 200ms | ease-in-out | User toggle |
| Button hover glow | 200ms | ease-in-out | Hover |
| Card hover border | 150ms | ease-in-out | Hover |
| Tab switch | 200ms | ease-in-out | Click |
| Toast appear | 300ms | spring (stiffness 200, damping 20) | Server action response |
| Carousel slide | 300ms | ease-out | Swipe / click |

**Constraint:** No animation exceeds 1500ms except the continuous logo loop. Respect `prefers-reduced-motion: reduce` — disable all non-essential animations (keep logo static, remove parallax, remove scroll-in, keep functional transitions like sidebar).
