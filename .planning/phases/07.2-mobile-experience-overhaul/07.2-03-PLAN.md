  <action>
Make these specific changes to `src/components/home/hero-section.tsx`:

**1. Responsive hero height:**
Change the section element from:
```html
<section className="relative h-[90vh] overflow-hidden">
```
to:
```html
<section className="relative h-[70vh] md:h-[90vh] overflow-hidden">
```

**2. Replace viewport-relative positioning with container-relative positioning (per review concern):**

The current code uses `top: "50vh"` and `bottom-[12vh]` which are viewport-relative -- when the hero is 70vh on mobile, "50vh" is no longer the vertical center of the hero section. These must become container-relative.

**Logo/subtitle container:**
Replace:
```typescript
style={{
  top: "50vh",
  left: 0,
  right: 0,
  transform: `translateY(-50%) translateX(${collapsed ? "-32px" : "0px"})`,
}}
```
with container-relative positioning using Tailwind classes:
```typescript
className={clsx(
  "absolute inset-x-0 top-1/2 -translate-y-1/2 z-10 flex flex-col items-center justify-center",
  collapsed ? "md:-translate-x-8" : ""
)}
```
Remove the inline `style` entirely. `top-1/2 -translate-y-1/2` centers vertically within the section (container-relative), not the viewport.

**CTAs container:**
Replace the current `bottom-[12vh]` (viewport-relative) with container-relative bottom:
```typescript
className={clsx(
  "absolute z-10 inset-x-0 bottom-16 md:bottom-24 flex flex-col items-center gap-6 px-4 text-center",
  collapsed ? "md:-translate-x-8" : ""
)}
```
Remove any inline `style={{ transform: ... }}`. `bottom-16` (64px) on mobile and `bottom-24` (96px) on desktop places CTAs near the bottom of the hero section, not relative to the viewport.

**Scroll indicator:**
Replace:
```typescript
style={{ opacity: indicatorOpacity, marginLeft: collapsed ? -32 : 0 }}
```
with:
```typescript
style={{ opacity: indicatorOpacity }}
className={clsx(
  "absolute bottom-4 left-1/2 -translate-x-1/2 z-10",
  collapsed ? "md:ml-[-32px]" : ""
)}
```

**3. Import clsx** if not already imported:
```typescript
import clsx from "clsx"
```

**4. CTA layout verification:**
The current CTA layout uses `flex-col sm:flex-row gap-4`. On 375px (below sm:640px), buttons stack vertically. At 70vh on a 667px phone = ~467px. With logo centered and CTAs at bottom-16, there should be enough space. No changes needed to CTA flex layout.

**Why container-relative matters (per review):** With `top: "50vh"`, the logo would be at 50% of the VIEWPORT height -- on a 667px phone, that's 333px from the top. But the hero is only 70vh = 467px tall. So 333px is at 71% of the hero, not centered. `top-1/2` centers within the hero section itself.
