  <action>
Make these specific changes to `src/components/home/hero-section.tsx`:

**1. Responsive hero height:**
Change the section element from:
```html
<section className="relative h-[90vh] overflow-hidden">
```
to:
```html
<section className="relative h-[70vh] md:h-[90vh] overflow-hidden">
```
This reduces mobile hero to 70vh (per research: 50-70vh for mobile) while keeping 90vh on desktop.

**2. Replace inline sidebar offset with CSS breakpoint guard:**

The current code uses inline `style={{ transform: ... }}` with the `collapsed` variable in 3 places. On mobile, the sidebar does not render, but `collapsed` may still be `true`, causing a -32px offset that misaligns the hero content.

Replace all 3 inline transform/margin usages with Tailwind responsive classes:

**Subtitle + Logo container (line ~56-63 area):**
Change from:
```typescript
style={{
  top: "50vh",
  left: 0,
  right: 0,
  transform: `translateY(-50%) translateX(${collapsed ? "-32px" : "0px"})`,
}}
```
to:
```typescript
style={{ top: "50vh", left: 0, right: 0 }}
className={clsx(
  "absolute z-10 flex flex-col items-center justify-center -translate-y-1/2",
  collapsed ? "md:-translate-x-8" : ""
)}
```
Note: `-translate-x-8` = -32px in Tailwind. The `md:` prefix ensures the offset only applies on desktop.

**CTAs container (line ~76-79 area):**
Change from:
```typescript
style={{ transform: `translateX(${collapsed ? "-32px" : "0px"})` }}
```
to:
```typescript
className={clsx(
  "absolute z-10 bottom-[12vh] left-0 right-0 flex flex-col items-center gap-6 px-4 text-center",
  collapsed ? "md:-translate-x-8" : ""
)}
```
Remove the inline style entirely.

**Scroll indicator (line ~108-110 area):**
Change from:
```typescript
style={{ opacity: indicatorOpacity, marginLeft: collapsed ? -32 : 0 }}
```
to:
```typescript
style={{ opacity: indicatorOpacity }}
className={clsx(
  "absolute bottom-8 left-1/2 -translate-x-1/2 z-10",
  collapsed ? "md:ml-[-32px]" : ""
)}
```

**3. Import clsx** if not already imported:
```typescript
import clsx from "clsx"
```
