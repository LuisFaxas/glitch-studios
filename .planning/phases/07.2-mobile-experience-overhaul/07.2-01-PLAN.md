async function createMobileContext(browser: any, vp: typeof viewports[number]): Promise<BrowserContext> {
  return browser.newContext({
    viewport: { width: vp.width, height: vp.height },
    isMobile: vp.isMobile,

    userAgent: "Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.0 Mobile/15E148 Safari/604.1",

**Describe block 1: Default page state screenshots + overflow assertions (11 pages x 3 viewports = 33 tests)**

For each viewport x page combination:
1. Create mobile context with `createMobileContext`
2. Create new page, add splash skip initScript
3. Navigate with `waitUntil: "networkidle"`
4. Wait 500ms for animations to settle
5. **Assert no horizontal overflow:** `await expect(page.locator('body')).toHaveJSProperty('scrollWidth', await page.evaluate(() => window.innerWidth))` -- BUT since some pages may have overflow (this is an audit), use a softer approach: evaluate `document.documentElement.scrollWidth <= window.innerWidth` and **log the result** but do NOT fail the test. Instead, capture it as test info:
```typescript
const hasOverflow = await page.evaluate(() => 
  document.documentElement.scrollWidth > window.innerWidth
)
test.info().annotations.push({
  type: hasOverflow ? "overflow-detected" : "no-overflow",
  description: `${pg.name} at ${vp.name}px: scrollWidth=${await page.evaluate(() => document.documentElement.scrollWidth)}, innerWidth=${await page.evaluate(() => window.innerWidth)}`
})
```
6. Take fullPage screenshot: `{page.name}-{vp.name}-default.png`
7. Close context

**Describe block 2: Overlay-open state (home page only, 3 viewports = 3 tests)**

For each viewport:
1. Create mobile context, navigate to `/`
2. Skip splash, wait for networkidle
3. Click the menu trigger button: `page.locator('button[aria-label="Open navigation menu"]')`
4. Wait 300ms for overlay animation
5. Take screenshot: `overlay-open-{vp.name}.png`
6. Close context

**Describe block 3: Player-active state (beats page, 3 viewports = 3 tests)**

For each viewport:
1. Create mobile context, navigate to `/beats`
2. Skip splash, wait for networkidle
3. Try to trigger playback by clicking a play button on a beat card: `page.locator('[data-testid="play-button"], button:has(svg.lucide-play)').first()`
4. If play button found, wait 1000ms for player bar to appear
5. Take screenshot: `player-active-{vp.name}.png`
6. If no play button found, still take screenshot with annotation `no-play-button-found`
7. Close context

**Total: 33 + 3 + 3 = 39 tests minimum.**
