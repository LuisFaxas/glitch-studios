The public layout currently uses a Server Component. To consume `useAudioPlayer()` for dynamic padding, extract the `<main>` element into a small client wrapper component.

**Option A (inline client component in same file):** Not possible -- layout.tsx is a server component and cannot use hooks.

**Option B (create a wrapper):** Create a `MobileContentPadding` wrapper or convert the relevant part.

Implementation approach -- create a thin client component in `src/components/layout/mobile-content-wrapper.tsx`:

```typescript
"use client"

import { useAudioPlayer } from "@/components/player/audio-player-provider"
import clsx from "clsx"

export function MobileContentWrapper({ children, className }: { children: React.ReactNode; className?: string }) {
  const { currentBeat, isMinimized } = useAudioPlayer()
  const playerActive = !!currentBeat && !isMinimized

  return (
    <main
      className={clsx(
        "page-content flex-1 min-w-0 min-h-screen overflow-x-hidden md:pb-0",
        playerActive
          ? "pb-[calc(var(--tab-bar-height)+var(--spacing-player)+env(safe-area-inset-bottom))]"
          : "pb-[calc(var(--tab-bar-height)+env(safe-area-inset-bottom))]",
        className,
      )}
    >
      {children}
    </main>
  )
}
```

Then update `src/app/(public)/layout.tsx`:
- Import `MobileContentWrapper` from `@/components/layout/mobile-content-wrapper`
- Replace `<main className="page-content flex-1 min-w-0 min-h-screen overflow-x-hidden pb-16 md:pb-0">` with `<MobileContentWrapper>`
- Move `{children}` and `<Footer />` inside `<MobileContentWrapper>`

This keeps the layout as a server component (for `WidgetLatestPost` async rendering) while adding client-side dynamic padding.

The CSS custom properties `--tab-bar-height` (4rem = 64px) and `--spacing-player` (4.5rem = 72px) are already defined in globals.css. The `env(safe-area-inset-bottom)` handles notched devices. When no player is active, padding = 64px + safe area. When player is active, padding = 64px + 72px + safe area = 136px + safe area.
