## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tile grid gaps (`--gap-tile`), icon gaps |
| sm | 8px | Compact element spacing, inner padding on small widgets |
| md | 16px | Default element spacing, tile internal padding (`--pad-tile`) |
| lg | 24px | Page content margin (`--pad-page`), overlay content padding |
| xl | 32px | Section breaks within overlay |
| 2xl | 48px | Major section breaks (`--gap-section`) |
| 3xl | 64px | Page-level vertical spacing (tab bar + player clearance zone) |

Exceptions:
- Touch targets: minimum 48px height on all interactive elements (tab bar buttons already use `min-h-[48px]`)
- Tab bar height: 64px (`--tab-bar-height: 4rem`)
- Player bar height: 72px (`--spacing-player: 4.5rem`) — preserved from Phase 07.1 implementation. Rationale: the mobile waveform scrub target inside the player bar requires sufficient vertical height for accurate touch interaction per WCAG 2.5.5 (Target Size). At 64px, the waveform scrub area (after subtracting vertical padding and the track title row) would fall below the 44px minimum touch target for the scrub hit zone. 72px provides 48px of usable waveform scrub height after 12px top padding and 12px track-title row, meeting WCAG touch-area requirements.
- Safe area inset: `env(safe-area-inset-bottom)` added to tab bar bottom padding for notched devices

Source: `globals.css` spacing tokens, WCAG 2.5.5 / 2.5.8 target size

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Label | 13px | 400 | 1.4 | Inter |
| Body | 15px | 400 | 1.5 | Inter |
| Heading | 22px | 700 | 1.2 | JetBrains Mono, uppercase |
| Display | 28px | 700 | 1.1 | JetBrains Mono, uppercase |

Exactly 4 sizes. No additional sizes permitted in the mobile context.

Mobile-specific adjustments:
- Hero title on mobile (`max-width: 640px`): uses Display (28px). No separate 40px breakpoint — the jump from desktop 64px scales directly to 28px Display on mobile.
- Page titles on mobile (`max-width: 640px`): uses Display (28px)
- Section headings remain at Heading (22px) on mobile (no scaling needed)
- Tab bar labels are icon-only (no text labels) at 24px icon size

Source: `DESIGN-LANGUAGE.md` type scale, existing `text-*` tokens

## Mobile Viewport Breakpoints

Test at these viewports during Playwright audits:

| Device Class | Width | Rationale |
|-------------|-------|-----------|
| Small phone | 375px | iPhone SE / iPhone 13 mini — minimum supported width |
| Standard phone | 390px | iPhone 14/15/16 — most common current iPhone |

Breakpoint contract:

- `>= 768px` (`md:` and above): Desktop layout active (sidebar, no tab bar)

## Player Bar + Tab Bar Coexistence Contract

When audio is playing on mobile, both the player bar and tab bar must be visible.

### Stack order (bottom to top):
```
[ Tab Bar — 64px, fixed bottom ]
[ Player Bar — 72px, fixed above tab bar ]
[ Page Content — scrollable, padded for both bars ]
```

Specifications:
- Tab bar: `bottom: 0`, `z-index: 50`
- Player bar: `bottom: calc(var(--tab-bar-height) + env(safe-area-inset-bottom))`, `z-index: 40`
- Main content bottom padding: `pb-[calc(var(--tab-bar-height)+var(--spacing-player)+env(safe-area-inset-bottom))]` when player is active, `pb-[calc(var(--tab-bar-height)+env(safe-area-inset-bottom))]` when player is inactive
- Player bar preserves Phase 07.1 mobile waveform (tap/drag scrubbing)
- Player bar on mobile shows: play/pause button, track title (truncated), waveform with scrubbing, close/dismiss button
- No volume control on mobile player bar (use device volume)

Source: D-07 (research Spotify/SoundCloud pattern — stacked bars), D-08 (preserve 07.1 waveforms)

## Mobile Nav Overlay Contract

Full-screen overlay triggered by menu button in tab bar.

### Layout:
```
+----------------------------------+
|                        [X Close] |
|                                  |
|  [GLITCH LOGO TILE] *           |
|                                  |
|  [Beats    ] [Services ]        |
|  [Portfolio] [Artists  ]        |
|  [Blog     ] [Contact  ]       |
|                                  |
|  ------- separator -------      |
|                                  |
|  [Now Playing] [Studio Status]  |
|  [Latest Post] [Social Links ]  |
|                                  |
|  ------- separator -------      |
|                                  |
|  [Sign In / Sign Out]           |
|                                  |
+----------------------------------+

* Logo tile is the primary visual anchor/focal point of the overlay.
```

Specifications:
- Background: `#000000` full screen
- Z-index: 60 (above both tab bar and player bar)
- Entry animation: 150ms opacity fade + glitch overlay flash (existing pattern)
- Exit animation: 150ms opacity fade
- Close button: 48x48px tile in top-right corner, `X` icon
- **Logo tile:** Primary visual anchor of the overlay. Rendered at `wide` size spanning the full 2-column grid width. Uses the existing `LogoTile` component. Positioned at the top of the overlay content (below close button) to establish brand identity as the first element the user sees.
- Navigation tiles: 2-column grid, 4px gap
- Tile sizes: `wide` for primary nav (Beats, Services, Portfolio), `small` for secondary (Artists, Blog, Contact)
- Active tile: inverted (white bg, black text)
- Widget section: 2-column grid, 4px gap
- Auth section: single `wide` tile at bottom
- Scroll: `overflow-y-auto` for small screens with many widgets
- Focus trap: active when overlay is open (already implemented)
- Body scroll lock: active when overlay is open (already implemented)
- Close triggers: X button, Escape key, route change (all already implemented)

Source: D-03 (research-backed overlay improvements), existing `mobile-nav-overlay.tsx`

## Page-by-Page Mobile Audit Checklist

Every public page must pass these checks at 375px, 390px, and 430px viewports:

| Check | Criteria |
|-------|----------|
| No horizontal overflow | Page width never exceeds viewport width; no horizontal scrollbar |
| No overlapping elements | All text, images, and controls have clear boundaries |
| Touch targets | All interactive elements are at least 48x48px or have 48px touch area |
| Text readability | Body text at 15px, no text smaller than 11px, sufficient contrast |
| Tab bar clearance | Page content does not render behind the fixed tab bar |
| Player bar clearance | When player is active, content does not render behind stacked bars |
| Images responsive | All images scale to container width, no cropping that loses meaning |
| Forms usable | Input fields are full-width, keyboards do not obscure active fields |
| Carousels swipeable | All Embla carousels respond to touch swipe gestures |
| Animations performant | No jank or frame drops during scroll or interaction animations |

## Interaction Contracts

### Swipe Gestures
- Tab bar: no swipe gestures (taps only)
- Player bar: horizontal swipe on waveform for scrubbing (existing from 07.1)
- Overlay: no swipe-to-close (use X button or Escape key)
- Carousels: horizontal swipe via Embla (existing)

### Scroll Behavior
- Main content: native scroll with momentum
- Overlay: internal scroll with body scroll locked
- Tab bar: fixed, never scrolls
- Player bar: fixed, never scrolls

### Animation Budget (mobile)
- Overlay entrance: 150ms opacity (existing)
- Glitch flash on overlay open: 200ms (existing)
- Tab press: 100ms scale(0.97) (matching tile pressed state)
- Page transitions: keep existing Framer Motion, but verify no jank at 60fps
- Reduced motion: all animations respect `prefers-reduced-motion: reduce` (existing CSS rule)

### Focus Management
- Overlay opens: focus moves to first focusable element (close button)
- Overlay closes: focus returns to menu trigger in tab bar
- Tab navigation: visible focus outlines at 1px `#f5f5f0` with -2px offset
