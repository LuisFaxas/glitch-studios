## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, beat card metadata spacing |
| sm | 8px | Compact element spacing, filter chip gaps, card grid gaps |
| md | 16px | Default element spacing, internal tile/card padding |
| lg | 24px | Page content margin, filter bar padding |
| xl | 32px | Layout gaps between major sections (filters to list) |
| 2xl | 48px | Section breaks (bundles to catalog) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Tile grid gap: 2-4px (per Cyberpunk Metro spec, tiles almost touch)
- Card grid gap in card view: 8px (slightly looser than tile grid to let cover art breathe)
- Touch targets: minimum 44px height on all interactive elements (play buttons, filter chips, view toggle)

## Layout Specifications

### Card View (Default on Desktop)

```
Desktop (>=1024px): 3-column grid, 8px gap
Tablet (768-1023px): 2-column grid, 8px gap
Mobile (<768px): 1-column grid, 8px gap
```

#### Beat Card Anatomy (Card View)

```
+------------------------------------------+
|                                          |
|           COVER ART                      |
|         (aspect-ratio 1:1)               |
|         (fills full width)               |
|                                          |
|   [PLAY OVERLAY on hover]               |
|   (centered play icon, 48px,            |
|    semi-transparent black backdrop)      |
|                                          |
+------------------------------------------+
|  TITLE                     FROM $29      |
|  Producer Name                           |
|  [HIP-HOP] [140 BPM] [Am]              |
|  [Dark] [Aggressive]                     |
+------------------------------------------+
```

- Cover art: `aspect-ratio: 1/1`, `object-fit: cover`, no border-radius
- Fallback (no art): #222 background with centered Music icon (24px, #555)
- Play overlay: appears on hover, 48x48px play button centered on art, white icon on `rgba(0,0,0,0.6)` backdrop
- Title: 15px JetBrains Mono bold, truncate to 1 line
- Producer: 13px Inter, #888, truncate to 1 line
- Metadata badges: 11px JetBrains Mono uppercase, #f5f5f0 on #222 background, #333 border, inline-flex with 4px gap
- Mood tags: 11px Inter, #888 text, no background, comma-separated or as subtle pills
- Price: 15px JetBrains Mono bold, right-aligned on title row, prefixed with "FROM"
- Card padding: 16px on info section, 0 on art section
- Card background: #111
- Card border: 1px solid #222
- Card hover: background #1a1a1a, border #444, glitch-hover on title text

### List View

```
Desktop: Full-width rows, column headers visible
Mobile: Condensed rows, hide BPM/Key badges
```

#### Beat Row Anatomy (List View - Redesigned)

```
+--[cover 56px]--[title/producer]--[waveform strip]--[BPM]--[KEY]--[PRICE]--[PLAY]--+
|  +---------+   Title Name          ~~~waveform~~~    140   Am     $29      [>]     |
|  | ART     |   Producer Name                        BPM   KEY                     |
|  +---------+                                                                       |
+------------------------------------------------------------------------------------+
```

- Cover art: 56px x 56px desktop, 48px x 48px mobile
- Waveform strip: inline mini waveform visualization (WaveSurfer.js), 120px wide, 32px tall, #333 waveform color, #f5f5f0 progress color. Hidden on mobile.
- Column headers row (desktop only): TITLE, BPM, KEY, PRICE -- 11px JetBrains Mono uppercase #555
- Row height: auto (min ~64px desktop, ~56px mobile)
- Row background: #111, hover #1a1a1a
- Row border-bottom: 1px solid #222, hover #444
- Active playing indicator: 2px solid #f5f5f0 left border accent bar
- Play button: 36x36px, border 1px #333, hover invert (white bg, black icon)

### Filter Bar

```
+--[SEARCH INPUT]--[GENRE v]--[KEY v]--[MOOD v]--[BPM 60-200 ====]--[CLEAR]--[GRID|LIST]--+
```

- Single horizontal row, flex layout, items-center
- Search input: flex-1 with min-width 200px, leading search icon (16px), placeholder "Search beats..."
- Genre/Key/Mood: shadcn Select components, compact trigger width (auto based on content), #111 background, #222 border
- BPM: inline label "BPM" + dual-thumb Slider, 120px wide
- Clear button: text-only "CLEAR" in 11px JetBrains Mono, #888 text, visible only when filters active
- View toggle: two icon buttons (LayoutGrid + List icons from Lucide), 36x36px each, selected state inverted
- Background: #0a0a0a
- Border: 1px solid #222
- Padding: 12px 16px
- Mobile: wraps to 2 rows. Row 1: search (full width). Row 2: horizontal scroll of filter controls + view toggle.
- Sticky: `position: sticky; top: 0; z-index: 10` so filter bar stays visible during scroll

## Interaction Contracts

### Play/Pause

- Click play on any card/row: triggers AudioPlayerProvider, global PlayerBar appears at bottom
- Currently playing beat: card shows subtle pulse animation on cover art overlay, row shows left accent bar
- Click pause: stops playback, visual indicators clear
- Only one beat plays at a time (enforced by AudioPlayerProvider)

### View Toggle

- Persisted in URL as `?view=card` or `?view=list`
- Default: `card` on all viewports
- Toggle animates with a 200ms crossfade (motion/react AnimatePresence)
- Both views maintain scroll position when toggling (no scroll-to-top)

### Filter Interactions

- All filters update URL params via nuqs (existing pattern)
- Selecting a filter triggers server re-fetch (shallow: false, existing behavior)
- Multiple filters combine with AND logic
- Active filter count shown as badge on mobile filter trigger
- Clear all: resets all params to null

### Card Click

- Click on card info area (not play button): opens license modal (existing LicenseModal)
- Click on play overlay: plays/pauses the beat
- These are distinct click zones to avoid conflict

### Row Click

- Click anywhere on row (except play button): toggles expand/collapse detail panel (existing behavior)
- Detail panel shows full waveform, description, license tiers, add-to-cart

### Glitch Hover

- Card hover: title text gets `glitch-hover` class (existing CSS animation)
- Row hover: title text gets `glitch-hover` class
- Play button hover: border inverts to #f5f5f0, background inverts
- Filter bar does not glitch (utility element, not content)

## Animation Contracts

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Card hover glitch | glitch-hover keyframes on title | 200ms | steps(1) |
| Card play overlay | opacity 0 to 1 | 150ms | ease-out |
| View toggle crossfade | AnimatePresence exit/enter | 200ms | ease-in-out |
| Row expand/collapse | motion height 0 to auto | 300ms | ease-out |
| Filter bar sticky | no animation (instant stick) | -- | -- |
| Skeleton pulse | standard shadcn skeleton pulse | 2000ms | infinite |
| Playing beat pulse | subtle opacity oscillation on card overlay | 1500ms | ease-in-out, infinite |

All animations respect `prefers-reduced-motion: reduce` with opacity-only fallbacks.
