**Part A: Create `src/db/seed-beat.ts` -- standalone seed script for BREAK 5**

This script inserts the test beat directly into the database via Drizzle, bypassing the admin UI entirely. This avoids all auth/session issues.

```typescript
/**
 * Seed script for real beat testing.
 * Inserts "BREAK 5" by Trap Snyder into the database with R2 URLs.
 *
 * Usage:
 *   npx tsx src/db/seed-beat.ts           # Insert/update beat
 *   npx tsx src/db/seed-beat.ts --clean   # Remove test beat
 *
 * Prerequisite: Upload files to R2 first:
 *   npx wrangler r2 object put glitch-beats/beats/audio/break-5.mp3 --file="_RESOURECES/BEATS/TRAP_SNYDER/BREAK 5.mp3"
 *   npx wrangler r2 object put glitch-beats/beats/covers/break-5-cover.png --file="_RESOURECES/BEATS/TRAP_SNYDER/trap_snyder_logo.png"
 */
```

Script structure:
1. Import postgres and drizzle-orm/postgres-js (same pattern as seed.ts -- standalone, does NOT import from @/lib/db)
2. Import schema tables: beats, beatPricing, beatProducers
3. Define the beat data with a deterministic UUID (use `crypto.randomUUID()` once and hardcode it, or use a namespaced UUID v5):
   - `id`: Use a fixed UUID like `"00000000-0000-4000-a000-000000000001"` so the script is idempotent
   - `title`: "BREAK 5"
   - `slug`: "break-5"
   - `bpm`: 140
   - `key`: "Am"
   - `genre`: "Hip-Hop"
   - `moods`: ["Dark", "Aggressive"]
   - `status`: "published"
   - `previewAudioKey`: "beats/audio/break-5.mp3" (R2 key)
   - `coverArtKey`: "beats/covers/break-5-cover.png" (R2 key)
   - `sortOrder`: 1
4. Use upsert (INSERT ... ON CONFLICT (id) DO UPDATE) so re-running is safe
5. Insert beatPricing rows for at least 2 tiers:
   - mp3_lease: $29, isActive: true
   - wav_lease: $49, isActive: true
6. Insert beatProducers row: name "Trap Snyder", splitPercent 100
7. If `--clean` flag: DELETE from beatProducers, beatPricing, beats WHERE id matches

Read `src/db/seed.ts` to match the exact import pattern for postgres and drizzle setup. Read `src/db/schema.ts` to get exact column names for the insert (they may use camelCase in Drizzle but map to snake_case columns -- Drizzle handles this).

Read `src/lib/r2.ts` to understand how `getPublicUrl` constructs URLs from keys. The seed script stores the R2 key, and the server action constructs the full URL. Verify the key format matches what `getPublicUrl` expects.

**Part B: Upload files to R2 via CLI**

Before running the seed script, upload the actual files to R2 using wrangler CLI:

```bash
# Upload audio file
npx wrangler r2 object put glitch-beats/beats/audio/break-5.mp3 \
  --file="_RESOURECES/BEATS/TRAP_SNYDER/BREAK 5.mp3" \
  --content-type="audio/mpeg"

# Upload cover art
npx wrangler r2 object put glitch-beats/beats/covers/break-5-cover.png \
  --file="_RESOURECES/BEATS/TRAP_SNYDER/trap_snyder_logo.png" \
  --content-type="image/png"
```

NOTE: The bucket name may differ. Read the wrangler.toml or .env to find the correct R2 bucket name. If wrangler is not configured or R2 is not accessible, fall back to using local file paths or mock URLs -- the visual verification still works with a placeholder. The key is that the database has a real beat record so the catalog renders it.

If R2 upload fails (no wrangler config, no credentials), use a fallback approach:
- Set `coverArtKey` and `previewAudioKey` to null in the seed
- The beat will still appear in the catalog with the Music icon fallback
- Document this in the summary for the user to upload manually later

Then run the seed: `npx tsx src/db/seed-beat.ts`

```typescript
import { test, expect } from "@playwright/test"

const BASE_URL = process.env.BASE_URL || "https://glitch-studios.codebox.local"

test.describe("Beats Catalog Visual Verification", () => {
  test.beforeEach(async ({ page }) => {
    // Disable animations for deterministic screenshots
    await page.emulateMedia({ reducedMotion: "reduce" })
  })

  test("desktop card view (1440px)", async ({ page }) => {
    await page.setViewportSize({ width: 1440, height: 900 })
    await page.goto(`${BASE_URL}/beats`)
    await page.waitForLoadState("networkidle")

    // Verify filter bar is visible via stable selector
    await expect(page.locator('[data-testid="filter-bar"]')).toBeVisible()
    // Verify beat card grid is visible
    await expect(page.locator('[data-testid="beat-card-grid"]')).toBeVisible()
    // Verify at least one beat card exists
    await expect(page.locator('[data-testid="beat-card"]').first()).toBeVisible()
    // Verify search input placeholder
    await expect(page.locator('input[placeholder="Search beats..."]')).toBeVisible()
    // Verify view toggle
    await expect(page.locator('[data-testid="view-toggle"]')).toBeVisible()

    await page.screenshot({
      path: ".planning/audit-screenshots/07-beats-card-1440.png",
      fullPage: true,
    })
  })

  test("desktop list view (1440px)", async ({ page }) => {
    await page.setViewportSize({ width: 1440, height: 900 })
    await page.goto(`${BASE_URL}/beats?view=list`)
    await page.waitForLoadState("networkidle")

    // Verify list view renders
    await expect(page.locator('[data-testid="beat-list"]')).toBeVisible()
    // Verify column headers
    await expect(page.locator('[data-testid="list-headers"]')).toBeVisible()
    // Verify at least one beat row
    await expect(page.locator('[data-testid="beat-row"]').first()).toBeVisible()

    await page.screenshot({
      path: ".planning/audit-screenshots/07-beats-list-1440.png",
      fullPage: true,
    })
  })

  test("mobile card view (375px) - no horizontal overflow", async ({ page }) => {
    await page.setViewportSize({ width: 375, height: 812 })
    await page.goto(`${BASE_URL}/beats`)
    await page.waitForLoadState("networkidle")

    // CRITICAL: Assert no horizontal overflow (addresses HIGH review concern)
    const hasHorizontalOverflow = await page.evaluate(() => {
      return document.documentElement.scrollWidth > document.documentElement.clientWidth
    })
    expect(hasHorizontalOverflow).toBe(false)

    // Verify filter bar renders
    await expect(page.locator('[data-testid="filter-bar"]')).toBeVisible()

    await page.screenshot({
      path: ".planning/audit-screenshots/07-beats-card-375.png",
      fullPage: true,
    })
  })

  test("mobile list view (375px) - no horizontal overflow", async ({ page }) => {
    await page.setViewportSize({ width: 375, height: 812 })
    await page.goto(`${BASE_URL}/beats?view=list`)
    await page.waitForLoadState("networkidle")

    // Assert no horizontal overflow
    const hasHorizontalOverflow = await page.evaluate(() => {
      return document.documentElement.scrollWidth > document.documentElement.clientWidth
    })
    expect(hasHorizontalOverflow).toBe(false)

    await page.screenshot({
      path: ".planning/audit-screenshots/07-beats-list-375.png",
      fullPage: true,
    })
  })

  test("filtered view (1440px)", async ({ page }) => {
    await page.setViewportSize({ width: 1440, height: 900 })

      fullPage: true,
```
