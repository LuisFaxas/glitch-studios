Props: `{ midiFileUrl: string }`

Per D-04 and UI-SPEC MidiSequencer spec:
- Fetch the .mid file from the URL, parse with `@tonejs/midi` (`new Midi(arrayBuffer)`)
- Group tracks by instrument name. If track names match common drum/instrument patterns, label them (kick, snare, hi-hat, bass, melody). Otherwise use track.name or "Track {N}".
- Render as SVG (not canvas — easier to style with CSS):
  - Container: `bg-[#111] border border-[#222] rounded-none p-4`
  - Grid lines: `stroke="#222"`
  - Each track gets its own horizontal lane (row), separated by `#222` lines
  - Track labels: 11px JetBrains Mono, uppercase, `#888` text, positioned at left column (80px wide)
  - Notes rendered as `<rect>` elements: x = note.time mapped to pixel, width = note.duration mapped to pixel, y = lane position, height = lane height - 2px gap, fill = `#f5f5f0`, opacity = note.velocity mapped from 0.4 to 1.0
  - Timeline scales to show first 8 bars (or full duration if < 8 bars)
  - No playhead sync for now (optional future enhancement)
- Loading state: skeleton placeholder with pulsing `#222` bg
- Error/empty state: if MIDI file fails to load or has no tracks, show "No MIDI data available" in muted text
- Monochrome only — no colors per instrument (per UI-SPEC: "Differentiate instruments by vertical position, not color")
