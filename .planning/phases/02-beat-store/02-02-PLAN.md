1. Create `src/components/player/audio-player-provider.tsx` — "use client" React context provider:

The provider manages:
- `currentBeat`: `{ id, title, artist, previewAudioUrl, coverArtUrl } | null`
- `isPlaying`: boolean
- `currentTime`: number (seconds)
- `duration`: number (seconds)
- `isMinimized`: boolean (player bar visibility)
- `audioRef`: RefObject for the shared HTMLAudioElement

Methods:
- `play(beat)`: Set currentBeat, load audio src on audioRef.current, call .play(). If same beat and paused, just resume.
- `pause()`: audioRef.current.pause()
- `toggle()`: If playing, pause. If paused and currentBeat exists, play.
- `seek(time)`: audioRef.current.currentTime = time
- `setMinimized(bool)`: Toggle player bar visibility (audio keeps playing)

Use `timeupdate` event listener on the audio element to update `currentTime`. Use `loadedmetadata` for `duration`. Use `ended` to set isPlaying=false.

Interface for the beat object passed to `play()`:
```typescript
export interface PlayerBeat {
  id: string
  title: string
  artist: string
  previewAudioUrl: string
  coverArtUrl: string | null
}
```

Export `AudioPlayerProvider` component and `useAudioPlayer` hook. The provider renders a hidden `<audio>` element as its last child (NOT inside the div tree — just before closing provider tag).

2. Update `src/app/layout.tsx`:

- Remove the `<div id="audio-player-root" />` placeholder — no longer needed
- The PlayerBar component will be added in Task 2 as a child rendered inside the provider

Keep existing fonts, metadata, suppressHydrationWarning, dark class — change nothing else.
