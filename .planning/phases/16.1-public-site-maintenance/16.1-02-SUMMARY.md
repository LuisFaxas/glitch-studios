## Plan 16.1-02 — Visual parity (splash, hero, button glitch)

### Root cause diagnosis (Task 1 / D-05)
`SplashOverlay` used `localStorage` with a single global key `"glitch-splash-seen"`. Any visitor who saw the splash ONCE EVER never saw it again — that's why it appeared "intermittent" (it was actually deterministic non-render for all returning visitors). Two separate bugs compounded this:
1. Wrong storage API (`localStorage` instead of `sessionStorage`).
2. Tech homepage (`src/app/(tech)/tech/page.tsx`) never mounted `SplashOverlay` at all — Tech was guaranteed no splash on any visit.

### Changes

| File | Change |
|------|--------|
| `src/components/home/splash-overlay.tsx` | swap `localStorage`→`sessionStorage`; remove global `SEEN_KEY`; derive per-brand keys in-effect; add `brand: "studios" \| "tech"` prop; D-05 + D-03/D-04 comment blocks |
| `src/app/(public)/page.tsx` | pass `brand="studios"` to `<SplashOverlay>` |
| `src/app/(tech)/tech/page.tsx` | import `SplashOverlay` + `getSplashMode`, mount `<SplashOverlay mode={splashMode} brand="tech">` wrapping tech homepage content |
| `src/components/cart/cart-icon.tsx` | add optional `className` prop merged onto the button (default `p-2` preserved) so it can stretch to fill a parent tile |
| `src/components/layout/tile-nav.tsx` | collapsed cart: replace bare `<div><ShoppingCart/></div>` with full-width `<CartIcon className="w-full flex items-center justify-center p-3"/>` inside a `group` container with the `animate-glitch-hover` overlay span pattern; collapsed sign-in (both signed-in + signed-out branches) and expanded cart + sign-in all get the same `group + overlay` hover-glitch treatment — 6 instances total of `group-hover:animate-glitch-hover`; drop now-unused `ShoppingCart` lucide import |
| `src/components/home/tech-hero-section.tsx` | add D-06 comment documenting that tech hero inherits heartbeat + motion timing from shared `HeroSection` — parity is structural |
| `tests/16.1-splash-per-brand.spec.ts` | new Playwright spec — 3 tests × 2 projects = 6/6 passed. Verifies sessionStorage keys per brand + regression guard against localStorage revert |
