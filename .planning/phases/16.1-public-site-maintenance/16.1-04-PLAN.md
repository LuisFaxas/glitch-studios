<objective>
Run a medium-depth public-site audit sweep (per D-13) across every route built in phases 1–14, capture visual + functional state at 3 viewports, triage findings into three buckets per D-14 (Fix-in-phase / Promote-to-999.x / Accepted-as-is), and produce a committed report. This is the pre-flight that informs which bugs get fixed inline vs promoted to 999.x backlog.

Purpose: Without a systematic sweep, the v3.0 design review will stumble over known-but-undocumented regressions. This plan surfaces them all in one pass so decisions are explicit, not reactive.

Output: `16.1-AUDIT-SWEEP.md` triage report + a committed screenshots directory forming the visual baseline before v3.0 redesign work begins. Also: `tests/16.1-route-inventory.ts` — the shared ROUTES array consumed by both this plan's sweep spec and Plan 05's breakpoint spec.
</objective>
