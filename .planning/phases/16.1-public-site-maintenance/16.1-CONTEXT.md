<decisions>
## Implementation Decisions

### Sub-brand Navigation (Q1)
- **D-01:** **Same-tab navigation between Studios and GlitchTech.** Remove the `target="_blank"` branch in `tech-cross-link-tile.tsx` and `studios-cross-link-tile.tsx`. Cross-origin navigation in the same tab is acceptable — audio stops because browsers cannot persist `<audio>` elements across origins. Document this limitation in the code comments so future engineers don't "fix" it by re-adding new-tab. Revisit later if/when brands consolidate onto one origin.
- **D-02:** Visible behavior: clicking the GlitchTech cross-link tile while a beat is playing on Studios **transitions the current tab** to glitchtech.io; beat stops mid-song. Same in reverse. No toast/warning UI — just the transition.

### Splash / Intro Logo Animation (Q2)
- **D-03:** **Splash animation fires once per session, per brand.** If the user visits Studios and sees the Studios splash, then navigates to GlitchTech, the GlitchTech splash should ALSO fire (independent session tracking per brand).
- **D-04:** Storage mechanism: `sessionStorage` keyed by brand (e.g., `glitch_studios_splash_seen` and `glitch_tech_splash_seen`). Cleared when the browser tab closes.
- **D-05:** First-fix step: investigate why the animation is intermittently not rendering. Likely candidates: admin `splash_mode` config flag, gate condition in `splash-overlay.tsx`, or hydration mismatch suppressing the overlay on certain route types. Fix the root cause before re-keying storage per brand.

### Visual Parity — GlitchTech Hero (Q3)
- **D-06:** **Same hero design as Studios + same copy structure**, with "studios" → "tech" substitution and adapted references. Visual rhythm (heartbeat animation, layout, motion timing) is identical. Copy is the minimum-viable swap — examples: section titles, CTAs, scroll indicators all get the tech parallel. Not a full fresh rewrite.

### Button Consistency — Cart + Sign-in (no gray area, per user list)
- **D-07:** Cart and sign-in in the collapsed sidebar state must render as proper interactive elements (`<Link>` or `<button>`) covering the full bounding box, not bare icons wrapped in `<div>`. Current bug in `src/components/layout/tile-nav.tsx:100-103`: the collapsed cart is a `<ShoppingCart>` icon in a plain `<div>`.
- **D-08:** Both Cart and Sign-in use the site-wide hover-glitch effect per the memory rule `feedback_glitch_headers.md` — every header/button element uses hover-only RGB-split glitch. Expanded state already uses `<CartIcon />`; align the collapsed state to the same pattern.

### Wiring & Content (no gray area, per user list)
- **D-09:** Add a `Blog` entry to `src/components/layout/tech-nav-config.ts` pointing at `/tech/blog`. Uses a BookOpen icon (matches Studios nav convention).
- **D-10:** Stale "phase 7.6" copy on the benchmarks page (`/tech/benchmarks`) is replaced with either a proper empty state or a stub pointing at the methodology page. No specific copy decision locked here — Claude's discretion; keep it honest ("benchmark data populates as reviews are published") rather than promising a specific phase number.

### Social Media (Q5)
- **D-11:** Both Studios and GlitchTech footers/sidebars render the SAME social handles:
  - Instagram: `@glitchtech.io` → `https://instagram.com/glitchtech.io`
  - TikTok: `@glitchtech.io` → `https://tiktok.com/@glitchtech.io`
  - YouTube: `@glitchtech_io` → `https://youtube.com/@glitchtech_io`
  - Facebook, X (Twitter), Threads: **"coming soon" placeholders** — render the icon in a disabled/muted state (not a link), or omit entirely; Claude's discretion on visual treatment. Whichever choice is used must be consistent across both brands.
- **D-12:** All active social icon links open in a new tab with `target="_blank" rel="noopener noreferrer"`. Same-tab rule in D-01 applies only to sub-brand internal nav, not external socials.

### Audit Sweep Depth (Q4)
- **D-13:** **Medium depth.** For each public route built in prior phases (1–14):
  - Screenshot at 3 viewports: mobile (375), laptop (1280), desktop (1920)
  - Light interaction: click the primary CTA on each page, confirm it doesn't crash or 404
  - No deep flow testing (cart add/checkout, full filter permutations, deep link paths) — those belong in dedicated QA phases if needed
- **D-14:** Audit triage output: each finding lands in one of three buckets:
  - **Fix in this phase** — small, localized, matches the sweep scope
  - **Promote to 999.x backlog** — larger or cross-cutting
  - **Accepted as-is** — intentional design, not a bug

### Responsive Breakpoint Matrix (Q6)
- **D-15:** Playwright coverage at: **375 / 393 / 768 / 1024 / 1280 / 1366 / 1440 / 1920**. 13" laptop range (1280–1440) is explicit priority after user reported homepage breakage on their 13" screen.
- **D-16:** The homepage glitch logo clipping is a specific first-test-case. Other routes get swept after the homepage is confirmed clean.
- **D-17:** Screenshots for each route × each viewport are committed to the phase directory or a dedicated `/tests/phase-16.1-snapshots/` path (Claude's discretion) as the visual-regression baseline.
