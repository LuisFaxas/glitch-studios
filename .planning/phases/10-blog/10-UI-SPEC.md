# Phase 10 — UI Design Contract

> Visual and interaction contract for the Blog quality overhaul. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | base-nova (configured in `components.json`, baseColor neutral) |
| Component library | radix (via shadcn) |
| Icon library | lucide-react |
| Font | JetBrains Mono (headers/labels/mono), Inter (body/excerpts) — both via next/font |

Aesthetic: Cyberpunk Metro — flat black & white surfaces, mono-uppercase headers, hover-only RGB-split glitch (site-wide rule). No auto-running heading animations.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, metadata-row separator spacing |
| sm | 8px | Category chip internal padding, metadata row item gap |
| md | 16px | PostCard inner padding (`p-4`), hero body-block gap |
| lg | 24px | Grid-to-hero vertical gap, hero content block padding |
| xl | 32px | Section padding above/below "Load More" button, category-chip-row → grid spacing |
| 2xl | 48px | Hero banner vertical padding (desktop), h1 → chip-row spacing |
| 3xl | 64px | Page top padding under global nav (desktop) |

Grid gap: `gap-1` (4px) between PostCards — intentional Metro tile density (unchanged from existing implementation).

Card image aspect: `aspect-video` (16:9) — applies to every PostCard image slot AND the hero banner image (per D-03, D-04).

Exceptions:
- Hero banner headline uses `clamp(28px, 5vw, 48px)` font-size (fluid), not the token table. This is intentional per D-02.
- Placeholder (no cover image) headline uses `clamp(18px, 3vw, 28px)` per D-04.
- Grid gap of 4px (xs) is below md spacing — intentional Metro density already locked by existing blog grid.

## Typography

Three sizes, two weights, two families. Fluid clamps used only for the hero headline and placeholder headline per D-02/D-04.

| Role | Family | Size | Weight | Line Height | Usage |
|------|--------|------|--------|-------------|-------|
| Body / Excerpt | Inter | 13px (card), 14px (hero) | 400 | 1.5 | Post excerpts, hero excerpt |
| Label / Metadata | JetBrains Mono | 11px | 700 | 1.4 | Category badge, "N MIN READ", date, "LOAD MORE" button, category chips |
| Heading | JetBrains Mono | 20px (`text-xl`) | 700 | 1.2 | PostCard title, placeholder headline at smallest viewport |
| Display | JetBrains Mono | clamp(28px, 5vw, 48px) | 700 | 1.1 | Hero banner headline (wrapped in `GlitchHeading`) and page `BLOG` h1 |

Rules:
- All mono text: `uppercase` + `tracking-wide` (tracking 0.02em–0.05em).
- All headings wrapped in `<GlitchHeading>` — hover-only RGB-split glitch, no auto-running animation (site-wide rule).
- Titles use `line-clamp-2`; excerpts use `line-clamp-3` (D-03) — enforces uniform card height.
- `GlitchHeading` is the ONLY heading wrapper used on this phase. Do not ship a bespoke h1/h2 variant.

## Color

Locked palette (Phase 10 may not introduce new hex values):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` | Page background, hero banner base, PostCard base |
| Secondary (30%) | `#0a0a0a` / `#111111` | Card surface contrast, hero gradient mid-stop, category-chip idle background (`#222222`) |
| Accent (10%) | `#f5f5f0` | Primary CTAs, active-chip inverse fill, glitch-hover inversion, hero "READ POST" button, "LOAD MORE" button hover state |
| Muted | `#888888` | Excerpt text, metadata labels |
| Subtle border | `#222222` | Chip background, card hairline borders, metadata dot separators |
| Destructive | n/a | No destructive actions in this phase |

Accent reserved for:
- "READ POST" CTA on the hero banner (primary CTA — solid `#f5f5f0` background, `#000000` text)
- Active category chip (inverse: `bg-[#f5f5f0] text-[#000000]`)
- "LOAD MORE" button on hover (idle = mono label on `#111111`, hover = `#f5f5f0` fill with `#000000` text)
- Glitch-hover RGB-split overlay accent on headings (the site-wide hover-only effect)

Accent is NOT used for: card backgrounds, chip idle state, metadata rows, hero excerpt, page chrome, or the placeholder card image. These stay within dominant/secondary/muted.

Hero gradient (D-02): vertical gradient overlaying the 16:9 cover image. Stops: `from-black/90 via-black/50 to-black/0` (top → bottom). All values derive from `#000000` at varying alpha — no new hex values introduced.

Placeholder radial gradient (D-04): `radial-gradient(ellipse at center, #111111 0%, #0a0a0a 60%, #000000 100%)` with the existing slice/line-repeat texture overlay.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page h1 | `BLOG` |
| Category chip: "all" state | `ALL` |
| Hero banner primary CTA | `READ POST` |
| PostCard click affordance | Entire card clickable; no visible "READ MORE" label (card title IS the link) |
| Reading-time badge | `{N} MIN READ` (mono uppercase, floor = 1 per D-06) |
| Date format | Mono uppercase short-form: `APR 20 2026` (one format, sitewide) |
| Metadata row separator | ` · ` (middle dot with single spaces) |
| Load More button — idle | `LOAD MORE` |
| Load More button — loading | `LOADING...` |
| Load More button — exhausted | Button hidden (no "No more posts" label) |
| Empty category results | Heading `NO POSTS IN THIS CATEGORY` / Body `Try a different category or clear the filter to see every post.` |
| Empty blog (zero posts total) | Heading `NO POSTS YET` / Body `New writing is in the works. Check back soon.` |
| Error loading more | `COULDN'T LOAD MORE POSTS. TAP TO RETRY.` (button text replacement — click = retry the same offset) |
| Zero-featured fallback | Hero banner is hidden entirely; grid starts at the top (no placeholder hero, no copy). Per D-01. |
| Admin `is_featured` toggle label | `Feature on blog page` |
| Admin `is_featured` toggle helper | `Only one post can be featured at a time. Turning this on will un-feature any currently featured post.` |
| Admin `is_featured` save confirmation (toast) | `Featured post updated.` |

Destructive actions: none in this phase. The `is_featured` toggle quietly un-flags the previously featured post without a confirmation modal — the helper text warns the admin in advance, so no blocking dialog is needed.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Hover a heading | RGB-split glitch effect runs only on hover (site-wide rule, no auto-run) |
| Hover a PostCard | Existing glitch-hover overlay; cursor: pointer; whole card is the link |
| Tap a category chip | Filters the grid to that category; URL updates via `nuqs` query-state; resets offset to 0 |
| Tap "LOAD MORE" | Appends next 9 posts; button swaps to `LOADING...` with subtle pulse; URL `?offset=N` updates; no page reload |
| Deep-link with `?offset=N` | Server renders the full batch up to that offset so scroll position and content match the link |
| Mobile category chip row | Horizontal scroll on overflow (`overflow-x-auto`), no wrap, no scrollbar visible |
| Hero image load | Uses `next/image` with `priority` prop for LCP optimization |
| Placeholder card image | Renders per D-04 (mono title on radial gradient + slice texture) |

Batch size: 9 posts initial, +9 per "Load More" click (D-05).

Reading-time compute: session-cached via `react.cache()`, 225 wpm, floor 1, HTML/markdown stripped before tokenization (D-06).

- `PostCardPlaceholder` — inline in `PostCard`; renders when `coverImageUrl` is null (D-04). Not a standalone file unless the planner decides otherwise.
- `ReadingTimeBadge` — stateless presentational component consuming the derived minute count. Reused on hero, PostCard, and post-detail metadata row (per D-06).

Admin (scope-limited to one toggle):
- A `Switch` component from the existing shadcn installation on the post-edit form — no new admin component files beyond wiring the toggle.
