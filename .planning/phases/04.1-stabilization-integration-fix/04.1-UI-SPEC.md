## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tile grid gaps (`--gap-tile`), icon gaps |
| sm | 8px | Compact element spacing, widget internal padding |
| md | 16px | Default tile internal padding (`--pad-tile`) |
| lg | 24px | Page content margin (`--pad-page`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Between page sections (`--gap-section`) |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- `--sidebar-width: 280px` (fixed)
- `--tab-bar-height: 4rem`
- `--spacing-player: 4.5rem` (72px) -- accommodates the persistent audio player bar which needs 44px touch target height plus 16px top padding plus 12px bottom padding for comfortable thumb reach on mobile

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` | Background, page surfaces (`--color-bg-dominant`) |
| Secondary (30%) | `#111111` | Cards, sidebar, tiles, widgets (`--color-bg-secondary`) |

## Visual Hierarchy & Focal Points

Each screen type has a single focal point that the user's eye should land on first:

| Screen | Focal Point | How Achieved |
|--------|-------------|--------------|
| Home page | Hero headline / studio identity | Display (40px) JetBrains Mono, largest text on page, top of viewport |
| Beat catalog | Beat grid tiles | Tile images with hover scale, occupy 70%+ of viewport width |
| Services page | Service cards with pricing | Cards use secondary bg against dominant bg, price in heading weight |
| Blog listing | Post titles | Heading (28px) in off-white against black, stacked vertically |
| Admin dashboard | Primary data table | Full-width table in content area, sidebar recedes via lower contrast |
| Auth pages | Form inputs | Centered single-column form, only interactive elements on page |
| Empty states | Empty state message | Centered vertically and horizontally, heading weight draws the eye |

### Empty State Copy (per surface)

Each surface that queries the database must show a context-specific empty state rather than a generic fallback. The heading uses Label style (13px, JetBrains Mono, uppercase). The body uses Body style (16px, Inter).

| Surface | Heading | Body |
|---------|---------|------|
| Beat catalog | "No beats added yet" | "Beats will appear here once they are published in the admin dashboard." |
| Services page | "No services configured" | "Services will appear here once they are added in the admin dashboard." |
| Blog listing | "No posts published yet" | "Blog posts will appear here once they are published." |
| Admin bookings table | "No bookings yet" | "Bookings will appear here as clients schedule sessions." |
| Admin orders table | "No orders yet" | "Orders will appear here as clients purchase beats." |
| Admin users table | "No clients registered" | "Client accounts will appear here as users sign up." |
| Fallback (any other surface) | "No content yet" | "This section will populate as content is added. If you expected to see something here, contact support at admin@glitchstudios.com." |

### Error State Copy

Error states must provide two resolution paths. The heading uses Label style. The body uses Body style.

| Scenario | Heading | Body |
|----------|---------|------|
| Generic page error | "Something went wrong" | "Try refreshing the page. If the problem continues, contact support at admin@glitchstudios.com." |
| Database connection error | "Unable to load data" | "The server could not reach the database. Try again in a few seconds, or check the Vercel dashboard for service status." |
| Auth error (admin 500) | "Sign-in failed" | "Your session may have expired. Try signing in again. If the problem continues, clear your cookies and retry." |

## Interaction Contracts (Stabilization-Specific)

This phase fixes broken interactions rather than creating new ones. The following contracts define what "working" looks like for verification:

### Route Rendering Contract
Every route must return a fully hydrated page with no console errors. The existing visual contracts apply:
- Public pages: dark background (`#000`), off-white text (`#f5f5f0`), sidebar with tile navigation
- Admin pages: AdminShell layout with sidebar navigation, content area with data tables/forms
- Auth pages: centered form with flat inputs, monochrome styling

### Empty State Contract (D-08)
Pages queried against an empty database must:
- Render without crashes (no "Cannot read properties of undefined")
- Show the surface-specific empty state copy defined in the Copywriting Contract above
- Home page hero renders independently of DB state (existing `Promise.allSettled` pattern)

### Seeded State Contract (D-09)
Pages queried against seeded data must:
- Display realistic Glitch Studios content (not lorem ipsum)
- Beat catalog shows filterable beats with playable audio previews
- Services page shows bookable services with pricing
- Blog shows at least one post with proper typography
- Admin dashboard shows populated data tables

### Auth Flow Contract
- Login page accepts email/password, redirects to dashboard or admin based on role
- Session persists across browser refresh (cookie-based)
- Admin routes return dashboard (not 500) for authenticated admin/owner users
- Unauthenticated admin access redirects to /login

### Payment Flow Contract (D-07)
- Beat purchase: add to cart, checkout with Stripe test card `4242 4242 4242 4242`, webhook fires, receipt email sent
- Booking deposit: select service, pick slot, pay deposit with test card, confirmation email sent
