## Interaction Contracts (Stabilization-Specific)

This phase fixes broken interactions rather than creating new ones. The following contracts define what "working" looks like for verification:

### Route Rendering Contract
Every route must return a fully hydrated page with no console errors. The existing visual contracts apply:
- Public pages: dark background (`#000`), off-white text (`#f5f5f0`), sidebar with tile navigation
- Admin pages: AdminShell layout with sidebar navigation, content area with data tables/forms
- Auth pages: centered form with flat inputs, monochrome styling

### Empty State Contract (D-08)
Pages queried against an empty database must:
- Render without crashes (no "Cannot read properties of undefined")
- Show the empty state copy defined above or hide the section entirely
- Home page hero renders independently of DB state (existing `Promise.allSettled` pattern)

### Seeded State Contract (D-09)
Pages queried against seeded data must:
- Display realistic Glitch Studios content (not lorem ipsum)
- Beat catalog shows filterable beats with playable audio previews
- Services page shows bookable services with pricing
- Blog shows at least one post with proper typography
- Admin dashboard shows populated data tables

### Auth Flow Contract
- Login page accepts email/password, redirects to dashboard or admin based on role
- Session persists across browser refresh (cookie-based)
- Admin routes return dashboard (not 500) for authenticated admin/owner users
- Unauthenticated admin access redirects to /login

### Payment Flow Contract (D-07)
- Beat purchase: add to cart, checkout with Stripe test card `4242 4242 4242 4242`, webhook fires, receipt email sent
- Booking deposit: select service, pick slot, pay deposit with test card, confirmation email sent
