**1. Create src/actions/admin-inbox.ts** ("use server"):

- `getMessages(filters?: { page?: number })`: requirePermission("reply_messages")
  - Query contactSubmissions ordered by createdAt desc
  - Include reply count per submission (LEFT JOIN contactReplies COUNT)
  - Paginate: 20 per page
  - Return { messages, totalPages, unreadCount }

- `getMessage(submissionId: string)`: requirePermission("reply_messages")
  - Fetch contactSubmission by ID
  - Fetch all contactReplies for this submission ordered by sentAt asc
  - Do NOT mark as read here (separate action to avoid accidental state changes -- addresses MEDIUM review concern)
  - Return { submission, replies }

- `markAsRead(submissionId: string)`: requirePermission("reply_messages")
  - UPDATE contactSubmissions SET isRead=true WHERE id=submissionId
  - Separate from getMessage to prevent accidental state changes on simple detail fetches

- `replyToMessage(submissionId: string, body: string)`: requirePermission("reply_messages")
  - Get the contact submission for sender email
  - Get current session for sender name
  - Insert into contactReplies: { submissionId, body, sentBy: session.user.name, sentAt: now }
  - Send reply email via Resend WITH proper replyTo:
    ```typescript
    await resend.emails.send({
      from: `Glitch Studios <${process.env.ADMIN_EMAIL || "admin@glitchstudios.com"}>`,
      to: submission.email,
      replyTo: process.env.ADMIN_EMAIL || "admin@glitchstudios.com",
      subject: `Re: ${submission.serviceInterest || "Your message"} - Glitch Studios`,
      text: body
    })
    ```
  - IMPORTANT: Use `from:` with ADMIN_EMAIL and `replyTo:` with ADMIN_EMAIL so client replies go to a monitored mailbox (addresses MEDIUM review concern about noreply)
  - Also mark submission as read (if not already)

- `getUnreadCount()`: COUNT contactSubmissions WHERE isRead=false
  - Used by AdminShell for sidebar badge (already wired in Plan 01)
