**1. Create src/actions/admin-blog.ts** ("use server"):

CRITICAL: Use the CORRECT schema field names. The blog schema uses `content` (not `body`), `coverImageUrl` (not `featuredImageUrl`), and has `excerpt` and `publishedAt`.

```typescript
interface BlogPostFormData {
  title: string
  slug: string
  excerpt: string          // REQUIRED -- used by public blog cards and SEO
  categoryId: string
  tagNames: string[]       // freeform tags, created if not exists
  coverImageUrl: string | null  // CORRECT field name (not featuredImageUrl)
  content: string          // CORRECT field name (not body) -- HTML from Tiptap
  status: "draft" | "scheduled" | "published"
  scheduledAt: string | null  // ISO date string
}
```

Actions:
- `createBlogPost(data: BlogPostFormData)`: requirePermission("manage_content")
  - Auto-generate slug from title if empty (reuse slugify pattern from admin-beats.ts)
  - Auto-generate excerpt from content if empty: strip HTML tags, take first 160 chars
  - If status="published": set publishedAt to NOW()
  - If status="scheduled" and scheduledAt is set: store scheduledAt, publishedAt stays null
  - If status="draft": publishedAt stays null, scheduledAt stays null
  - Set authorId from session.user.id
  - Insert into blogPosts table
  - For each tagName: find-or-create in blogTags (auto-generate slug), then insert blogPostTags junction
  - Return created post

- `updateBlogPost(id: string, data: BlogPostFormData)`: requirePermission("manage_content")
  - Update blogPosts row
  - Handle status transitions:
    - draft -> published: set publishedAt to NOW()
    - draft -> scheduled: set scheduledAt
    - scheduled -> published: set publishedAt to NOW(), clear scheduledAt
    - scheduled -> draft: clear scheduledAt
    - published -> draft: clear publishedAt (unpublish)
  - Delete-and-reinsert blogPostTags for the post (established pattern from Phase 2-3)
  - Find-or-create tags as above

- `deleteBlogPost(id: string)`: requirePermission("manage_content")
  - Delete from blogPosts (cascade deletes blogPostTags)

- `getBlogPosts(filters?: { status?: string, search?: string, page?: number })`:
  - Join blogCategories for category name
  - Filter by status if provided, ILIKE search on title
  - Paginate: 20 per page, order by updatedAt DESC
  - Return { posts, totalPages, currentPage }

- `getBlogPost(id: string)`:
  - Fetch post with category
  - Fetch tags via blogPostTags -> blogTags join
  - Return full post data for edit form (including tagNames array)
