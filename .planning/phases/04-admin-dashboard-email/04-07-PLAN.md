**4. Create src/actions/admin-newsletter.ts** ("use server"):

- `getSubscribers(filters?: { search?: string, page?: number })`: requirePermission("send_newsletters")
  - Query newsletterSubscribers with optional email ILIKE search
  - Include isActive status and tags
  - Paginate: 50 per page, order by subscribedAt DESC
  - Return { subscribers, totalPages }

- `getSubscribersBySegment(segment: "all" | "beat_buyers" | "studio_clients")`: requirePermission("send_newsletters")
  - "all": all subscribers WHERE isActive=true
  - "beat_buyers": subscribers WHERE isActive=true AND 'beat_buyer' = ANY(tags)
  - "studio_clients": subscribers WHERE isActive=true AND 'studio_client' = ANY(tags)
  - Return { subscribers, count }

- `getSegmentCounts()`: requirePermission("send_newsletters")
  - Return { all: number, beat_buyers: number, studio_clients: number } (only active subscribers)

- `sendNewsletter(data: { subject: string, body: string, segment: "all" | "beat_buyers" | "studio_clients" })`: requirePermission("send_newsletters")
  - Get active subscribers by segment
  - If no subscribers: throw "No active subscribers in this segment."
  - For each subscriber: generate per-subscriber unsubscribe URL
  - Render NewsletterBroadcastEmail with body and unsubscribeUrl
  - Chunk subscribers into batches of 100
  - Track results: sentCount, failedCount, errors[]
  - For each batch: call `resend.batch.send()`, catch errors per batch
  - After all batches:
    - Determine status: all succeeded -> "sent", some failed -> "partial_failure", all failed -> "failed"
    - Insert into newsletterBroadcasts: { subject, body, segment, recipientCount: sentCount, status, errorMessage: errors.join("; ") if any, sentAt: now, sentBy: session.user.name }
  - Return { sent: sentCount, failed: failedCount, status }

- `getBroadcasts(page?: number)`: requirePermission("send_newsletters")
  - Query newsletterBroadcasts ordered by createdAt desc
  - Paginate: 20 per page
  - Return { broadcasts, totalPages }

- `removeSubscriber(subscriberId: string)`: requirePermission("send_newsletters")
  - SET isActive=false (soft remove, not hard delete -- preserve for compliance)

- `tagSubscriberOnEvent(email: string, tag: "beat_buyer" | "studio_client")`:
  - Called from Stripe webhook (not directly from admin UI)
  - Find subscriber by email where isActive=true
  - If found and tag not already in tags array: UPDATE SET tags = array_append(tags, tag)
  - If not found: no-op (user may not be subscribed to newsletter)
