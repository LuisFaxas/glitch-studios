**1. Create src/actions/admin-services.ts** ("use server"):

- `getServices()`: query all services ordered by sortOrder, also query serviceBookingConfig to show if each service has booking config
- `createService(data)`: requirePermission("manage_content"), insert into services table
  - Data: { name, slug (auto-gen from name), type (serviceTypeEnum value), description (HTML from Tiptap), shortDescription, priceLabel, features (string[]), ctaText, sortOrder, isActive }
  - NOTE: No heroImageUrl -- the services schema does not have this column. Do NOT add it.
- `updateService(id, data)`: requirePermission("manage_content"), update services row
  - Restrict slug changes if service has booking config (would break URLs)
- `deactivateService(id)`: requirePermission("manage_content") -- SOFT DELETE
  - Check if service has bookings (query bookings WHERE serviceId=id AND status IN ('pending', 'confirmed'))
  - If has active bookings: throw "Cannot deactivate service with pending or confirmed bookings. Cancel or complete those bookings first."
  - Otherwise: SET isActive=false (soft-delete)
  - Do NOT hard-delete services that are FK targets for serviceBookingConfig, sessionPackages, bookingSeries, or bookings
- `deleteService(id)`: requirePermission("manage_content") -- HARD DELETE only if safe
  - Check if service has ANY bookings, booking config, session packages
  - If has ANY references: throw "This service has booking history. Use deactivate instead."
  - If no references: hard delete
- `reorderServices(orderedIds: string[])`: requirePermission("manage_content"), update sortOrder for each
