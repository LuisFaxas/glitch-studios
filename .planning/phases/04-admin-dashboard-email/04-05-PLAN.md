**1. Create src/actions/admin-clients.ts** ("use server"):

The client list is built by UNIONING three sources:
1. Registered users (from Better Auth's `user` table where role='user')
2. Guest purchasers (from orders.guestEmail where userId is null)
3. Guest bookers (from bookings.guestEmail where userId is null)

```typescript
interface ClientRecord {
  id: string          // user.id for registered, email hash for guests
  name: string        // user.name for registered, guestName or email prefix for guests
  email: string
  type: "registered" | "guest"
  createdAt: string   // earliest interaction date
  purchaseCount: number
  totalSpend: number
  bookingCount: number
}
```

- `getClients(filters?: { search?: string, type?: "all" | "registered" | "guest", page?: number })`: requirePermission("view_clients")

  Build a unified client list:

  Step 1: Query registered users with purchase/booking counts:
  ```sql
  SELECT u.id, u.name, u.email, 'registered' as type, u.created_at,
    COALESCE(o.purchase_count, 0) as purchase_count,
    COALESCE(o.total_spend, 0) as total_spend,
    COALESCE(b.booking_count, 0) as booking_count
  FROM "user" u
  LEFT JOIN (SELECT user_id, COUNT(*) as purchase_count, SUM(total_amount::numeric) as total_spend FROM orders WHERE user_id IS NOT NULL GROUP BY user_id) o ON o.user_id = u.id
  LEFT JOIN (SELECT user_id, COUNT(*) as booking_count FROM bookings WHERE user_id IS NOT NULL GROUP BY user_id) b ON b.user_id = u.id
  WHERE u.role = 'user'
  ```

  Step 2: Query guest emails from orders (no user account):
  ```sql
  SELECT guest_email as email, COUNT(*) as purchase_count, SUM(total_amount::numeric) as total_spend
  FROM orders WHERE user_id IS NULL AND guest_email IS NOT NULL
  GROUP BY guest_email
  ```

  Step 3: Query guest emails from bookings (no user account):
  ```sql
  SELECT guest_email as email, guest_name as name, COUNT(*) as booking_count
  FROM bookings WHERE user_id IS NULL AND guest_email IS NOT NULL
  GROUP BY guest_email, guest_name
  ```

  Step 4: Merge by email -- if a guest email also appears in registered users, attach to registered record. Otherwise create a guest record with id as `guest-${email}`.

  Filter by type if provided, ILIKE search on name and email.
  Paginate: 20 per page, order by createdAt DESC.
  Return { clients: ClientRecord[], totalPages, currentPage }

- `getClientDetail(clientId: string)`: requirePermission("view_clients")
  - If clientId starts with "guest-": extract email, query orders and bookings by guestEmail
  - Otherwise: fetch user by id, query orders WHERE userId=id, query bookings WHERE userId=id
  - For orders: join orderItems for beat details
  - Return { client: ClientRecord, purchases: Order[], bookings: Booking[] }
