<objective>
Fix Audit §D.1 bug: `/admin/media` drag-drop upload fails on drop. RESEARCH.md §Bug 4 confirmed via code read that `media-upload-zone.tsx` is TEXTBOOK CORRECT — `onDragOver` / `onDrop` both call `preventDefault`, MIME/size validation is sane, presigned-URL → XHR-PUT flow is standard. The failure is almost certainly config-level: either R2 CORS blocking the presigned PUT from the prod origin, or `getMediaUploadUrl()` / `confirmMediaUpload()` erroring due to missing `R2_*` env vars, or auth cookie not reaching the Server Action on prod hosts (related to the audit §C.4 auth cascade).

Per D-17 the manual file-picker path must keep working — if picker also fails, the bug is NOT in the drop handler. Per D-01 we diagnose before coding.

This plan follows: (1) Playwright + manual probe to isolate picker-vs-drop failure, (2) inventory env + R2 CORS, (3) apply targeted config or code fix, (4) re-verify.

Purpose: Unbreak admin media uploads before launch without rewriting a correct component.
Output: Diagnosis note with evidence, targeted fix (likely Cloudflare/Vercel config), Playwright regression.
</objective>
