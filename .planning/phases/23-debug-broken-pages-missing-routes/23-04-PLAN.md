<objective>
Fix Audit §B.1 (cross-brand tile), §B.2 (Reviews link), and mitigate §A.12 (Beats icon) — the "mobile nav double-tap bug." RESEARCH.md identified the root cause via code read: `src/components/layout/mobile-nav-overlay.tsx` lines 192-216 wrap every interactive child inside a Framer Motion `<motion.div drag="y" style={{ touchAction: "none" }}>`, which swallows the first tap. The canonical Motion v12 fix is `useDragControls()` + `dragListener={false}` so drag starts ONLY from the grab-handle.

The §A.12 Beats-icon symptom is a DIFFERENT issue (plain `<Link>`, no Motion wrap — likely a slow RSC route transition). Phase 23 mitigates with `router.prefetch` on mount. Full diagnosis is explicitly deferred to Phase 25 (Performance) per RESEARCH.md open question #5.
