<objective>
Fix Audit §A.12 mobile-checkout bug. Per D-11, D-19, and RESEARCH.md §Bug 8: the code at `/api/checkout/route.ts` has no try/catch, no env validation, and returns HTML 500 on failure. The client's `fetchClientSecret` does `response.json()` without checking `response.ok`, so the real error is swallowed and Stripe shows its generic "Something went wrong" message. Before coding a fix we need Vercel runtime logs to identify WHICH underlying failure is happening (missing `STRIPE_SECRET_KEY`, missing `NEXT_PUBLIC_SITE_URL`, empty cart, test/live mode mismatch, etc.).

The plan sequence is: (1) harden the route so failures produce actionable JSON errors, (2) pull Vercel logs with a fresh reproduction, (3) apply the targeted env/config/code fix, (4) verify on a real mobile device on HTTPS prod per D-19 (Playwright cannot simulate real iOS Safari + real Stripe prod state).

Purpose: Unblock revenue path. Mobile checkout is the last step between a customer and a purchase.
Output: Hardened route + client, diagnosis note with Vercel evidence, targeted fix, real-device verification passed.
</objective>

    export async function POST(request: Request) {
      try {
        let body: unknown
        try {
          body = await request.json()
        } catch {
          return Response.json({ error: "Invalid JSON body" }, { status: 400 })
        }

        const { items } = body as { items?: unknown }
        if (!Array.isArray(items) || items.length === 0) {
          return Response.json({ error: "Cart is empty" }, { status: 400 })
        }

        // Env validation — surface specific missing vars
        const missing: string[] = []
        if (!process.env.STRIPE_SECRET_KEY) missing.push("STRIPE_SECRET_KEY")
        if (!process.env.NEXT_PUBLIC_SITE_URL) missing.push("NEXT_PUBLIC_SITE_URL")
        if (missing.length) {
          console.error("[checkout] missing env vars:", missing)
          return Response.json(
            { error: `Server misconfigured (${missing.join(", ")})`, code: "ENV_MISSING" },
            { status: 500 }
          )
        }

        // ... existing stripe.checkout.sessions.create(...) call ...

    Example patch:
    ```tsx
    const fetchClientSecret = useCallback(async () => {
      const response = await fetch("/api/checkout", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ items }),
      })
      if (!response.ok) {
        let msg = `Checkout failed (${response.status})`
        try {
          const body = await response.json()
          if (body?.error) msg = body.error
        } catch {}
        throw new Error(msg)
      }
      const { clientSecret } = await response.json()
      if (!clientSecret) throw new Error("No clientSecret in response")
      return clientSecret
    }, [items])

    // Above the EmbeddedCheckoutProvider:
    if (items.length === 0) {
      return (
        <div className="...">
          <p>Your cart is empty.</p>
          <Link href="/beats">Browse beats</Link>
        </div>
      )
    }
    ```
