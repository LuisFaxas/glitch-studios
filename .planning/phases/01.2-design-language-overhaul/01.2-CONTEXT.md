# Phase 1.2: Design Language Overhaul - Context

**Gathered:** 2026-03-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Transform the entire site UI from generic dark theme into "Cyberpunk Metro" — a dark brutalist tile grid system with glitch hover animations, sidebar widget dashboard, flat tile-based components across all pages, and monochrome-only palette. The design spec at `.planning/DESIGN-LANGUAGE.md` is the authoritative reference for all visual decisions.

</domain>

<decisions>
## Implementation Decisions

### Logo Tile
- **D-01:** Use the official PNG logo (Untitled-2.png) as the sidebar logo tile image — not the CSS text animation
- **D-02:** Add a warm white incandescent glow around the logo tile: `rgba(255,250,240,0.15)` box-shadow — monochrome-safe, no color accent
- **D-03:** Framer Motion dramatic entrance animation on page load (scale + opacity + glitch)
- **D-04:** Glitch hover animation on the logo tile (same clip-path jitter as other tiles)

### Widget Data Strategy
- **D-05:** Now Playing widget shows a featured beat teaser from seed data (hardcoded beat name + fake progress bar). Will become live in Phase 2 when audio player ships.
- **D-06:** Studio Status widget: Claude's discretion on data source — simplest approach that works now and can be upgraded to admin-managed in Phase 4
- **D-07:** Latest Post widget pulls most recent blog post from database (already have blogPosts table and queries)
- **D-08:** Social Links widget: Instagram, YouTube, SoundCloud, X — 4 small square tiles with icon-only Lucide icons and independent glitch hover

### Service Detail Expansion
- **D-09:** Services page uses master-detail split panel on desktop — tile grid on left, detail panel on right
- **D-10:** Mobile fallback: accordion-style expansion below the tile grid (pushes content down)
- **D-11:** Selected service tile inverts (white bg/black text), detail panel appears with service description, pricing, and "Book Now" CTA

### Mobile Navigation
- **D-12:** Full-screen tile grid navigation triggered by TWO affordances: (a) tapping the GLITCH logo in the bottom tab bar center, (b) a grid icon tile as the 5th tab item
- **D-13:** NEEDS RESEARCH: How top apps implement dual-trigger mobile nav menus without confusing users. Researcher should investigate industry patterns for logo-as-menu-button combined with explicit grid/menu tile.
- **D-14:** Full-screen tile overlay shows all nav tiles + widgets in the same Metro tile layout as the desktop sidebar

### Claude's Discretion
- Studio Status widget data source implementation (D-06)
- Exact Framer Motion entrance animation parameters for logo tile
- How to handle the shadcn/ui Sidebar component replacement — rewrite from scratch or wrap existing primitives
- Whether to keep the existing shadcn card.tsx and adapt it to tile styling, or create a new Tile component from scratch
- Page transition glitch animation integration with existing Framer Motion patterns (ScrollSection wrapper)

</decisions>
