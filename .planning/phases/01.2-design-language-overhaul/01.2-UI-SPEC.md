# Phase 1.2 — UI Design Contract

> Visual and interaction contract for the Cyberpunk Metro design language overhaul. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (base-nova style) |
| Preset | base-nova, neutral base color, CSS variables enabled |
| Component library | base-ui (via shadcn base-nova) |
| Icon library | Lucide React |
| Fonts | JetBrains Mono (headings/labels/nav), Inter (body) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| `--gap-tile` | 4px | Between tiles in nav and content grids |
| `--gap-tile-lg` | 4px | Between content cards (portfolio, blog, artist grids) |
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, widget internal padding |
| md / `--pad-tile` | 16px | Default tile internal padding |
| lg / `--pad-page` | 24px | Page content margin, section internal padding |
| xl | 32px | Layout gaps between sidebar and main content |
| 2xl / `--gap-section` | 48px | Between page sections |
| 3xl | 64px | Page-level vertical spacing (hero to first section) |
| `--sidebar-width` | 280px | Fixed sidebar width on desktop |
| `--tab-bar-height` | 64px (4rem) | Mobile bottom tab bar height |

Exceptions:
- Mobile touch targets on bottom tab bar tiles: 48px minimum height (accessibility)
- Tile grid columns use CSS Grid `repeat(2, 1fr)` for sidebar nav (2-column Metro layout)
- DESIGN-LANGUAGE.md specifies `--gap-tile: 2px` (Section 8). Consolidated to 4px for grid compliance. At 280px sidebar width the visual difference is negligible and all spacing tokens now align to the 4px base unit.

## Typography

### Global Type Scale (4 sizes)

| Role | Font | Size | Weight | Line Height | Style |
|------|------|------|--------|-------------|-------|
| Display | JetBrains Mono | 64px | 700 | 1.1 | Uppercase, letter-spacing: 0.05em |
| Heading | JetBrains Mono | 28px | 700 | 1.2 | Uppercase, letter-spacing: 0.05em |
| Body | Inter | 15px | 400 | 1.6 | Normal case |
| Micro | Inter | 11px | 400 | 1.4 | Normal case |

### Component-Level Overrides

These sizes are used in specific components only and are not part of the global type scale. They are intentional extensions documented here for executor clarity.

| Size | Font | Weight | Line Height | Style | Used In |
|------|------|--------|-------------|-------|---------|
| 40px | JetBrains Mono | 700 | 1.15 | Uppercase, letter-spacing: 0.05em | Page titles (interior pages) |
| 22px | JetBrains Mono | 700 | 1.25 | Uppercase, letter-spacing: 0.05em | Section subheadings |
| 18px | JetBrains Mono | 700 | 1.3 | Uppercase, letter-spacing: 0.05em | Sidebar tile labels |
| 13px | Inter | 400 | 1.4 | Normal case | Tile sublabels, card metadata, widget track names |

Weights used: 400 (regular) and 700 (bold). No other weights.

Exceptions:
- DESIGN-LANGUAGE.md (Section 5) specifies 8 type sizes. The global scale distills these to 4 implementable tokens. The remaining 4 sizes are documented as component-level overrides above and are explicitly permitted where listed.

## Color

This phase enforces strict monochrome. Zero accent colors anywhere.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` | Page background, main content area background |
| Secondary (30%) | `#111111` | Sidebar background, tile default background, card surfaces, widget backgrounds |
| Tertiary | `#222222` | Tile borders, input borders, dividers, muted backgrounds |
| Muted text | `#888888` | Sublabels, timestamps, metadata, placeholder text |
| Subdued text | `#555555` | Disabled states, secondary info |
| Foreground | `#f5f5f0` | Primary text, tile labels, headings (off-white) |
| Inverted bg | `#f5f5f0` | Active/current page tile background, primary buttons |
| Inverted text | `#000000` | Text on inverted tiles, text on primary buttons |
| Tile hover bg | `#1a1a1a` | Subtle lift on tile hover |
| Tile hover border | `#444444` | Brightened border on tile hover |
| Tile pressed bg | `#0a0a0a` | Background on tile press/tap |
| Destructive | `#dc2626` | Destructive actions only (already defined in dark theme) |
| Logo glow | `rgba(255,250,240,0.15)` | Warm white incandescent glow around logo tile (D-02) |
| Active tile glow | `rgba(255,255,255,0.08)` | Subtle glow on current page tile |
| Scan line | `rgba(255,255,255,0.1)` | 1px horizontal sweep line on tile hover |

Accent reserved for: NOTHING. There is no accent color. Hierarchy is expressed through size, weight, motion, and black/white inversion only.

## Focal Point

**Primary focal point:** Logo tile -- 2x2 Large, warm white glow (`rgba(255,250,240,0.15)` box-shadow), Framer Motion entrance animation (scale 0.8 to 1 + opacity 0 to 1 over 600ms with cubic-bezier(0.215, 0.61, 0.355, 1), followed by single glitch-hover animation on mount completion). The logo tile is the first element the eye should find on page load.

## Tile System

### Tile Sizes (Metro Grid)

| Size | Grid Span | Use |
|------|-----------|-----|
| Small (1x1) | 1 col, 1 row | Social link icons, secondary nav, widget indicators |
| Medium (2x1) | 2 cols, 1 row | Primary nav items (Beats, Services, Portfolio, About, Blog, Contact) |
| Large (2x2) | 2 cols, 2 rows | Logo tile, featured content hero widget |
| Wide (2x1) | 2 cols, 1 row (full sidebar width) | Contact CTA tile, Now Playing widget, Latest Post widget |

### Tile States

| State | Background | Border | Text | Shadow | Animation |
|-------|-----------|--------|------|--------|-----------|
| Default | `#111` | 1px solid `#222` | `#f5f5f0` | none | none |
| Hover | `#1a1a1a` | 1px solid `#444` | `#f5f5f0` | none | glitch-hover 200ms + scan-line 400ms |
| Active (current page) | `#f5f5f0` | 1px solid `#f5f5f0` | `#000` | `0 0 20px rgba(255,255,255,0.08)` | none (locked in) |
| Pressed | `#0a0a0a` | 1px solid `#222` | `#f5f5f0` | none | scale(0.97) 100ms |

### Tile Anatomy

- Icon: Lucide, 20px, positioned top-left or center depending on tile size
- Label: JetBrains Mono, 18px, 700, uppercase, letter-spacing 0.05em
- Sublabel (optional): Inter, 13px, 400, `#888`
- Internal padding: 16px all sides
- Border-radius: 0px (sharp corners, max 2px if needed for subpixel rendering)

## Glitch Animation Keyframes

### glitch-hover (tiles, buttons, interactive elements)

```css
@keyframes glitch-hover {
  0%   { clip-path: inset(0 0 0 0); transform: translate(0); }
  20%  { clip-path: inset(20% 0 60% 0); transform: translate(-2px, 0); }
  40%  { clip-path: inset(60% 0 10% 0); transform: translate(2px, 0); }
  60%  { clip-path: inset(40% 0 30% 0); transform: translate(-1px, 0); }
  80%  { clip-path: inset(80% 0 5% 0); transform: translate(1px, 0); }
  100% { clip-path: inset(0 0 0 0); transform: translate(0); }
}
/* Duration: 200ms, timing: steps(1), iteration: 1, trigger: hover */
```

### scan-line (tile hover accent)

```css
@keyframes scan-line {
  0%   { top: -1px; opacity: 0; }
  10%  { opacity: 1; }
  90%  { opacity: 1; }
  100% { top: 100%; opacity: 0; }
}
/* Duration: 400ms. 1px white line (rgba(255,255,255,0.1)) sweeps top-to-bottom via ::after pseudo-element */
```

### Content card hover (subtler variant)

- Same glitch-hover keyframes but duration: 100ms and translate values halved (1px instead of 2px)

### Page transition glitch

- On route change: main content area plays glitch-hover at 200ms full-area
- Content fades in after glitch clears (opacity 0 to 1, 150ms ease-out)
- Framer Motion `AnimatePresence` wrapping page content with `layoutId` for shared tile-to-page animations

### Logo tile entrance (Framer Motion)

- Initial: `{ scale: 0.8, opacity: 0 }`
- Animate: `{ scale: 1, opacity: 1 }`
- Transition: `{ duration: 0.6, ease: [0.215, 0.61, 0.355, 1] }`
- Plus glitch-hover animation plays once on mount completion

### Reduced motion

- All glitch and scan-line animations replaced with simple opacity fade (200ms)
- `prefers-reduced-motion: reduce` media query wrapping animation disablement (already in globals.css)

## Component Patterns

### Page Headers

- No hero banners on interior pages
- Title: JetBrains Mono, 40px, 700, uppercase, left-aligned
- Optional subtitle: Inter, 15px, 400, `#888`, below title
- Divider: 1px `#222` horizontal rule below header, 24px margin-bottom

### Content Cards (portfolio, blog, artists)

- Background: `#111`
- Border: 1px solid `#222`
- Border-radius: 0px
- Gap between cards: 4px
- Image: fills tile completely, no internal padding on image area
- Text: below image or overlaid at bottom with gradient fade (`linear-gradient(transparent, #000)`)
- Hover: subtle glitch-hover (100ms, 1px displacement)

### Service Tiles (services page -- D-09, D-10, D-11)

- Desktop: master-detail split panel. Tile grid on left (2-column), detail panel on right
- Tile grid: each service is a Medium (2x1) or Small (1x1) tile depending on hierarchy
- Selected tile: inverts to white bg / black text (same as active nav tile state)
- Detail panel: appears on right with service description, pricing table, "Book Now" CTA button
- Mobile: accordion-style expansion below the tile grid (tile tap pushes content down to reveal detail)
- Transition: detail panel fades in 150ms, tile inversion is instant

### Forms (contact, auth)

- Input fields: flat, 0px border-radius, 1px `#333` border, `#111` background, `#f5f5f0` text
- Focus state: border changes to `#f5f5f0`, add `0 0 8px rgba(255,255,255,0.1)` glow
- Placeholder text: `#555`
- Buttons: flat tiles matching nav tile visual language
- Primary button: inverted -- `#f5f5f0` background, `#000` text, glitch-hover on hover
- Secondary button: default tile styling -- `#111` background, `#f5f5f0` text

### Mobile Bottom Tab Bar (D-12, D-13, D-14)

- 5 icon-only tile tabs: Beats, Services, Portfolio, Blog, Grid (menu trigger)
- Tab tile height: 48px minimum
- Icon size: 24px, Lucide icons
- Active tab: inverted (white bg, black icon)
- Inactive tab: `#111` bg, `#f5f5f0` icon
- Center position: GLITCH logo as additional menu trigger (tapping opens full-screen tile overlay)
- Full-screen overlay: all nav tiles + widgets in Metro tile layout matching desktop sidebar
- Overlay entrance: glitch transition (200ms) covering full viewport

## Anti-Patterns (from DESIGN-LANGUAGE.md Section 9)

These are contractual violations. The executor must not introduce:

- Rounded corners beyond 2px on any element
- Any color accent (cyan, neon green, purple, blue, etc.)
- Soft shadows or blur-based depth (use borders and contrast only)
- Thin/delicate UI -- tiles must feel chunky and substantial
- Generic hover states without glitch energy
- Empty space in the sidebar -- fill with widgets
- Gradients on UI elements (gradients only on image overlays)

## Accessibility Requirements

- All tiles must be keyboard-focusable with visible focus indicator (1px `#f5f5f0` outline, 2px offset)
- Tile hover animations must respect `prefers-reduced-motion: reduce`
- Minimum contrast ratio: `#f5f5f0` on `#111` = 15.7:1 (AAA), `#888` on `#111` = 4.9:1 (AA)
- `#555` on `#111` = 2.9:1 -- use only for decorative/non-essential text (timestamps, sublabels alongside visible labels)
- Mobile touch targets: 48px minimum on all interactive tiles
- Sidebar navigation tiles must use `<nav>` landmark with `aria-label="Main navigation"`
- Current page tile must use `aria-current="page"`
- Social link tiles must have `aria-label` with platform name (icon-only)
- Bottom tab bar tiles must have `aria-label` with the destination name (e.g., `aria-label="Beats"`, `aria-label="Services"`, `aria-label="Portfolio"`, `aria-label="Blog"`, `aria-label="Menu"`)
