## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside tile labels (`gap-1`) |
| sm | 8px | Compact stack inside tile metadata (`gap-2`, `p-2` chips) |
| md | 16px | Tile inner padding (`p-4`), grid gaps on mobile |
| lg | 24px | Section padding-y on mobile, gaps between admin form fields (`gap-6`) |
| xl | 32px | Desktop grid gap between the 3 home tiles (`gap-8`) |
| 2xl | 48px | Vertical breathing room between home "Our Work" section header and grid |
| 3xl | 64px | Section padding-y on desktop (`py-16`) — matches existing `<ScrollSection>` rhythm |

Exceptions:
- Hover-preview iframe is full-bleed inside the tile aspect-video frame (zero inset). Intentional — the preview must read as a video, not a card-within-a-card.
- Play-icon overlay button is centered on the thumbnail at `48px` (`size-12`) hit target; this is intentionally larger than the 44px mobile minimum for thumb confidence on the home grid.

## Typography

Maximum 4 declared pixel sizes. Final scale (matches existing `<VideoCard>` and `<VideoPortfolioCarousel>` precedent):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 11px (`text-[11px]`, mono, uppercase, tracking-wide) | 700 | 1.3 |
| Body | 14px (`text-sm` — cards, descriptions, admin form labels, primary CTA, chips) | 400 | 1.5 |
| Heading (H3 tile title) | 18px (`text-lg`, mono, uppercase) | 700 | 1.2 |
| Display (H2 section "Our Work") | 28px (`text-2xl`, mono, uppercase, hover-glitch) — single size at all breakpoints | 700 | 1.1 |

**Consolidation notes (per checker revision):**
- Dropped the 16px body tier — primary CTA and admin form labels now use 14px (`text-sm`), matching existing carousel CTA + card body precedent.
- Dropped the responsive 36px desktop upscale on H2 display — H2 stays at 28px (`text-2xl`) at all breakpoints. This is a minor downscale from existing carousel (`text-2xl md:text-4xl`) and is intentional to honor the 4-size cap. The existing carousel will be replaced by `<HomeFeaturedWorkGrid />` in this phase, so the new component lands at 28px from day one.
- Final declared sizes: **11px / 14px / 18px / 28px** (4 distinct values).

**Rules carried from project memory (`feedback_glitch_headers.md`):**
- Every H1/H2/H3 wraps in `<HoverGlitchHeading>` (or the existing `<GlitchHeading>` already in use on the carousel — use whichever the planner chooses to standardize on; both honor `prefers-reduced-motion`).
- No auto-running animations on headings. Hover-only glitch.
- All headings remain font-mono uppercase to match site-wide rhythm.

## Color

Anchored to the existing flat-monochrome palette already declared in `src/styles/globals.css` and used across `<VideoCard>`, `<ReviewVideoEmbed>`, and `<VideoPortfolioCarousel>`. No new color tokens introduced in this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` (true site bg) and `#111111` (tile bg) | Page background, tile surface, iframe fallback |
| Secondary (30%) | `#222222` (border default), `#1a1a1a` (chip bg), `#888888` (muted text) | Tile borders, metadata chips, secondary copy, admin form inputs |
| Accent (10%) | `#f5f5f0` (off-white) | Primary text, primary CTA fill, play-icon glyph fill on hover, admin "Save" button |
| Destructive | `#ef4444` (existing Tailwind `red-500`, used for admin "Remove video" / "Delete attachment" only) | Confirmation dialogs only — never used as ambient/decorative |

**Accent reserved for:**
1. Tile title text (H3) — `#f5f5f0`
2. Primary CTA `See all work →` text fill on hover — invert to `bg-[#f5f5f0] text-[#0a0a0a]`
3. Play-icon overlay glyph stroke on tile hover (`text-[#f5f5f0]` over a `bg-[#0a0a0a]/60` scrim)
4. Admin "Save video", "Attach video" primary buttons
5. Drag-handle active state (`#f5f5f0` on grab)

**Accent NEVER used for:**
- Border colors (always `#222`, hover `#444`)
- Body copy (always `#888` for muted, `#f5f5f0` only for primary text)
- Backgrounds in tile bodies (always `#111`)
- Section headers (already mono uppercase white per site-wide convention; not "accent-treated" beyond hover-glitch)

## Interaction States — `<MediaEmbed>` (canonical contract)

| State | Visual | Trigger | Notes |
|-------|--------|---------|-------|
| Default (thumbnail) | `<Image>` cover, no overlay glyph visible | initial render | Zero YouTube JS loaded (D-01). Lazy `<Image>`. |
| Hover (desktop, full-motion) | Thumbnail swaps to muted iframe (no controls, looped) | `pointer: fine` + `mouseenter` + `prefers-reduced-motion: no-preference` | D-02. Iframe `src=youtube-nocookie.com/embed/{id}?autoplay=1&mute=1&controls=0&modestbranding=1&loop=1&playlist={id}` |
| Hover (desktop, reduced-motion) | Thumbnail + centered play-glyph fades in | `mouseenter` + `prefers-reduced-motion: reduce` | D-05. NO iframe swap. Static. |
| Hover (mobile / `pointer: coarse`) | Same as default — no hover state at all | n/a | D-03. Touch is tap-only. |
| Click / Tap | Full iframe with sound + controls (autoplay) | `click` or `touchend` | `?autoplay=1&controls=1` (no mute). |
| Loading (during iframe swap) | Thumbnail stays visible until iframe `onLoad` fires | iframe insertion | No skeleton flash. The thumbnail IS the skeleton. |
| Focus (keyboard) | Same overlay as hover-reduced-motion (centered play glyph) + `outline-2 outline-[#f5f5f0]/40 outline-offset-2` | `:focus-visible` on the wrapping button | Accessibility: tab focus must look like hover-static. Enter/Space triggers full play. |

**Hit target:** Entire aspect-video frame is the click target. Wrap with `<button type="button">` so it's a real interactive element (not a div with onClick).

**ARIA:** `aria-label="Play video: {title}"`. When playing, swap to `aria-label="Pause video: {title}"` if the iframe is rendered (YouTube controls take over from there).

- Position: centered below grid, `mt-12 md:mt-16`

## Accessibility Contract

| Surface | Requirement |
|---------|-------------|
| `<MediaEmbed>` wrapper button | `aria-label="Play video: {title}"`, keyboard-activatable (Enter/Space), focus-visible outline `outline-2 outline-[#f5f5f0]/40` |
| Hover-autoplay swap | Skipped entirely under `prefers-reduced-motion: reduce` (D-05) — verify with Playwright `emulateMedia({ reducedMotion: 'reduce' })` |
| Play-icon overlay | `aria-hidden="true"` (decorative; the wrapping button carries the label) |
| Iframe (full-play state) | `title="{video title}"`, `loading="lazy"`, `allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"`, `allowFullScreen` |
| Admin "Attach video" dialog | Focus trap inside dialog, Esc closes, focus returns to trigger button on close (shadcn Dialog handles this — verify) |
| Admin `<MediaItemAttachmentList>` icon-only "Remove" button (`<Trash2>`) | **Required:** `aria-label="Remove {video title}"` per row (e.g., `aria-label="Remove Beat preview — Midnight Frequency"`). Falls back to `aria-label="Remove video"` only if the row title is unavailable at render time. The icon itself remains `aria-hidden="true"`. |
| Admin `<MediaItemAttachmentList>` icon-only drag-handle (`<GripVertical>`) | `aria-label="Reorder {video title}"`. Icon `aria-hidden="true"`. |
| Drag-to-reorder | Keyboard alternative: `↑` / `↓` arrow keys move focused row up/down (planner: use `dnd-kit/sortable` keyboard sensor) |
| Color contrast | `#f5f5f0` on `#0a0a0a` = 17:1 (AAA). `#888` on `#111` = 4.5:1 (AA). Already site-wide; no new low-contrast pairs introduced. |

## Verification Hooks (for ui-checker / ui-auditor)

The auditor will look for:

1. Every public surface that renders a video uses `<MediaEmbed>` (no rogue `<iframe src="youtube.com">` left over). Grep target: `iframe.*youtube` returns ONLY `MediaEmbed.tsx`.
2. Every embed uses `youtube-nocookie.com`, never `youtube.com` (D-04).
3. The home grid renders `null` when no items are attached (no placeholder slots).
4. `prefers-reduced-motion: reduce` Playwright pass: hovering a tile does NOT swap iframes; only the static play-glyph appears.
5. `pointer: coarse` Playwright pass (mobile emulation): hovering does nothing; only tap triggers full-play.
6. Admin "Add video" with a non-YouTube URL surfaces the enumeration-safe error copy verbatim.
7. Detach destructive flow shows the AlertDialog with the exact "Remove this video?" copy before mutation.
8. All H2/H3 headings are wrapped in a hover-glitch heading component (no plain `<h2>` text in rendered DOM for new components).
9. Every icon-only button in `<MediaItemAttachmentList>` has a non-empty `aria-label`. Grep target: `<Trash2` and `<GripVertical` inside that file must each be wrapped by an element carrying `aria-label`.
10. Typography audit: only 4 distinct font-size values appear in new component CSS — `11px`, `14px` (`text-sm`), `18px` (`text-lg`), `28px` (`text-2xl`). No `text-base`, `text-xl`, `text-3xl`, `text-4xl`, or arbitrary `text-[Npx]` outside this set in new files.
