# Phase 11 — UI Design Contract

> Visual and interaction contract for the Portfolio detail navigation and index refinement. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | base-nova (configured in `components.json`, baseColor neutral) |
| Component library | radix (via shadcn) |
| Icon library | lucide-react |
| Font | JetBrains Mono (headers/labels/mono), Inter (body/excerpts) — both via next/font |

Aesthetic: Cyberpunk Metro — flat black & white surfaces, mono-uppercase headers, hover-only RGB-split glitch (site-wide rule). No auto-running heading animations.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, metadata-row separator spacing, grid gap between portfolio cards |
| sm | 8px | Category chip internal padding, metadata row item gap |
| md | 16px | VideoCard inner padding (`p-4`), hero body-block gap, sticky footer padding |
| lg | 24px | Grid-to-hero vertical gap, hero content block padding |
| xl | 32px | Section padding above/below prev/next footer, category-chip-row → grid spacing |
| 2xl | 48px | Hero banner vertical padding (desktop), h1 → chip-row spacing, sticky footer internal spacing |
| 3xl | 64px | Page top padding under global nav (desktop) |

Grid gap: `gap-1` (4px) between VideoCards — intentional Metro tile density (locked by Phase 10 D-03 for consistency).

Card image aspect: `aspect-video` (16:9) — applies to every VideoCard image slot AND the featured hero image.

Sticky footer spacing: `md` (16px) internal padding on mobile-collapsed state; `2xl` (48px) on desktop expanded state for title/nav breathing room.

Exceptions:
- Featured hero headline uses `clamp(28px, 5vw, 48px)` font-size (fluid), not the token table. This is intentional per D-08.
- Placeholder (no thumbnail/no YouTube) headline uses `clamp(18px, 3vw, 28px)` per D-12.

## Typography

Three sizes, two weights, two families. Fluid clamps used only for the featured hero headline and placeholder headline per D-08/D-12.

| Role | Family | Size | Weight | Line Height | Usage |
|------|--------|------|--------|-------------|-------|
| Body / Description | Inter | 13px (card), 14px (hero) | 400 | 1.5 | Portfolio item descriptions, hero excerpt |
| Label / Metadata | JetBrains Mono | 11px | 700 | 1.4 | Type chip (`CASE STUDY` or `VIDEO`), category badge, year label, prev/next footer title snippets (or arrows-only on mobile) |
| Heading | JetBrains Mono | 20px (`text-lg`) | 700 | 1.2 | VideoCard title |
| Display | JetBrains Mono | clamp(28px, 5vw, 48px) | 700 | 1.1 | Featured hero headline (wrapped in `GlitchHeading`), page `PORTFOLIO` h1 |
| Sticky Footer | JetBrains Mono | 14px (mobile arrow), 13px (desktop title snippet) | 700 | 1.2 | Prev/next navigation titles or arrow-only labels |

Rules:
- All mono text: `uppercase` + `tracking-wide` (tracking 0.02em–0.05em).
- All headings wrapped in `<GlitchHeading>` — hover-only RGB-split glitch, no auto-running animation (site-wide rule).
- VideoCard titles use `line-clamp-2` per D-11 (mirrors PostCard from Phase 10).
- Descriptions use `line-clamp-2` per D-11 (mirrors PostCard from Phase 10).
- Metadata row (type chip + year) at 11px mono per D-11.
- Sticky footer title text abbreviated or converted to arrows-only on viewports where full `← PREV {title}` doesn't fit (D-01 collapse rule).

## Color

Locked palette (Phase 11 may not introduce new hex values):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` | Page background, featured hero base, VideoCard base, sticky footer base |
| Secondary (30%) | `#0a0a0a` / `#111111` | Card surface contrast, hero gradient mid-stop, category-chip idle background (`#222222`), sticky footer border |
| Accent (10%) | `#f5f5f0` | Primary CTAs ("VIEW WORK"), active-chip inverse fill, glitch-hover inversion, sticky footer text |
| Muted | `#888888` | Description text, metadata labels (year, "MIN READ" equivalents) |
| Subtle border | `#222222` | Chip backgrounds, card hairline borders, metadata dot separators, sticky footer top-border |
| Destructive | n/a | No destructive actions in this phase |

Accent reserved for:
- "VIEW WORK" CTA on the featured hero (primary CTA — solid `#f5f5f0` background, `#000000` text)
- Active category chip (inverse: `bg-[#f5f5f0] text-[#000000]`)
- Glitch-hover RGB-split overlay accent on headings and VideoCard (the site-wide hover-only effect)
- Sticky footer text (`#f5f5f0` on `#111111` bg) — high contrast for readability while scrolling

Accent is NOT used for: card backgrounds, chip idle state, metadata rows, hero description, page chrome, type chip backgrounds, or placeholder images. These stay within dominant/secondary/muted.

Featured hero gradient (D-08): vertical gradient overlaying the 16:9 cover image. Stops: `from-black/90 via-black/50 to-black/0` (top → bottom). All values derive from `#000000` at varying alpha — no new hex values introduced.

Placeholder radial gradient (D-12): `radial-gradient(ellipse at center, #111111 0%, #0a0a0a 60%, #000000 100%)` with the existing slice/line-repeat texture overlay. Matches Phase 10 D-04 placeholder pattern.

Sticky footer visual (D-01): `bg-[#111111] border-t border-[#222222] text-[#f5f5f0]`. Collapses to thinner design on mobile (arrows-only navigation).

## Interaction Contract

### Index Page (`/portfolio`)

| Interaction | Behavior |
|-------------|----------|
| Hover a heading (page h1, featured hero headline, or card title) | RGB-split glitch effect runs only on hover (site-wide rule, no auto-run) |
| Hover a VideoCard | Existing glitch-hover overlay; cursor: pointer; whole card is the link to `/portfolio/{slug}` |
| Tap a category chip | Filters the grid to that category; state can be client-only (useState) or URL-based (nuqs — Claude's discretion); resets scroll to grid top |
| Filter behavior | Category filter is optional cosmetic polish; recommended approach is client-only state (no URL param) to keep routing simple. If planner uses `nuqs`, ensure deep-links honor the selected category on page load. |
| Featured hero image load | Uses `next/image` with `priority` prop for LCP optimization (desktop hero image is above-the-fold) |
| Featured hero visibility | Shows only if `isFeatured = true` exists in the query result. If no featured item, page renders h1 + category chips + grid with no hero placeholder. Per D-10. |
| Placeholder card image | Renders per D-12 (mono title on radial gradient + slice texture); matches Phase 10 PostCard placeholder pattern |
| Mobile layout | Grid switches to single column at mobile breakpoints; featured hero stacks full-width before grid; category chips horizontal-scroll on overflow without visible scrollbar |

Batch behavior: All active items render at once (no pagination — portfolio is typically smaller than blog).

---

### Detail Page (`/portfolio/[slug]`)

| Interaction | Behavior |
|-------------|----------|
| Page type branching | Route renders based on `item.type`: `case_study` → full layout (hero + 4 sections); `video` → minimal layout (embedded player, title, category, description, metadata) |
| Sticky footer visibility | Always visible while reading detail page; pins to bottom of viewport |
| Sticky footer keyboard nav | Desktop: `←` arrow key = prev, `→` arrow key = next. Mobile: no keyboard (not typical mobile workflow). Wrap-around applies (last → first, first → last). |
| Sticky footer swipe nav | Mobile: horizontal swipe on page body = prev/next. Use `touch-action: pan-y` to preserve vertical scroll. Swipe threshold: planner's discretion (typical: 50–100px horizontal displacement). No swipe on desktop (keyboard covers it). |
| Sticky footer click nav | Tap/click the sticky footer left section (or `←` button) = prev; right section (or `→` button) = next. Cursor pointer on hover. |
| Prev/next wrap-around | Prev on first item loops to last; next on last item loops to first. No dead-end state. |
| Transition on prev/next | Animation between detail pages: Claude's discretion (fade, slide, or none). Recommend: minimal (fade or none) to keep focus on content. |
| Back-to-portfolio link | Existing text link on case-study detail page (`← Back to Portfolio`). Behavior: navigates to `/portfolio` with no filter/position memory. Works alongside the sticky footer — no conflict required. |
| Video detail minimal layout | No case-study sections (Client, Challenge, Approach, Result); instead: YouTube embed (with play button before first interaction), title in GlitchHeading, category chip (if present), description line-clamp-2, metadata row (type + year). All contained in single column. |
| Case study detail layout | Preserved verbatim per D-07 (hero media, back link, h1 in GlitchHeading, category chip, description, 4 sections). Only addition: sticky prev/next footer below sections. |

Prev/next data fetching: Use server-side Drizzle query for neighbors (asc(sortOrder) ordering) cached via `react.cache()` to avoid repeated DB hits.

## Mobile Rendering Targets

Per success criterion 3, must render correctly at 375px, 390px, 430px (Phase 7.2 standard):

| Viewport | Behavior |
|----------|----------|
| 375px | Single-column grid; featured hero full-width; category chips horizontal-scroll; sticky footer collapses to arrows-only (if text doesn't fit); swipe gesture active on detail page |
| 390px | Same as 375px (typically) |
| 430px | Single-column grid; featured hero full-width; category chips horizontal-scroll; sticky footer may show abbreviated titles depending on title length (planner adjusts break point); swipe gesture active |
| 768px (md breakpoint) | Two-column grid; featured hero full-width; category chips static layout; sticky footer shows full titles (`← PREV {snippet}` | `NEXT {snippet} →`) |
| 1024px (lg breakpoint) | Three-column grid; featured hero full-width; category chips static layout; sticky footer shows full titles |

Grid columns: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` per D-08 (matches Phase 10).

Sticky footer breakpoint: Recommend conditional render at `hidden md:flex` for "desktop mode" (full titles) and default compact on smaller screens. Exact breakpoint is Claude's discretion during implementation.

## Accessibility & Performance

- **Focus management:** Sticky footer nav buttons are keyboard-focusable (tab order after page h1 and cards)
- **ARIA labels:** Sticky footer buttons should have `aria-label` (e.g., "Previous portfolio item" / "Next portfolio item")
- **Motion:** Glitch-hover overlay respects `motion-reduce` (existing `motion-reduce:hidden` pattern from Phase 1.4)
- **Link semantics:** All portfolio item links are true `<Link>` or `<a>` elements (not buttons with onClick handlers)
- **Image optimization:** Featured hero thumbnail uses `next/image` with `priority`; card thumbnails use default lazy loading
- **Keyboard trap:** Sticky footer is not a trap; focus can move past it to other page elements
