# Phase 29.3-06 — Final macOS Verification (post-root-cause fix)

## What changed since Plan 29.3-05 failed

Plan 29.3-05 ran on commit `e6a1ba2` and Safari froze on chip click at 03:15Z. Codex then reproduced the freeze locally on **headless Chromium with REAL MOUSE input** (not synthetic `.click()`) and identified the root cause:

> When React state updates run synchronously inside a native pointerdown/click event handler, the resulting style invalidation fires synthetic pointer events that re-enter the input handler before it has unwound. The browser's pointer/style processing cycle starts feeding itself.

Five commits + the 29.3-06 gating commit address the full fix family:

| Commit | What |
|---|---|
| `6af8177` | `setFilters` defers `setFiltersState` via `setTimeout(0)`; `CustomDropdown.setOpen` defers via `setOpenAfterInput`. Verified by Codex on real-mouse Chromium + Firefox + WebKit headless × 6 cycles each. |
| `12214c7` | Price slider commits on release, revert table-fixed |
| `ba1e747` | Replace Base UI Slider with hand-rolled `PriceRangeSlider`, defer mobile sheet open |
| `c9d8c60` | Comprehensive Playwright timeline test passes 4/4 cross-engine on the post-fix tree |
| `3cf5991` | (this plan's gating commit) Replace BPR `<Tooltip>` with native `title=`, additional defer cycles, page-lifecycle close handler |

The diagnostic premise is unchanged: Linux-headless can prove the regression class is gone, but only physical macOS hardware can authenticate that the original Metal/WebRender crash class is gone.
