# Phase 06.1: Homepage Flair - Context

**Gathered:** 2026-03-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Add wow factor to the homepage — splash logo animation on first visit, scroll-driven Framer Motion effects, animated scroll indicator, replace text-based GlitchLogo with the real logo image in the hero, increase beat card spacing, and implement scroll-aware collapsible sidebar so the splash and hero logos are perfectly centered. No new features or backend changes.

</domain>

<decisions>
## Implementation Decisions

### Logo
- **D-01:** Hero section MUST use the real logo image (`/logo-white.png`) via the `LogoTile` component pattern (CSS background-image with glitch layers), NOT the text-based `GlitchLogo` component that just renders "GLITCH" in monospace.
- **D-02:** Footer should also switch from text-based `GlitchLogo` to the real logo image for consistency.

### Splash/Intro Animation
- **D-03:** Claude's Discretion — create a splash/intro animation where the Glitch logo appears large on first visit, then transitions into the hero layout. Research ReactBits catalog for suitable entrance effects. Should feel dramatic and memorable.
- **D-04:** Splash should only play on first visit per session (use sessionStorage to track). Returning visitors skip directly to the hero.

### Scroll Effects
- **D-05:** Claude's Discretion — enhance scroll-driven Framer Motion effects throughout homepage sections. Current `ScrollSection` wrapper only does basic opacity+translateY. Add more dynamic effects (parallax, stagger, text reveals, etc.). Research ReactBits components for inspiration.
- **D-06:** Add an animated scroll indicator arrow at the bottom of the hero section. Should pulse/bounce to signal "keep scrolling."

### Sidebar Collapse (scroll-aware)
- **D-10:** Sidebar starts COLLAPSED on page load (thin icon strip, no labels, no logo). This allows splash and hero logos to be perfectly centered on the full viewport with no offset.
- **D-11:** When user scrolls past the hero section, sidebar AUTOMATICALLY expands to full width (240px) with labels and logo visible. Framer Motion animation for smooth expansion.
- **D-12:** While in the hero section, user can manually expand the sidebar by clicking/hovering on the collapsed strip — don't trap them.
- **D-13:** When user scrolls back up to the hero section, sidebar collapses again to restore the clean centered look.
- **D-14:** The collapsed sidebar should show navigation icons only (no text labels, no logo). The LogoTile is hidden in collapsed state — the hero IS the logo.
- **D-15:** This is a homepage-specific behavior. On other pages (beats, services, etc.) the sidebar should default to expanded. The collapse is tied to the hero section scroll position on the homepage only.

### Beat Card Spacing
- **D-07:** Increase spacing between featured beat cards in the carousel. Current Metro `gap-[2px]` is too tight for this section — beats need breathing room to stand out individually.

### Design Language
- **D-08:** All animations must follow Cyberpunk Metro aesthetic — monochrome, no color, sharp edges. Glitch distortion effects are on-brand.
- **D-09:** Respect `prefers-reduced-motion` — all animations must have reduced-motion fallbacks.

### Claude's Discretion
- Specific splash animation technique and timing
- Scroll effect types per section (parallax vs stagger vs wipe etc.)
- Scroll indicator design and animation style
- Exact beat card gap value
- Collapsed sidebar icon strip width (48-64px range)
- Scroll threshold for sidebar expand/collapse trigger
- Sidebar expand/collapse animation easing and duration

</decisions>
