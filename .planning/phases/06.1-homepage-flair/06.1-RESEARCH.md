# Phase 06.1: Homepage Flair - Research

**Researched:** 2026-03-30
**Domain:** Frontend animation (Framer Motion, ReactBits, CSS keyframes)
**Confidence:** HIGH

## Summary

This phase adds "wow factor" to the homepage through five discrete changes: (1) a splash/intro animation for first-visit users, (2) replacing the text-based GlitchLogo with the real logo image in hero and footer, (3) enhanced scroll-driven Framer Motion effects, (4) an animated scroll indicator arrow in the hero, and (5) increased beat card spacing. All animation work uses the already-installed `motion` package (v12.38.0) and the ReactBits MCP server available at `~/.claude/.mcp.json` for component code retrieval.

The existing codebase has strong foundations: `LogoTile` with CSS glitch layers, `ScrollSection` wrapper with `whileInView`, and the Cyberpunk Metro design language. This phase enhances existing patterns rather than replacing them. No new npm dependencies are needed -- `motion/react` covers all scroll and animation needs, and ReactBits components are copy-paste (not npm packages).

**Primary recommendation:** Build a full-screen splash overlay using Framer Motion `AnimatePresence` + `motion.div` with scale/opacity/glitch keyframes, tracked via `sessionStorage`. Enhance `ScrollSection` to support multiple animation variants (parallax, stagger, clip-reveal). Use custom Framer Motion for the scroll indicator rather than a ReactBits component, since a simple bouncing chevron is trivial and avoids copy-paste overhead.

### Pattern 1: Splash Overlay with sessionStorage Gate
**What:** A full-screen overlay that plays the logo animation on first visit, then unmounts. Subsequent visits in the same session skip it entirely.
**When to use:** First page load of a session (no sessionStorage key set).

```typescript
// Source: Framer Motion AnimatePresence + sessionStorage pattern
"use client"

import { useState, useEffect } from "react"
import { AnimatePresence, motion } from "motion/react"

export function SplashOverlay({ children }: { children: React.ReactNode }) {
  const [showSplash, setShowSplash] = useState(false)
  const [mounted, setMounted] = useState(false)

  useEffect(() => {
    setMounted(true)
    const seen = sessionStorage.getItem("glitch-splash-seen")
    if (!seen) {
      setShowSplash(true)
      // Lock body scroll during splash
      document.body.style.overflow = "hidden"
    }
  }, [])

  const handleSplashComplete = () => {
    sessionStorage.setItem("glitch-splash-seen", "true")
    document.body.style.overflow = ""
    setShowSplash(false)
  }

  // On server or before hydration, render children only
  if (!mounted) return <>{children}</>

  return (
    <>
      <AnimatePresence>
        {showSplash && (
          <motion.div
            key="splash"
            className="fixed inset-0 z-50 flex items-center justify-center bg-black"
            initial={{ opacity: 1 }}
            exit={{ opacity: 0 }}
            transition={{ duration: 0.5 }}
            onAnimationComplete={(def) => {
              // Only trigger on exit animation
              if (def === "exit") handleSplashComplete()
            }}
          >
            {/* Logo with glitch entrance */}
            <motion.div
              className="w-[80vw] max-w-[600px]"
              initial={{ opacity: 0, scale: 1.5 }}
              animate={{ opacity: 1, scale: 1 }}
              transition={{
                duration: 1.2,
                ease: [0.215, 0.61, 0.355, 1], // easeOutCubic
              }}
            >
              {/* Reuse logo-tile CSS pattern here */}
              <div className={styles.glitchWrapper}>
                <div className={styles.glitchImg} />
                <div className={styles.glitchLayer1} />
                <div className={styles.glitchLayer2} />
              </div>
            </motion.div>
          </motion.div>
        )}
      </AnimatePresence>
      {children}
    </>
  )
}
```

**Key details:**
- `useState(false)` for `showSplash` prevents SSR flash (splash only shows after client hydration)
- `sessionStorage.getItem` check happens in `useEffect` (client only)
- Body scroll locked during splash, restored after
- `AnimatePresence` handles the exit animation
- Splash duration: ~2.5s total (1.2s logo entrance + 0.8s hold + 0.5s fade out)
- `prefers-reduced-motion`: skip splash entirely or reduce to simple fade

function ScrollIndicator() {

### Anti-Patterns to Avoid
- **Constant glitch animation on idle:** Logo glitch only on hover/entrance, never looping. Constant animation is distracting and violates design language ("idle state is clean").
- **Color in animations:** No neon, no RGB split with actual colors. Monochrome only per D-08.
- **Heavy scroll-linked blur/filter:** Animating `blur()` or `filter` on scroll tanks performance. Stick to `transform` and `opacity`.
- **Splash on every page load:** Only homepage, only first visit per session. Other pages must load instantly.
- **Blocking render with splash:** Page content should exist behind the splash overlay. When splash dismisses, content is already there -- no loading state.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Scroll progress tracking | Manual scroll event listeners + RAF | `useScroll` from `motion/react` | Handles all edge cases (resize, container changes, RAF batching). Already installed. |
| Entrance animations | Custom IntersectionObserver + CSS classes | `whileInView` from `motion/react` | Built-in viewport detection, configurable margin/threshold, `once` option. |
| Spring physics | Custom easing math | `useSpring` from `motion/react` | Physically accurate spring, configurable stiffness/damping. |
| Exit animations | Manual unmount timing | `AnimatePresence` from `motion/react` | Handles component exit animations before DOM removal. Critical for splash overlay. |

**Key insight:** `motion/react` v12 covers every animation need in this phase. Adding ReactBits components would increase code surface area without meaningfully improving the result. The glitch aesthetic is already implemented in CSS modules -- extending it is simpler than adapting a third-party component.

## Common Pitfalls

### Pitfall 1: SSR Hydration Mismatch with sessionStorage
**What goes wrong:** Checking `sessionStorage` during render causes server/client mismatch. Server renders "no splash" while client wants to show splash.
**Why it happens:** `sessionStorage` is browser-only. Any render-time check produces different output on server vs client.
**How to avoid:** Initialize splash state as `false`. Check `sessionStorage` inside `useEffect` and set state there. This ensures server and initial client render match (both show no splash), then client mounts splash after hydration.
**Warning signs:** React hydration warnings in console, flash of content before splash appears.

### Pitfall 2: Body Scroll Lock Leaking
**What goes wrong:** If splash component unmounts unexpectedly (navigation during splash), `overflow: hidden` stays on body forever.
**Why it happens:** Cleanup function in useEffect not properly restoring scroll.
**How to avoid:** Always restore `document.body.style.overflow = ""` in both the completion handler AND a useEffect cleanup return.
**Warning signs:** Page becomes unscrollable after navigating away during splash.

### Pitfall 3: Scroll Indicator Visible During Splash
**What goes wrong:** The bounce-down chevron appears on top of the splash overlay.
**Why it happens:** Splash is `z-50` but indicator is rendered inside hero which is beneath it.
**How to avoid:** Conditionally render the scroll indicator only after splash completes, or ensure z-index layering is correct. The splash overlay at `z-50` should cover everything.
**Warning signs:** Chevron bouncing over the black splash screen.

### Pitfall 4: Performance with Multiple Scroll-Linked Animations
**What goes wrong:** Janky scrolling on mobile when every section has a parallax effect.
**Why it happens:** Too many `useScroll` hooks creating too many motion value subscriptions. Mobile browsers throttle scroll events.
**How to avoid:** Use `whileInView` (fires once) for most sections. Reserve `useScroll` + `useTransform` for 1-2 hero-adjacent effects only. Stick to `transform` and `opacity` properties.
**Warning signs:** Visible jank when scrolling on iPhone Safari.

### Pitfall 5: prefers-reduced-motion Not Respected
**What goes wrong:** Users with motion sensitivity see all animations, causing discomfort.
**Why it happens:** Framer Motion does not auto-respect `prefers-reduced-motion`. CSS animations do if you add the media query, but JS animations need manual handling.
**How to avoid:** Use Framer Motion's `useReducedMotion()` hook. When true: skip splash entirely, disable scroll-linked transforms, keep `whileInView` but reduce to simple opacity fade.
**Warning signs:** Animations play with system accessibility settings enabled.

```typescript
import { useReducedMotion } from "motion/react"

function SplashOverlay() {
  const shouldReduceMotion = useReducedMotion()

  useEffect(() => {
    if (shouldReduceMotion) {
      // Skip splash entirely for reduced motion users
      sessionStorage.setItem("glitch-splash-seen", "true")
      return
    }
    // ... normal splash logic
  }, [shouldReduceMotion])
}
```

## Code Examples

### useScroll + useTransform for Parallax Hero Background
```typescript
// Source: https://motion.dev/docs/react-use-scroll
import { useRef } from "react"
import { motion, useScroll, useTransform } from "motion/react"

function HeroSection() {
  const ref = useRef(null)
  const { scrollYProgress } = useScroll({
    target: ref,
    offset: ["start start", "end start"], // track from hero top to hero leaving viewport
  })

  // Background moves at 50% speed (parallax)
  const backgroundY = useTransform(scrollYProgress, [0, 1], ["0%", "30%"])
  // Content fades out as user scrolls
  const contentOpacity = useTransform(scrollYProgress, [0, 0.8], [1, 0])

  return (
    <section ref={ref} className="relative h-[90vh] overflow-hidden">
      <motion.div
        className="absolute inset-0 bg-[#0a0a0a]"
        style={{ y: backgroundY }}
      />
      <motion.div className="relative z-10" style={{ opacity: contentOpacity }}>
        {/* Hero content */}
      </motion.div>
    </section>
  )
}
```

### Stagger Children Animation

// Source: https://motion.dev/docs/react-motion-component
import { motion } from "motion/react"

const containerVariants = {
  hidden: { opacity: 0 },
  visible: {
    opacity: 1,
    transition: {
      staggerChildren: 0.1,

const itemVariants = {
  hidden: { opacity: 0, y: 20 },
  visible: { opacity: 1, y: 0 },

// Usage with whileInView:
<motion.div
  variants={containerVariants}
  initial="hidden"

## Open Questions

1. **Splash animation timing feel**
   - What we know: 2-3 seconds total is standard for splash screens. Longer than 3s feels slow.
   - What's unclear: Exact easing and timing that feels "dramatic" for this brand.
   - Recommendation: Start with 1.2s scale-down + 0.5s hold + 0.5s fade = 2.2s total. Adjust during visual verification with Playwright.

2. **Beat card gap value**
   - What we know: Current `gap-[2px]` is Metro standard but too tight for beat cards. Design language says `--gap-tile: 2px`.
   - What's unclear: Exact gap that gives "breathing room" while still feeling intentional.
   - Recommendation: `gap-4` (16px) for beat cards. This is the `--pad-tile` value from the design system, creating visual separation without breaking the grid language.
