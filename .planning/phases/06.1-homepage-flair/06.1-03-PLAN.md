# Plan 06.1-03: Scroll-Aware Sidebar Collapse

<objective>
Implement a scroll-aware collapsible sidebar on the homepage: sidebar starts collapsed (icon strip only, no logo) so the splash and hero logos are perfectly centered. When the user scrolls past the hero section, the sidebar automatically expands to full 280px width with labels and logo. Manual expand via click is available while in the hero. On other pages, sidebar defaults to expanded. This creates a seamless splash → hero → browse flow.
</objective>

<context>
## Prior Work
- 06.1-01: Splash overlay, hero logo, scroll indicator, scroll effects all implemented
- 06.1-02: Playwright verified, user approved splash + effects but identified logo centering jump

## User Decisions (D-10 through D-15)
- D-10: Sidebar starts COLLAPSED on page load (icon strip, no labels, no logo)
- D-11: Scrolling past hero auto-expands sidebar to 280px
- D-12: Manual expand available while in hero (click on collapsed strip)
- D-13: Scrolling back to hero collapses sidebar again
- D-14: Collapsed state shows icons only (no text, no logo)
- D-15: Homepage-specific behavior — other pages default to expanded

## Integration Points
- `TileNav` is the public sidebar (280px, sticky, tile grid layout)
- `PublicLayout` wraps TileNav + main content
- Homepage is at `src/app/(public)/page.tsx`
- Splash overlay wraps the entire public layout children
- Hero section has `h-[90vh]` height — scroll threshold at ~90vh
</context>

<interfaces>
## Data Contracts

SidebarContext:
- `collapsed: boolean` — current sidebar state
- `setCollapsed: (v: boolean) => void` — manual toggle
- `isHomepage: boolean` — whether scroll-aware behavior is active

TileNav changes:
- Reads `collapsed` from context
- When collapsed: `w-[64px]`, icons only, no LogoTile, no labels, no widgets
- When expanded: `w-[280px]`, full current layout
- Animated transition between states via `motion/react`

Homepage scroll tracking:
- Uses Framer Motion `useScroll` on the main content area
- When `scrollY > heroHeight` → expand sidebar
- When `scrollY <= heroHeight` → collapse sidebar
</interfaces>

<action>
**Step 1: Create a HomepageScrollWatcher client component**

Create a small client component (inline in page.tsx or separate file) that:
- Uses `useScroll()` from `motion/react` to track scroll position
- Uses `useTransform` or `useMotionValueEvent` to watch `scrollY`
- When `scrollY > window.innerHeight * 0.7` (70vh — slightly before hero ends): call `setCollapsed(false)`
- When `scrollY <= window.innerHeight * 0.3` (scrolled back near top): call `setCollapsed(true)`
- Uses `useSidebar()` to access `setCollapsed`

**Step 2: Update PublicLayout for homepage detection**

Modify `src/app/(public)/layout.tsx`:
- The layout is a server component, so it can't use `usePathname`
- Instead, pass `isHomepage` as a prop from page-level, OR use a simpler approach:
- Create a `HomepageSidebarController` client component that wraps the SidebarProvider and detects the homepage via `usePathname()`:
  - If pathname is `/`: set `defaultCollapsed={true}` and `isHomepage={true}`
  - Otherwise: set `defaultCollapsed={false}` and `isHomepage={false}`

**Step 3: Place HomepageScrollWatcher in page.tsx**

Add the scroll watcher component inside the homepage content so it only runs on the homepage. This component renders nothing visible — it just monitors scroll and toggles the sidebar.

**Step 4: Coordinate with splash overlay**

The splash overlay locks `document.body.style.overflow = "hidden"` during the animation. The sidebar should stay collapsed during splash. After splash completes (sessionStorage set), the scroll watcher takes over. No special coordination needed — the sidebar starts collapsed and the scroll watcher only expands it when the user actually scrolls.
</action>
