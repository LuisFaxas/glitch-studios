Inherited from Phase 29.2 ‚Äî no new colors introduced.

| Destructive | `#ef4444` | NOT USED ‚Äî Phase 30 has zero destructive actions |

Accent reserved-for list (complete ‚Äî never "all interactive elements"):
- Cyan gradient: `tone="cyan"` TechHero on `/tech/benchmarks` (landing) and every `/tech/benchmarks/[slug]` (detail). Cyan does NOT bleed into tiles, badges, table cells, sort indicators, or links.

Sort-active column treatment (no color shift):
- Active column header: `text-[#f5f5f0]` + `border-b-2 border-[#f5f5f0]` (currently sorted column).
- Inactive column header: `text-[#888]` + `border-b border-[#222]`.
- Sort direction glyph: ASCII `‚ñ≤` (asc) / `‚ñº` (desc) inline, `text-[#f5f5f0]`, font-mono 11px.

BPR ratio pill (per row, when both AC and Battery present):
- Background: `bg-transparent`, text only ‚Äî no pill chrome on the data cell itself.
- Format: `0.87` (always 2 decimals, monospace, tabular-nums). Values ‚â• 1.00 render as `1.02` (no `+` prefix); values < 1.00 render as plain `0.87`.
- Color: `text-[#f5f5f0]` always. No green/red ‚Äî monochrome rule.
- Tooltip (native `title` attr only ‚Äî no JS tooltip): `"Battery score / AC score. Higher is better (closer to 1.00 = less throttling on battery)."`

Baseline-relative pill (per row, when test has `reference_score`):
- Background: `bg-[#1a1a1a] border border-[#222]`
- Padding: `px-2 py-0.5`
- Text: `font-mono text-[10px] uppercase tracking-[0.05em] text-[#888]`
- Format for `higher_is_better`: `+47%` (above baseline) / `-12%` (below). Always include sign.
- Format for `lower_is_better` (e.g. ripgrep seconds): inverted. A run that is FASTER than baseline shows `+18%` (still semantic "better"); slower shows `-9%`. The inversion is computed once in code from `direction`; the pill never shows a misleading sign.
- If `reference_score` is null on a test: omit the pill (and the column on detail page) entirely. Do not render an empty cell, do not render `‚Äî`.

##### Sort behavior (D-04)

- Default sort on initial page load:
  - `mode: "both"` ‚Üí AC descending (or ascending if `lower_is_better`)
  - `mode: "ac"` ‚Üí AC descending/ascending per direction
  - `mode: "battery"` ‚Üí Battery descending/ascending per direction
- Click on AC/Battery/BPR column header toggles that column's direction. Active column gains the underline treatment.
- Sort happens client-side (corpus < 50 rows per benchmark). No URL state, no nuqs (29.3 baseline lesson ‚Äî keep persistent shell light).
- Direction logic: `RUBRIC_V1_1[key].direction === "higher_is_better"` ‚Üí first click descending; `lower_is_better` ‚Üí first click ascending. Subsequent clicks toggle.
- BPR column sort: descending first click always (higher BPR = better, regardless of test direction).

##### Honest-omit-row policy (D-05 discretion)

A product is included in this benchmark's leaderboard ONLY if it has at least
one measurement for this test (`mode: "both"` requires either AC or Battery; a
row with both missing is omitted). Phase 30 does NOT render "Not measured" rows.

##### Row count threshold

If `< 50` rows: render all rows in a single table (no pagination ‚Äî CONTEXT
deferred). If ‚â• 50 rows: still render all in v1, escalate to a polish phase

### Keyboard contracts

- Tab order on landing: hero CTA ‚Üí methodology blurb link ‚Üí jump-nav anchor 1 ‚Üí ‚Ä¶ ‚Üí jump-nav anchor 13 ‚Üí discipline section 1 tile 1 ‚Üí tile 2 ‚Üí ‚Ä¶ ‚Üí discipline section 2 tile 1 ‚Üí ‚Ä¶ (left-to-right, top-to-bottom within each section).
- Tab order on detail: hero CTA ‚Üí "what this measures" methodology link ‚Üí first sort header (AC) ‚Üí second sort header (Battery) ‚Üí third sort header (BPR) ‚Üí row 1 product link ‚Üí row 1 category link ‚Üí row 2 product link ‚Üí ‚Ä¶ ‚Üí methodology footer link.
- Focus ring: every focusable element receives `focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-[#f5f5f0]` (matches existing GlitchTech focus pattern). Sort header buttons inherit the same.
- Keyboard activation: `Enter` and `Space` activate sort headers (button semantics). `Enter` activates tile/row/jump-nav links (anchor semantics).
- Skip-to-content not new in this phase (already provided by `(tech)` layout).

### Sort interaction (detail page)

- Click sort header ‚Üí table re-renders sorted by that column. Re-render is a single React state update (sorted column key + direction). No DOM mutation outside the `<tbody>`.
- After sort: focus remains on the clicked header. Screen readers announce the new sort order via `aria-sort` attribute on the `<th>`:
  - Active ascending: `aria-sort="ascending"`
  - Active descending: `aria-sort="descending"`
  - Inactive: `aria-sort="none"`
- No animation on sort. Rows snap into new order instantly.

### Scrolling and layout

- Desktop sidebar must continue to fit one screen without scroll
  (`feedback_sidebar_no_scroll.md`). Phase 30 does NOT add new fixed-height
  widgets to the sidebar ‚Äî landing and detail render in the main content
  region only.
- No sticky behavior on hero, jump-nav, table headers, or any other Phase 30
  element. Per 29.3 baseline lesson: avoid persistent layered surfaces.
- Mobile leaderboard table uses horizontal scroll within its container ‚Äî the
  table is NOT collapsed to a card list (preserves cross-row scanability of
  AC/Battery/BPR numbers).
