**Gathered:** 2026-04-26

<domain>
## Phase Boundary

Polish the leaderboard SURFACE based on Phase 29 UAT. The data layer (`leaderboard.ts`), the seed (6 placeholder reviews), the table component shell (TanStack + nuqs + 7-filter set), the URL state schema, and the cache invalidation wiring all stay. This phase changes how users **discover** the rankings (top-level nav + hero), how they **filter** them (top bar instead of sidebar), how the table **fits** on the viewport (horizontal scroll bug fix), how the **mobile** experience offers choice (cards ↔ table view toggle), how the **GlitchMark number reads** (×10 display), and fixes a **production bug** in mobile nav active-state.

Sequential execution (no parallel waves) so each visual change can be verified inline with Playwright before the next plan starts.

**In scope:**
- New canonical route `/tech/rankings/[slug]/page.tsx` + new hub route `/tech/rankings/page.tsx`
- 301 redirect from old `/tech/categories/[slug]/rankings` → `/tech/rankings/[slug]` (in proxy.ts/middleware OR in the route file itself)
- Sidebar nav reorder: add `Rankings` (between Reviews and Categories) and `About` (last) in `tech-nav-config.ts` (desktop nav + mobile overlay menu — verify mobile tab bar does NOT need both)
- About page becomes the hub for methodology — section anchors `#bpr` and `#glitchmark` live on `/tech/about` so existing leaderboard column-header methodology links can keep working with a redirect from `/tech/methodology` (or update the links to point at `/tech/about#bpr` directly)
- Hero sections on three surfaces: `/tech/rankings`, `/tech/rankings/[slug]`, `/tech/categories/[slug]` — mirror beats-store full-bleed dark hero pattern
- Top filter bar replacing left sidebar (all 7 facets always visible; price as popover; Reset filters at right)
- Horizontal-scroll bug fix on the desktop table (set table `min-width` based on column count + remove the outer max-width cap on the table wrapper)
- Mobile view toggle (Cards ↔ Table) — nuqs URL state `?view=cards|table`; default = Cards
- GlitchMark display: store base 100 (unchanged schema), display `×10` rounded to integer (e.g. `1653` instead of `165.32`); update methodology page to document the convention
- Mobile nav active-state fix: normalize pathname before comparing in `BottomTabBar` and any other mobile nav surface that uses `usePathname()` for active highlighting
- Playwright spec covering each visual change (one spec per plan, runs against the running dev server)

**Out of scope (other phases / deferred):**
- Affiliate cloaking → Phase 41
- Compare integration (row-pinning, slug acceptance) → future phase
- Per-benchmark cross-category pages → Phase 30
- Editorial reframe of `/tech/categories/[slug]` → Phase 31 (this phase only adds the hero, not the editorial product cards)
- Replacing placeholder reviews with real ones → Phase 36 + later review phases
- Recompute GlitchMark internal scores at base 1000 → not needed; ×10 display is presentation-only
- New filter facets beyond the 7 already locked

  Existing `/tech/methodology` page either (a) 301-redirects to `/tech/about#methodology` OR (b) becomes a thin landing that links to the same anchors. **Researcher decides** based on incoming-link analysis of `/tech/methodology` — preserve any existing deep links into anchor sections (e.g. `#disciplines`).
- **D-03:** No standalone "Methodology" sidebar entry. The About button is the single discovery surface for methodology; column-header methodology icons on the leaderboard table point to `/tech/about#bpr`, `/tech/about#glitchmark`, `/tech/about#disciplines` (existing anchor names if reusable; otherwise new anchors).

### URL Structure

- **D-04:** New canonical leaderboard URL = `/tech/rankings/[slug]` (e.g. `/tech/rankings/laptops`).
- **D-05:** New hub page = `/tech/rankings` — landing index of categories. Today only Laptops has reviews, so the page renders one large category tile linking to `/tech/rankings/laptops` plus a "More categories coming soon" message. Future categories add tiles automatically (read from `tech_categories WHERE level=1`).
- **D-06:** Old URL `/tech/categories/[slug]/rankings` 301-redirects to `/tech/rankings/[slug]`. Implementation: per-route file using `permanentRedirect()` from `next/navigation`, OR via `proxy.ts` (middleware). **Researcher recommends** based on which is cleanest in Next 16; per-route is preferred for SEO traceability.
- **D-07:** "View Rankings" CTA on `/tech/categories/[slug]/page.tsx` updated to point at the new canonical `/tech/rankings/[slug]`.

### Hero Sections

- **D-08:** Hero pattern mirrors the beats-store full-bleed dark hero. Read `src/app/(public)/beats/page.tsx` (or wherever the beats hero lives) and replicate the visual rhythm — full-bleed dark background, large `<GlitchHeading>` h1, value-prop subhead, primary CTA. Tweak copy per page. Per-memory rule (`feedback_glitch_headers.md`): hover-only RGB-split glitch on every header; no auto-running animations.
- **D-09:** Hero copy per surface:
  - `/tech/rankings` (hub): h1 "Rankings", subhead "Every laptop, phone, and PC we've reviewed — ranked side-by-side by GlitchMark, BPR, and benchmark." CTA: "Read methodology" → `/tech/about#methodology`.
  - `/tech/rankings/[slug]` (leaderboard): h1 "{Category Name} Rankings", subhead "Compare every {category} we've reviewed — sortable by GlitchMark, BPR, and key benchmarks." CTA: "Read methodology" → `/tech/about#methodology`.
  - `/tech/categories/[slug]` (existing page): h1 "{Category Name}", subhead "Browse our {category} reviews." CTA: "View Rankings" → `/tech/rankings/[slug]`.
- **D-10:** Hero visual element: TBD by researcher / planner. Options: (a) generative geometric placeholder until a real hero image is uploaded, (b) reuse the placeholder_first build pattern with a single shared Cyber/laptop silhouette image, (c) text-only hero (no image) — match whatever beats hero does. Don't ingest new assets unless beats hero requires them.
- **D-11:** Hero CTA on `/tech/rankings/[slug]` is **single secondary** "Read methodology" link only. The table itself is the primary action — no flagship-review CTA in this phase (deferred until Phase 36 ships the real review). Hero CTA on `/tech/categories/[slug]` is "View Rankings" (the bridge to the new route).

### Filter UI Rework

- **D-12:** Replace the left sidebar (`<LeaderboardFilters>` in sidebar mode) with a **top filter bar** above the table. All 7 facets always visible:
  - Price → button "Price: $X–$Y" that opens a Slider popover
  - Year, CPU, RAM, Storage, Medal, Sub-cat → inline chip rows that wrap at narrow widths
  - "Reset filters" button at the right end
  - Total bar height target: 80–120px (researcher confirms readability)
- **D-13:** Existing `LeaderboardFilters` component is **rewritten** in place (same export name) so `LeaderboardTable` doesn't have to change much. The mobile filter `Sheet` keeps reusing it (now opens to a vertical layout of the same 7 facets).
- **D-14:** Active filter chips show their selected value count inline (e.g. `CPU (2)`). Hover/active state: cyberpunk subtle tint per `references/ui-brand.md`.

### Horizontal-Scroll Fix (Desktop Table)

- **D-15:** Set `min-width` on `<table>` based on column count (e.g. `style={{ minWidth: \`${columns.length * 140}px\` }}` or a fixed `1600px` for the 11-column current set + 4 benchmark cols). Remove the outer `max-w-[1400px]` cap on the table wrapper — the wrapper's `overflow-x-auto` then kicks in when viewport is narrower than table min-width. Sticky `#` and Product columns hold position.
- **D-16:** Verify Playwright that horizontal scrollbar APPEARS at viewports < table min-width and DOES NOT appear at viewports ≥ table min-width. Sticky-cell visual test: scroll the table, confirm `#` + Product cells stay anchored without bleed-through (Phase 29 RESEARCH §"Pitfall 2" already documented the explicit-bg pattern).

### Mobile View Toggle (Cards ↔ Table)

- **D-17:** Mobile view toggle component lives in the mobile filter bar. Pattern: `[ Cards | Table ]` segmented control. Default = **Cards**. URL state via nuqs: `?view=cards|table` with `clearOnDefault: true` (no garbage param when default).
- **D-18:** When `view=table` on mobile (`<768px`), render the desktop `<table>` element with horizontal scroll enabled (sticky cells still hold). The `<LeaderboardCard>` list is hidden. View toggle persists across navigation within the leaderboard surface.

### GlitchMark Display Scale

- **D-19:** GlitchMark display = stored value × 10, rounded to integer. Reference device displays as `1000`. MBP M5 Max placeholder (stored 165.32) displays as `1653`. Acer (stored NULL) still displays as `—`.
- **D-20:** Schema unchanged — `tech_reviews.glitchmark_score` stays `numeric(7,2)` with the base-100 reference. Display conversion happens at the rendering layer (single helper `formatGlitchmarkDisplay(score)` exported from `src/lib/tech/glitchmark.ts` — keeps the convention in one place).
- **D-21:** Methodology section on `/tech/about#glitchmark` documents the ×10 display convention explicitly. Mobile card and desktop GlitchMark cell both call the same formatter. Partial-flag rendering (`· partial`) preserved.
- **D-22:** Sort comparator on the GlitchMark column unchanged — it sorts on the underlying numeric value, not the display string. NULLS LAST holds in both directions.

### Mobile Nav Active-State Fix

- **D-23:** Root cause: on `glitchtech.io`, middleware rewrites `/foo` → `/tech/foo` but `usePathname()` returns the browser URL `/foo`. Mobile nav items are configured with `/tech/*` hrefs, so `pathname === item.href` never matches in production. Fix in `src/components/layout/bottom-tab-bar.tsx`: normalize both sides before comparing — strip leading `/tech` from `item.href`, then compare against `pathname`. Same logic for `pathname.startsWith(item.href + "/")`.
- **D-24:** Audit every mobile nav surface that uses `usePathname()` for active highlighting and apply the same normalization. Surfaces: `BottomTabBar` (tab items), `BottomTabBar` overlay menu items, any mobile sidebar / drawer items in the (tech) layout. Helper: extract `compareTechPath(itemHref, pathname)` to `src/lib/tech/nav.ts` (NEW) so all callers share one source of truth.
- **D-25:** Playwright regression test: emulated mobile viewport, navigate to `/reviews` (proxied as `/tech/reviews`), assert the Reviews tab has `aria-current="page"` AND the active background color (`bg-[#f5f5f0]`). Repeat for `/categories`, `/compare`, `/`. Test passes only when active state actually applies.
