## What was built

`formatGlitchmarkDisplay(score, options)` helper landed and is now the
single source of truth for the user-visible GlitchMark string. Schema is
unchanged (numeric(7,2), base-100 reference); the display layer
multiplies by 10 and rounds to integer so the reference device reads
`1000` and the ROG Strix placeholder reads `1425`.

## Notes / deviation from plan

- The plan asked for the helper in `src/lib/tech/glitchmark.ts`, but
  that file starts with `import "server-only"` (it owns
  recomputeGlitchmark which uses the Drizzle db client). Importing it
  from a client component fails. The helper therefore lives in
  `src/lib/tech/glitchmark-display.ts` (client-safe sibling) and the
  spec lives in `glitchmark-display.spec.ts`. Behavior, signature, and
  acceptance match the plan; only the file path differs.
- Sort is unchanged — `accessorFn: (row) => row.glitchmarkScore` still
  references the raw numeric column, `sortingFn: nullsLastNumeric` still
  applies. D-22 holds.
