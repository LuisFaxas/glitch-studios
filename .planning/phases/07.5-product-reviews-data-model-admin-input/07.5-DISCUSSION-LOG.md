## Admin Structure (Deep Debate)

**User surfaced this as their main concern:** "are we bundling them together? launch three discussion agents to argue about this"

Three advocacy agents were spawned in parallel, each arguing for one position:

### Positions argued

| Position | Advocate's core argument | Selected |
|----------|---------------------------|----------|
| A — Unified /admin with collapsible "Tech" section | "Tech is just more content. Add a sidebar group, ship it, move on. Media/roles/auth already shared — splitting duplicates for zero benefit." | |
| B — Separate /tech-admin dashboard | "07.4 made Tech a first-class entity on the front-end; admin should mirror that. Future-proof for subdomain migration or spin-off." | |
| C — Hybrid Studios ↔ Tech context switcher | "Shared infra where it matters (media, users, roles), switched context where it hurts (per-brand sidebar). Matches how you actually work — in sessions, not parallel." | ✓ |

**User's choice:** C — Hybrid context switcher
