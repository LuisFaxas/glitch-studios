## Color

**Strict monochrome, inherited from 07.4.** No Tech accent introduced in 7.5. The `[data-brand="tech"]` CSS hook from 07.4 remains empty in admin вҖ” admin chrome does NOT reskin when switcher is on Tech tab (see Context Switcher section for why).

| Role | Value | Usage in this phase |
|------|-------|---------------------|
| Dominant (60%) | `#000000` | Admin page background, main content area, sidebar background |
| Secondary (30%) | `#111111` | Form input fill, textarea fill, select fill, card/tile fill, modal fill, drawer fill |
| Tertiary surface | `#1a1a1a` | Hover-state lift on inputs/tiles; active row background in category tree |
| Borders (default) | `#222222` | Input borders, tile borders, modal borders, dividers between sections, sidebar section separator |
| Borders (focus) | `#f5f5f0` | Focused input border (existing pattern in blog form); focused context switcher pill border |
| Text (primary) | `#f5f5f0` | Input values, primary body text, active nav link text |
| Text (label / muted) | `#888888` | Field labels, muted metadata, unselected nav tab label |
| Text (tertiary) | `#555555` | Placeholder text, help text, section-heading muted variant, inactive items |
| Text (inverted) | `#000000` | Text on white active-state surfaces (active context switcher pill, primary button) |
| Active surface (inverted) | `#f5f5f0` | Active sidebar link background, primary button background, active context switcher pill background, selected tree node |
| Destructive | `#dc2626` | Delete button text, destructive confirmation label, validation error text |
| Destructive hover | `#ef4444` | Hover state on destructive action text (existing pattern `blog-post-form.tsx` line 275) |

**Accent reserved for (10% rule):** The inverted `#f5f5f0` surface is treated as this phase's "accent." It is reserved strictly for:

1. The **active sidebar nav link** (`active ? "bg-[#f5f5f0] text-[#000000]"` вҖ” existing pattern)
2. The **active context switcher pill** (Studios or Tech, whichever matches the current route)
3. The **primary form action button** ("Save Draft" secondary вҶ’ dark; "Publish" / "Save Changes" primary вҶ’ inverted per `blog-post-form.tsx` lines 347вҖ“398)
4. The **focused input border** (thin 1px ring)
5. The **selected category tree node** background

No other element uses the inverted surface. Hover-states use `#1a1a1a` lift, not inversion.

**Why no Tech accent in admin:** The 07.4 UI-SPEC deferred accent color introduction to 7.5/7.6. In 7.5 we keep the admin **deliberately monochrome** because (a) the context switcher pills already carry mode signal via bg inversion, and (b) recoloring admin chrome per mode would visually fragment a single backstage tool that the solo operator uses in rapid context-switches. A Tech accent may still launch on the **public** review display in 7.6; that decision is deferred.

### Switcher Visual Spec

**Decision (resolving "chip-group vs tabs vs segmented control"):** **Segmented control of two pills**, inline side-by-side, full-width within the sidebar padding. This is the winner because:

1. The user is always in exactly one of two modes вҖ” segmented control is the conventional UX (iOS-native, shadcn `Tabs` pattern). Chip-group suggests multi-select.
2. It gives a **clear binary mental model**: "you are HERE" not "these are filters."
3. It reuses the existing **inverted-surface active-state** pattern (`bg-[#f5f5f0] text-[#000000]`) already established for sidebar links вҖ” no new visual vocabulary needed.

| Property | Value |
|---|---|
| Container | `div` with `display: flex; gap: 4px; padding: 8px 16px; border-bottom: 1px solid #222` |
| Pill element | `button` вҖ” each ~50% width, `min-height: 36px`, no border-radius (brutalist, sharp edges), `border: 1px solid #222` |
| Pill typography | `font-mono text-[13px] font-bold uppercase tracking-[0.05em]` |
| Pill icon (optional, left of label) | Lucide `Music` for Studios, Lucide `Cpu` for Tech вҖ” 14px, `shrink-0` |
| Inactive pill | `bg-[#111111] text-[#888888] border-[#222]`; hover вҶ’ `bg-[#1a1a1a] text-[#f5f5f0] border-[#444]` |
| Active pill | `bg-[#f5f5f0] text-[#000000] border-[#f5f5f0]` вҖ” fully inverted |
| Transition | `transition-colors duration-150` (no glitch effect вҖ” admin is quiet) |
| Focus ring | `focus-visible:outline focus-visible:outline-2 focus-visible:outline-[#f5f5f0] focus-visible:outline-offset-2` |

### Route-Aware Active State (D-16)

Active mode is derived from `usePathname()`:

```
pathname.startsWith("/admin/tech") вҶ’ Tech active
otherwise                          вҶ’ Studios active
```

The switcher does **not** maintain its own state. Clicking a pill is a `<Link>` navigation:

| Current path | Click "Studios" вҶ’ | Click "Tech" вҶ’ |
|---|---|---|
| `/admin` | (already active, no-op nav) | `/admin/tech` |
| `/admin/beats` | (already active, no-op nav) | `/admin/tech` (land on tech dashboard) |
| `/admin/tech` | `/admin` | (already active, no-op nav) |
| `/admin/tech/reviews/[id]/edit` | `/admin` | (already active) |

**The "click Studios from a deep Tech route" case:** always navigates to `/admin` (Studios dashboard root), not "back one level." This prevents orphan-route confusion.

### Switching Feel

- **Instant**: clicking a pill navigates Next.js route; the sidebar re-renders with the new mode's nav list
- **No animation on the switch itself** вҖ” the active pill recomputes from the new pathname. The user's attention jumps to the main content anyway. (Contrast: on **public** site, cross-entity transitions use glitch flicker. Admin is quiet.)
- **Sidebar nav sections below animate in with a subtle 150ms fade-in** using Framer Motion `AnimatePresence` on the section list container вҖ” so the user sees "these items refreshed," not layout jank.
- **Tooltip on hover** (via shadcn `Tooltip`): "Switch to Glitch Tech admin" / "Switch to Glitch Studios admin" вҖ” confirms destination for first-time users.

### Tree Node Spec

**Decision (resolving "native HTML tree vs library"):** **Custom recursive component** built on a lightweight drag library. Recommendation: **`dnd-kit`** (`@dnd-kit/core` + `@dnd-kit/sortable`) вҖ” already the industry standard for React 19, composable, keyboard-accessible out of the box. If `dnd-kit` is not already a dep (check `package.json`), planner adds it. Do NOT use HTML5 native DnD (too fiddly for nested trees) and do NOT use `react-beautiful-dnd` (unmaintained, React 19 concerns).

| Action | Interaction |
|---|---|

### Keyboard Accessibility

- `Tab` between nodes (tree follows list-of-rows focus order, not tree-widget ARIA вҖ” simpler, still accessible)

- `F2` on a focused row = enter rename mode (alternative to double-click)
- `Cmd/Ctrl+вҶ‘` / `Cmd/Ctrl+вҶ“` on a focused row = reorder up/down among siblings (keyboard fallback for drag)
- `Delete` on a focused row = open delete confirmation

- **Empty (no categories yet):** Centered content in the tree pane: headline "No categories yet" (mono, 15px, text-[#888]); body "Add your first parent category to start building your product catalog." (sans, 13px, text-[#555]); CTA button `[+ Add parent category]` centered below

### Autosave Indicator Spec

`<AutosaveIndicator>` renders in the header bar, right of the page title.

| State | Display | Style |
|---|---|---|

| Unsaved changes (debounce window) | `Unsaved changes` | `font-mono text-[11px] text-[#dc2626]` |
| Save failed | `Save failed вҖ” retry` (clickable retry link) | `font-mono text-[11px] text-[#dc2626] underline` |

**Autosave behavior (resolving CONTEXT ambiguity):**
- **Debounce: 2 seconds** after the last keystroke / field change
- **Periodic: every 30 seconds** while the tab is focused, even if the debounce hasn't fired (catches idle writers)
- **On blur of title/verdict field**: immediate save
- **Implementation note for planner:** debounced server action call with the full form state. Conflict resolution: last-write-wins (single admin user assumption). Status transitions (draft вҶ’ published) are NOT autosave вҖ” those require explicit click.

6. **Review editor last** вҖ” most complex component; builds on `TiptapEditor`, `MediaPickerDialog`, shadcn `Slider`, new `AutosaveIndicator`, new `ProsConsInput`.

### Known Constraints
