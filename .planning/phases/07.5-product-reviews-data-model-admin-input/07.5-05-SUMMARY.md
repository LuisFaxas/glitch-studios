## Plan 07.5-05 — Product Form (Create + Edit)

### Outcome

`/admin/tech/products/new` and `/admin/tech/products/[id]/edit` are fully wired with explicit Save (no autosave per D-22). Categories, hero image, marketing fields, and dynamic specs per template all persist.

### Components

- `category-picker.tsx` — shadcn Command-in-Popover, leaf-only selection, fuzzy search, `Parent > Category > Subcategory` display
- `dynamic-spec-fields.tsx` — typed per-field rendering (text / number+unit / enum-Select / boolean-Switch)
- `product-form.tsx` — flat form with base / media / marketing / specifications sections + category-change confirmation dialog + MediaPickerDialog integration

### MediaPickerDialog additive change

Extended `onSelect` signature from `{ url, alt }` → `{ id, url, alt }` (additive; existing blog-post-form.tsx destructures only `url` + `alt`, still works). Lets the product form capture the media asset FK id for `hero_image_id`.

### Deviations

- PopoverTrigger is base-ui (not radix) — no `asChild`. Styled PopoverTrigger directly instead of wrapping a native button.
- Spec grid is 2-col on `md+` (UI-SPEC) but stacks on mobile (single col) for readability — documented here as the mobile behavior for the audit trail.
