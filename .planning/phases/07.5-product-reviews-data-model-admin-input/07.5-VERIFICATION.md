### Human Verification Required

#### 1. Category tree interaction

**Test:** Navigate to /admin/tech/categories. Click the add-parent button, enter a name in the window.prompt dialog. Then select the new parent and click add-subcategory. Then double-click a node name (or press F2) to rename it.
**Expected:** Nodes appear in the tree without full page reload (or with reload); rename commits on Enter, cancels on Escape.
**Why human:** `window.prompt` cannot be driven by grep; rename state requires browser interaction.

#### 2. Spec template drawer

**Test:** Select a category, click "Edit spec template". Add one field of each type (text, number+unit, enum with options, boolean). Save.
**Expected:** Fields appear in the drawer list; when creating a product in that category, the spec fields render under Specifications.
**Why human:** Requires live DB roundtrip + browser form interaction.

#### 3. Review editor autosave flow

**Test:** Go to /admin/tech/reviews/new. Fill in Title, pick a Product, pick a Hero Image. Wait 2+ seconds without clicking.
**Expected:** AutosaveIndicator shows "Saving…" then "Saved Xs ago"; browser URL changes to /admin/tech/reviews/{id}/edit.
**Why human:** Timer-based side effects and `history.replaceState` require live browser session.

#### 4. Review publish gate

**Test:** On a new review page (reviewId=null), verify the Publish button is disabled/aria-disabled. Complete required fields and wait for autosave to succeed. Then check Publish button state.
**Expected:** Publish becomes active only after first successful save.
**Why human:** Depends on runtime reviewId state after autosave callback.

#### 5. Playwright suite with credentials

**Test:** Set ADMIN_EMAIL and ADMIN_PASSWORD env vars, run `pnpm exec playwright test tests/07.5-*`.
**Expected:** 17 tests pass (tests run, none skipped due to missing creds); exit 0.
**Why human:** Requires live dev server + admin credentials not available in CI.
