## Plan 07.5-06 — Review Editor with Autosave

### Outcome

`/admin/tech/reviews/new` and `/admin/tech/reviews/[id]/edit` render the full review editor — Tiptap body with live preview on desktop, Write|Preview tabs below 1280px, right-rail Details sheet with rating/pros-cons/media/audience/publishing, autosave with debounce + periodic + blur-trigger. Publish/Unpublish is separate from autosave.

### Components

- `use-autosave.ts` — hook with 2s debounce + 30s periodic + `flushNow()` + state machine (idle/saving/saved/error)
- `autosave-indicator.tsx` — `aria-live="polite"` chip with "Saved Xs ago" / "Saving…" / "Unsaved changes" / "Save failed — retry"
- `rating-slider.tsx` — labelled Slider wrapper, 1-10 integer step, with keyboard accessible value
- `markdown-preview-pane.tsx` — read-only Tiptap editor rendering the same HTML live
- `pros-cons-input.tsx` — two-list dynamic input, Enter-to-add-row on last field, X-to-remove
- `review-editor.tsx` — 550+ lines. All D-22 fields. Desktop 2-col (body / preview) at xl:, tab mode below. Right-rail Sheet with Details. 3 MediaPickerDialog instances (hero, gallery, inline body).

### Server action addition

- `listReviewers()` — filters `user` table to non-"user" roles, returns `{ id, name }[]` for the reviewer selector dropdown

### Autosave behavior

- **Debounce:** 2000ms after last data mutation
- **Periodic:** 30000ms interval while idle
- **Flush triggers:** `onBlur` of Title and Verdict textarea
- **Payload gate:** `null` until title + product + hero image are all present; first save requires manual completion of required fields
- **First save (create mode):** upsertReview(null, data) returns new id; URL replaced via `history.replaceState` to `/admin/tech/reviews/{id}/edit` so subsequent refreshes land on the edit route without unmounting the editor

### Deviations from UI-SPEC

- Drag-reorder of pros/cons rows deferred (inputs ordered by insertion; explicit X-remove available)
- `canPublish` gated on payload validity AND reviewId !== null (the editor forces a first save before allowing publish, which means the user clicks Save Draft first if they somehow bypassed autosave — normal flow is autosave → reviewId filled → Publish enabled)
