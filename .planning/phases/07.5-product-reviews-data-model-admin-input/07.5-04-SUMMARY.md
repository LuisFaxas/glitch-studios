## Plan 07.5-04 — Categories + Template Drawers

### Outcome

`/admin/tech/categories` now renders a fully interactive 3-level category tree with right-side detail panel and two drawers (spec template, benchmark template).

### Components

- `category-tree-node.tsx` — recursive node with dnd-kit `useSortable`, inline rename (F2 or double-click), keyboard nav, `+ Add subcategory` button at depth <= 2
- `category-tree-view.tsx` — DndContext + KeyboardSensor + PointerSensor; same-parent reorder; `window.prompt` for add parent/child
- `category-detail-panel.tsx` — selected node metadata + two "Edit template" buttons + Danger zone delete with Dialog confirmation
- `spec-template-editor.tsx` — right drawer (480px) with typed field CRUD (text/number+unit/enum options/boolean); full-replace save
- `benchmark-template-editor.tsx` — right drawer with test CRUD (name/unit/direction); full-replace save

### UI-SPEC Deviations

- Add-parent and add-subcategory use native `window.prompt()` instead of an inline-appending input pattern. Ships faster; revisit if admin UX feedback calls for inline polish.
- Cross-parent reparenting via drag is blocked (toast error explains to use `+ Add subcategory`). Same-parent reorder ships; cross-parent move is deferred to a future "move" secondary action.
- Tree refresh after server mutation uses `window.location.reload()` instead of optimistic merging. Simpler for low-frequency admin screen.
