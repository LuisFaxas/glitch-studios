<interfaces>
Domains in scope:
- `glitchstudios.io` — primary brand, all v4.0 emails send from this domain
- `glitchtech.io` — secondary brand, Phase 38 owns brand-split emails

DNS records needed per Resend docs:
- **SPF** (root TXT): `v=spf1 include:amazonses.com ~all` OR `include:_spf.resend.com` — Resend dashboard shows exact value
- **DKIM** (typically 3 CNAME records): `resend._domainkey`, `resend2._domainkey`, `resend3._domainkey` — exact target shown in Resend dashboard after domain add
- **DMARC** (TXT `_dmarc`): `v=DMARC1; p=none; rua=mailto:admin@glitchstudios.io` — p=none monitoring for launch; tighten later

Tools available:
- Cloudflare MCP — lists/reads DNS zones if API token scope allows.
- Resend dashboard — user-owned; agent cannot read it directly.
- `dig` / `host` CLI — can verify current DNS state from the public resolver.
</interfaces>
