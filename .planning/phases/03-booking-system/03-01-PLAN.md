    Create `src/lib/booking/` directory with 6 modules:

    **1. `src/lib/booking/availability.ts`** — Core availability calculation:
    - `getRoomAvailability(roomId: string, date: string)`: Query weeklyAvailability for the room+dayOfWeek. Check availabilityOverrides for date-specific closures/altered hours. Return `{ isOpen: boolean, startTime: string | null, endTime: string | null }`.
    - `getAvailableDates(serviceId: string, month: Date)`: For the given month, check all rooms for each day. A day is "available" if at least one room has open hours AND at least one slot can fit the service duration. Use date-fns `startOfMonth`, `endOfMonth`, `eachDayOfInterval`, `format`, `getDay`. Return `Map<string, boolean>` keyed by "YYYY-MM-DD".
    - Import from `drizzle-orm` the `eq`, `and` operators. Import `db` from `@/lib/db`. Import tables from `@/db/schema`.

    **2. `src/lib/booking/slots.ts`** — Time slot generation:
    - `generateTimeSlots(startTime: string, endTime: string, durationMinutes: number, bufferMinutes: number)`: Generate slot candidates from room open-to-close. E.g., 09:00-21:00 with 120min duration and 15min buffer → slots at 09:00, 11:15, 13:30, 15:45, 18:00, 20:15 (only if endTime fits). Use 24h "HH:mm" strings, parse hours/minutes with split(":").map(Number).
    - `getAvailableSlots(serviceId: string, date: string)`: For each active room, get availability for that date, generate slot candidates, then query existing bookings (status IN ("pending", "confirmed")) for that room+date. Subtract occupied slots (including buffer on both sides). Return `TimeSlot[]` sorted by startTime then roomName. Each TimeSlot has roomId, roomName, startTime, endTime.
    - Import `getRoomAvailability` from `./availability`.

    **3. `src/lib/booking/deposit.ts`** — Deposit calculation:
    - `calculateDeposit(serviceId: string)`: Query serviceBookingConfig for the service. If depositType is "percentage", compute `totalPrice * (depositValue / 100)`. If "flat", return depositValue directly. Return `{ depositAmount: number, totalPrice: number, depositType: DepositType }`.
    - Note: totalPrice comes from the service's priceLabel. Since priceLabel is a string like "$150/hr", the admin-set depositValue for "flat" is the dollar amount. For percentage, the base price should be stored in serviceBookingConfig or derived. For MVP: use a `basePrice` pattern where totalPrice = hourlyRate * (durationMinutes / 60). If room has hourlyRateOverride, use that. Otherwise use a service-level base computed from serviceBookingConfig.
    - Simpler approach: `calculateDeposit(depositType: DepositType, depositValue: number, totalPrice: number)`: Pure function. Takes the already-known values. Returns `{ depositAmountCents: number }`. Convert to cents with `Math.round(depositAmount * 100)`.

    **4. `src/lib/booking/policy.ts`** — Cancellation/reschedule policy:
    - `canCancel(booking: { date: string, startTime: string, status: BookingStatus }, config: { cancellationWindowHours: number })`: Compare current time to booking start time. If difference >= cancellationWindowHours, return `{ allowed: true, withinPolicy: true }`. If difference < cancellationWindowHours, return `{ allowed: true, withinPolicy: false }` (deposit forfeited). If status is "cancelled" or "completed", return `{ allowed: false, reason: "Booking already {status}" }`.
    - `canReschedule(booking, config)`: Same logic as canCancel — rescheduling is allowed within the same policy window.
    - `getRefundAmount(depositAmountCents: number, refundPolicy: RefundPolicy, withinPolicy: boolean)`: If not withinPolicy, return 0. If "full", return depositAmountCents. If "partial", return Math.floor(depositAmountCents * 0.5). If "none", return 0.
    - Use date-fns `parse`, `differenceInHours`, `isBefore` for time comparison. Parse date+startTime as "yyyy-MM-dd HH:mm".

    **5. `src/lib/booking/recurring.ts`** — Recurring series:
    - `generateRecurringSeries(startDate: string, dayOfWeek: number, startTime: string, totalSessions: number)`: From the start date, generate N weekly dates. Return `string[]` of "YYYY-MM-DD" dates. Use date-fns `addWeeks`, `format`.
    - `validateSeriesAvailability(serviceId: string, dates: string[], startTime: string)`: For each date, call `getAvailableSlots(serviceId, date)` and check if any slot matches the startTime. Return `{ available: string[], conflicts: string[] }`.
    - `calculatePackagePrice(basePrice: number, sessionCount: number, discountPercent: number)`: Return `{ perSession: number, total: number, savings: number }`.

    **6. `src/lib/booking/ics.ts`** — ICS file generation:
    - `generateBookingIcs(booking: { serviceName: string, date: string, startTime: string, endTime: string, roomName: string, address: string })`: Use `ics` package's `createEvent`. Parse date "YYYY-MM-DD" and times "HH:mm" into the `[year, month, day, hour, minute]` tuple format that ics expects. Set title to `{serviceName} - Glitch Studios`, location to address, description to `Studio session in {roomName}`. Return the ICS string.

    - Singleton pattern matching `src/lib/stripe.ts`.
