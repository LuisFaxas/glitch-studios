  <action>
    1. Create `src/app/api/bookings/confirm/route.ts` — POST endpoint:
       - Body: `{ bookingId: string }`
       - Auth check: must be admin (use Better Auth session from headers, check role)
       - Query booking, verify status is "pending"
       - Update status to "confirmed", set updatedAt
       - If booking is part of a series and this is the first confirmation, don't auto-confirm others (each confirmed individually or series-wide)
       - Return `{ success: true, booking }`

    2. Create `src/app/api/bookings/cancel/route.ts` — POST endpoint:
       - Body: `{ bookingId: string, reason?: string, isAdmin: boolean }`
       - Auth check: if isAdmin, verify admin role
       - If isAdmin (per D-11): skip policy check, always allow cancellation
         - If booking has stripePaymentIntentId, issue full refund via `stripe.refunds.create({ payment_intent: booking.stripePaymentIntentId })`
         - Update booking: status="cancelled", cancelledAt=now, cancelledBy="admin", cancellationReason=reason
       - If client cancellation:
         - Load serviceBookingConfig for policy
         - Call `canCancel(booking, config)` — if not allowed, return 403 with reason
         - If withinPolicy: calculate refund via `getRefundAmount(depositAmountCents, config.refundPolicy, true)`. Issue Stripe refund if amount > 0.
         - If outside policy window: no refund (deposit forfeited)
         - Update booking: status="cancelled", cancelledAt=now, cancelledBy="client", cancellationReason=reason
       - Return `{ success: true, refundAmount }`

    3. Create `src/app/api/bookings/reschedule/route.ts` — POST endpoint:
       - Body: `{ bookingId: string, newDate: string, newStartTime: string, newEndTime: string, newRoomId: string, isAdmin: boolean }`
       - Auth check: if isAdmin, verify admin role
       - If client reschedule and not admin:
         - Load serviceBookingConfig, call `canReschedule(booking, config)`
         - If not allowed, return 403
       - Verify new slot is available via `getAvailableSlots(booking.serviceId, newDate)` — check the specific slot exists
       - Update booking: set date, startTime, endTime, roomId to new values, updatedAt=now
       - Return `{ success: true, booking }`

    All three endpoints:
    - Use `auth.api.getSession({ headers: await headers() })` for auth
    - Return 401 if not authenticated
    - Return 403 if not authorized
    - Return 404 if booking not found
    - Use sonner-compatible error messages
