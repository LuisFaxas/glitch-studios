## Interaction Patterns

### Booking Flow (5 Steps)

**Step 1: Select Service**
- Tile grid of bookable services (reuses existing service data)
- Pre-selected via `?service=mixing` query param from services page CTA
- Selected service tile inverts (white bg, black text)
- Each tile shows: service icon, name, duration, starting price

**Step 2: Select Date**
- Month calendar grid rendered as flat tiles
- Available days: `#111111` bg with glitch hover
- Unavailable days: `#0a0a0a` bg, `#555555` text, no hover effect, no pointer cursor
- Selected day: inverted (`#f5f5f0` bg, `#000000` text)
- Today: 2px `#f5f5f0` bottom border
- Month navigation: left/right arrow tiles flanking month/year heading
- Tile grid gap: 4px between day tiles

**Step 3: Select Time Slot**
- Vertical list of available slots for selected date
- Each slot tile: 44px tall, full width, shows time range (e.g., "2:00 PM - 4:00 PM")
- Room name shown as sublabel if multiple rooms
- Glitch hover on available slots
- Selected slot: inverted
- If no slots available: empty state message

**Step 4: Your Details**
- Flat form: name (required), email (required), phone (required), notes (optional)
- If logged in: pre-fill from session, show "Booking as [name]" instead of name/email
- Guest flow: fields visible, optional "Create account" checkbox below
- Form inputs: flat, 1px `#333333` border, white border + subtle glow on focus
- Booking summary tile visible alongside form (desktop: right column, mobile: collapsed accordion above)

**Step 5: Payment & Confirm**
- Booking summary tile with all details
- Deposit amount prominently displayed (e.g., "Deposit: $50.00" or "Deposit: 50%")
- Stripe Embedded Checkout for deposit payment (same pattern as Phase 2)
- After payment: confirmation screen with .ics download button

### Step Indicator

- Horizontal bar at top of booking flow
- 5 segments: SERVICE > DATE > TIME > DETAILS > PAYMENT
- Completed steps: `#f5f5f0` text
- Current step: `#f5f5f0` text, 2px bottom border
- Future steps: `#555555` text
- All caps, JetBrains Mono 11px, letter-spacing 0.05em

### Calendar Navigation

- Previous month: left arrow tile (glitch hover)
- Next month: right arrow tile (glitch hover)
- Month/year: JetBrains Mono 22px bold, centered
- Past months: navigable but all days greyed out
- Max future: 3 months ahead (configurable)

### Cancellation Flow

1. Client clicks "Cancel Booking" on their booking card
2. Dialog appears with policy info: "Cancellations more than {N} hours before your session receive a full deposit refund."
3. If within policy window: "Cancel Booking" destructive button
4. If outside policy window: "Your deposit of ${amount} will be forfeited. Continue?" with destructive confirmation
5. Admin override: no policy restriction, always can cancel with reason field

### Reschedule Flow

1. Client clicks "Reschedule" on their booking card
2. Opens calendar view (same as booking step 2-3) pre-filtered to same service
3. Select new date and time slot
4. Confirm reschedule (no additional payment if within policy)

### Recurring Booking Flow

1. After selecting service, date, and time: "Make this recurring?" toggle
2. If enabled: show frequency (weekly), number of sessions (dropdown: 4, 8, 12, custom)
3. Package pricing displayed if applicable (e.g., "8 sessions: $X/session, save 25%")
4. Calendar highlights all recurring dates
5. Summary shows full series with total and per-session pricing

## Animation Contract

All animations inherit from the existing glitch system in `globals.css`:

| Interaction | Animation | Duration | Reduced Motion Fallback |
|-------------|-----------|----------|------------------------|
| Calendar day tile hover | `glitch-hover` keyframes + scan-line | 200ms + 400ms | None (static hover bg change) |
| Time slot tile hover | `glitch-hover` keyframes | 200ms | None (static hover bg change) |
| Step transition | Framer Motion `opacity` + `x` slide | 300ms | `opacity` only, 200ms |
| Calendar month change | Framer Motion `opacity` + `x` direction | 250ms | `opacity` only, 200ms |
| Booking confirmation | Framer Motion `scale` from 0.95 + `opacity` | 400ms | `opacity` only, 200ms |
| Tile selection (invert) | CSS transition `background-color`, `color` | 150ms | Instant |
| Service tile press | `transform: scale(0.97)` | 100ms | None |

Source: `DESIGN-LANGUAGE.md` section 6, `globals.css` keyframes
