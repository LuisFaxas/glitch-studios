**Glitch hover fix (C7):** The current glitch overlay is an empty `<span>` with `animate-glitch-hover` — clip-path on nothing is invisible. The overlay gets `absolute inset-0 overflow-hidden` and the duplicated content gets the same layout. This means the clip-path animation actually distorts visible content.

Concrete implementation for the glitch overlay:
```
{isHovered && !isActive && (
  <div className="pointer-events-none absolute inset-0 overflow-hidden animate-glitch-hover" aria-hidden="true">
    <div className={contentLayoutClasses}>
      {icon && <span className="flex-shrink-0 [&>svg]:h-5 [&>svg]:w-5">{icon}</span>}
      {label && !compact && <span className="font-mono text-lg font-bold uppercase tracking-[0.05em]">{label}</span>}
    </div>
  </div>
)}
```

The scan line overlay stays as-is (it works fine as a separate sweeping line).

**Active state enhancement (P3):** The active tile already inverts correctly. Add a slightly stronger glow: change `shadow-[0_0_20px_rgba(255,255,255,0.08)]` to `shadow-[0_0_20px_rgba(255,255,255,0.12)]`.

**Also in globals.css:** No CSS changes needed — the existing `@keyframes glitch-hover` and `.animate-glitch-hover` utility work correctly. The fix is structural (applying animation to visible content, not an empty span).
