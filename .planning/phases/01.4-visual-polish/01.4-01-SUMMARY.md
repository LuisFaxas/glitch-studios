## Decisions Made
- Tile layout prop defaults to vertical -- horizontal is opt-in to prevent regressions in widgets/auth tiles
- Glitch hover duplicates content (icon+label) into overlay div rather than animating empty span -- clip-path on empty element was invisible
- LogoTile uses CSS filter:invert(1) on background-image rather than swapping to a white logo asset
- Focus trap implemented inline via querySelectorAll + Tab keydown handler -- no external dependency needed for this simple case
- Large tiles always use vertical layout regardless of layout prop to preserve their 2x2 visual weight
