## Implementation Notes

- **Discriminated-union CTA dispatch.** `Slide.action` is typed as
  `{ kind: "scroll"; target: string } | { kind: "link"; href: string }`, so the
  outer ternary narrows the branch without inline re-checks. The scroll branch
  captures `target` in an IIFE-bound const so the `onClick` closure cannot
  observe an aliased `slide.action.target` after a React re-render.
- **Native-event safety (CLAUDE.md ranking filter rule).** No `onTouchStart`,
  `onFocus`, `onBlur`, or `visibilitychange` handler in this component writes
  to React state synchronously. Embla's internal `select` event is the only
  state-write trigger, and that is library-managed (the same pattern in
  production at `beats-hero-carousel.tsx`).
- **`type="button"` on every `<button>`** — both the scroll-CTA and dot
  indicators — so a future form ancestor in Plan 04 cannot trigger a submit.
- **No bundle changes.** `embla-carousel-react` and `embla-carousel-autoplay`
  are already in `package.json` (`^8.6.0`) and used by `beats-hero-carousel.tsx`;
  no new dependency.
