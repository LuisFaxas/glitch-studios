## Decisions Made

- **Mount guard uses setTimeout(0)-deferred setState** — Plan literal `useEffect(() => setMounted(true), [])` triggers `react-hooks/set-state-in-effect`. Project convention from Plan 48-16 ("Deferred React state writes from effect bodies with zero-delay timers and cleanup instead of disabling compiler rules") applies. Same convention as the ranking-filter-safety rule. Cleanup clears the timeout.
- **Base UI Dialog uses `render` prop, not `asChild`** — `src/components/ui/dialog.tsx` is `@base-ui/react/dialog` (not Radix). Base UI's component composition API uses `render={<h2 .../>}`. The plan code snippet used `asChild` which would have been a Radix idiom; converted to Base UI's `render` API. Vaul Drawer still uses `asChild` correctly (Vaul follows Radix conventions).
- **Custom Request tile NOT extracted to shared primitive** — Plan permitted "render as SIBLING `<Link>`". Duplication is acceptable because (a) click target differs (Link vs button), (b) chip set is locked literals not formatted from DB, (c) extracting a TileShell for one extra tile is over-engineering this phase. Inline comment documents the rationale.
- **JSX section labels collapsed onto single h3 lines** — Prettier's natural wrap put `>\n PRICING\n </h3>` across 3 lines, which broke the plan's literal `>PRICING<` acceptance grep. Collapsed each label-bearing h3 to one line. The verbatim-grep contract is the source of truth; ergonomic Prettier wrapping is not.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Plan literal `setMounted(true)` inside `useEffect` failed react-hooks/set-state-in-effect**

- **Found during:** Task 2 (ServiceDetailOverlay)
- **Issue:** The literal mount-guard snippet in `<action>` (`useEffect(() => setMounted(true), [])`) is flagged by React 19's `react-hooks/set-state-in-effect` rule. ESLint blocks commit.
- **Fix:** Wrapped `setMounted(true)` in `window.setTimeout(...0)` with cleanup — matches Plan 48-16's documented decision ("Deferred React state writes from effect bodies with zero-delay timers and cleanup instead of disabling compiler rules") and the CLAUDE.md ranking-filter-safety rule.
- **Files modified:** src/components/services/service-detail-overlay.tsx
- **Verification:** ESLint exit 0, lint output empty
- **Committed in:** 2d1a8bc (Task 2 commit)

**2. [Rule 1 - Bug] Plan's `DialogTitle asChild` snippet would not compile against @base-ui/react/dialog**

- **Found during:** Task 2 (ServiceDetailOverlay)
- **Issue:** The plan's code snippet used Radix's `<DialogTitle asChild>` idiom, but `src/components/ui/dialog.tsx` wraps `@base-ui/react/dialog`. Base UI's primitive composition API uses `render={<...>}`, not `asChild`. Compiling against Base UI's `Title.Props` with `asChild` would be a TypeScript error.
- **Fix:** Rewrote Dialog's title node as `<DialogTitle render={<h2 className="..." />}>{service.name}</DialogTitle>`. Drawer's title kept `asChild` (Vaul follows Radix-style).
- **Files modified:** src/components/services/service-detail-overlay.tsx
- **Verification:** `tsc --noEmit --pretty false` returns 0 errors for service-detail-overlay
- **Committed in:** 2d1a8bc (Task 2 commit)

**3. [Rule 3 - Blocking] JSX section labels split onto multiple lines broke literal `>PRICING<` grep**

- **Found during:** Task 1 (ServiceDetailPanel)
- **Issue:** Initial Write produced `<h3 ...>\n        PRICING\n      </h3>` per Prettier-ergonomic JSX. The acceptance criteria's literal grep `>PRICING<` returned 0 across all 6 section labels.
- **Fix:** Collapsed each label-bearing h3 onto a single line: `<h3 ...>PRICING</h3>`. Verbatim grep contract is authoritative; Prettier ergonomics are not.
