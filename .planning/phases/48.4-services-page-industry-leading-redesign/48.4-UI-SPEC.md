## 0. Intent in one paragraph

`/services` is a six-tile catalog, not a marketing brochure. Within the first 1-2 viewports a visitor must see (a) what Glitch Studios actually sells, (b) entry-level price for each option, and (c) one tap to a sticky-CTA detail overlay. Hero is an Embla autoplay carousel of credibility-builders. Cards are content-rich (typography-driven), no icon-in-empty-frame. Mobile is a dense 2-column grid that reads like an app launcher; desktop is a 3-column grid. Detail opens in a Drawer (mobile) / Dialog (desktop) reusing the `/beats` license-modal primitive. Geometry, color, type, and motion follow the beats-page rubric literally.

## 2. Anti-Patterns — Explicitly Ruled Out

These are the failure modes from the two rejected redesigns. The checker must flag any implementation that introduces them.

| # | Anti-pattern | Why ruled out |
|---|---|---|
| AP-01 | Static text hero (eyebrow + headline + buttons) | User explicitly wants an Embla autoplay carousel matching the beats hero pattern |
| AP-02 | Lucide icon centered in an empty 4:3 / 16:9 cover frame | Underpowered visual unit. Card cover must be content-rich (typography or media-driven) |
| AP-03 | `grid-cols-1` mobile card stack | Reads as a "long weird scroll" feed. Mobile MUST be `grid-cols-2` minimum |
| AP-04 | Generic responsive (same desktop blocks shrunk) | Mobile is a dedicated design layer with its own structural decisions |
| AP-05 | `rounded-md` / `rounded-lg` / any non-zero border-radius | Brutalist DNA — `rounded-none` everywhere |
| AP-06 | Drop shadows, soft glows | Beats rubric: depth via stacked surfaces and borders only. **One exception:** the optional selected-tile glow already in service-grid.tsx (`shadow-[0_0_20px_rgba(255,255,255,0.08)]`) is removed — replace with inverted (white-on-black) tile treatment |
| AP-07 | Tailwind-named color tokens (`bg-gray-900`, `text-zinc-400`) | Beats rubric: literal hex from the palette only |
| AP-08 | `placehold.co` URLs, lorem ipsum, "Coming Soon" boilerplate inside the live surface | Surface must read as production, not as scaffold |
| AP-09 | Detail panel inline in the card grid (current mobile accordion) | Replaced by Drawer/Dialog overlay — same primitive as beats license-modal |
| AP-10 | Long single-column scroll of service cards on mobile | Mobile must show 4+ options before any scroll within the grid |

## 4. Hero Carousel — Spec

Component: `<ServicesHeroCarousel />` at `src/components/services/services-hero-carousel.tsx` (new). Mirrors `BeatsHeroCarousel` mechanics.

### Mechanics

| Property | Value | Source |
|---|---|---|
| Library | `embla-carousel-react` + `embla-carousel-autoplay` | beats hero |
| Loop | `true` | beats hero |
| Align | `start` | beats hero |
| Autoplay delay | `6000ms` | bumped from beats' 5000ms — service slides have denser copy, need a beat longer |
| `stopOnInteraction` | `false` | beats hero |
| `stopOnMouseEnter` | `true` | beats hero (pause-on-hover) |
| Dot indicators | yes, 3-5 dots centered bottom (`absolute bottom-3 left-1/2 -translate-x-1/2 gap-2`) | beats hero |
| Dot style | active `bg-[#f5f5f0]` size `h-2 w-2 rounded-full`, inactive `bg-[#444]` | beats hero |
| Click dot | `emblaApi.scrollTo(i)` | beats hero |
| Reduced motion | `prefers-reduced-motion: reduce` halts autoplay (autoplay plugin honors it); dots still functional | accessibility default |

### Slide count

**4 slides.** This is the locked default. The slide order is fixed (no DB-driven content this phase).

Each slide is a flexbox column, centered both axes, on a layered-gradient background. Same composition as beats hero slides.

### Per-slide content (locked)

| # | Eyebrow | Headline | Sub-copy | CTA label | CTA action | Accent hue |
|---|---|---|---|---|---|---|
| 1 | `SERVICES` | `WE BUILD WHAT YOU HEAR.` | Sessions, mixes, masters, video, design — under one roof. | `BROWSE SERVICES` | scroll into `#service-grid` | green `rgba(0,255,65, …)` (matches beats slide 1) |
| 2 | `STUDIO` | `RECORDING FROM $50/HR.` | Pro-tooled rooms. Real engineer. No padded-room demo loops. | `BOOK A SESSION` | `<Link href="/book?service=recording-session">` | purple `rgba(139,92,246, …)` (matches beats slide 2) |
| 3 | `POST` | `MIXING & MASTERING — $75/TRACK.` | One-track turnaround. Stems delivered. Industry chain. | `START A MIX` | `<Link href="/book?service=mixing-mastering">` | amber `rgba(245,158,11, …)` (matches beats slide 3) |
| 4 | `VISUAL` | `VIDEO. SFX. GRAPHICS.` | Music videos, sound design, cover art — for the artists we record. | `BROWSE SERVICES` | scroll into `#service-grid` (same target as slide 1; §22 Q1 resolved option (a)) | cyan `rgba(34,211,238, …)` (new — extends beats palette) |

**Copywriting rules:**
- Headlines are ≤ 5 words, all-caps, no punctuation mid-line. Period at end is allowed (used as a confident full-stop, not as a question or ellipsis).
- Sub-copy is one sentence, ≤ 12 words, no marketing fluff.
- CTA labels are imperative verb + noun, all-caps mono, 2-3 words.
- No exclamation points. Ever. Brutalist confidence ≠ shouting.

### Background treatment (per slide)

Each slide uses the exact layered-gradient technique from `beats-hero-carousel.tsx` lines 75-82. Format:

```css
background:
  radial-gradient(ellipse at 70% 50%, {accent rgba 0.08} 0%, transparent 60%),
  repeating-linear-gradient(0deg, transparent, transparent 3px, {accent rgba 0.03} 3px, {accent rgba 0.03} 4px),
  repeating-linear-gradient(90deg, transparent, transparent 8px, {accent rgba 0.02} 8px, {accent rgba 0.02} 9px),
  linear-gradient(to right, #0a0a0a 0%, {accent-deep-tint} 50%, #0a0a0a 100%);
```

The accent-deep-tint per slide (used in the horizontal linear-gradient midpoint):
- Slide 1 (green): `#0a0f0a`
- Slide 2 (purple): `#0d0a14`
- Slide 3 (amber): `#140d0a`
- Slide 4 (cyan): `#0a1014`

**Anti-pattern reminder:** these gradients are textural noise, not imagery. No photos, no illustrations, no `bg-image-url` in this carousel. The brand DNA is typographic confidence, not stock photography.

### Card cover treatment options (Claude's discretion — pick ONE direction per surface, justify in spec)

The rejected attempts both failed because the cover slot promised media but delivered an icon. This contract has two acceptable directions; the executor chooses **direction A** as the default. Direction B remains available for a future iteration when real media exists.

**Direction A: Typography-driven (DEFAULT — locked for this phase).** No isolated icon. The card *is* the cover — service name takes the visual weight. The index (`01`, `02`, …) in the top-left becomes the "graphic" element. Subtle dotted grid texture (8px × 8px, `rgba(245,245,240,0.02)`) sits behind everything to give the surface depth without imagery.

**Direction B: Media-driven (future — explicitly NOT this phase).** Top 40% of the card is real product photography (a tracking room, a mix board, a video set). Requires real photos which don't exist yet. Deferred.

### Chip primitives (price + duration)

Both chips use the beats badge pattern.

```html
<span class="rounded-none border border-[#333] bg-[#222] px-1.5 py-0.5 font-mono text-[11px] uppercase text-[#f5f5f0]">
  $50/HR
</span>
```

**Chip content rules:**
- **Price chip:** parse `service.priceLabel`. If contains `/hr` or `/track` → render as-is, mono, uppercase. If begins with `from $` → render `FROM $X`. Examples: `$50/HR`, `$75/TRACK`, `FROM $500`, `FROM $200`, `FROM $150`.
- **Duration chip:** if `service.durationMinutes` is set → `formatDuration(minutes)` rendered as `2 HRS` / `1 HR` / `45 MIN`. If null → omit the chip entirely (do not render an empty slot or a `—`).
- Custom Request tile: chips are `CUSTOM` and `BY BRIEF` (see §7).

### Hover-only "more" affordance (top-right dots)

Two horizontal dots `●  ●` in the top-right corner of the card, `text-[#555]`, `opacity-0` at rest, `opacity-100` on `group-hover` (desktop only — hidden on mobile). Visual hint that the card opens to more content. NOT a clickable element — the whole card is the click target.

```html
<span class="absolute top-3 right-3 hidden md:block opacity-0 group-hover:opacity-100 transition-opacity font-mono text-[14px] text-[#555] tracking-[0.2em]">··</span>
```

### Click behavior

The entire card is a `<button type="button">` (not a Link). On click:
1. Open `<ServiceDetailOverlay slug={service.slug} />` (see §8).
2. Push `#${service.slug}` to URL via `history.replaceState` (deep-link support — matches existing service-grid).
3. No navigation. Stays on `/services`.

**Keyboard:** `Enter` / `Space` opens the overlay. `Tab` cycles through cards in grid order.

## 14. Motion + Interaction

| Element | Trigger | Duration | Easing | Notes |
|---|---|---|---|---|
| Card hover bg/border | `:hover` | 150ms | default (linear-equiv) | `transition-colors duration-150` |
| Glitch overlay on card | `group-hover` | 200ms | site `animate-glitch-hover` | `motion-reduce:hidden` |
| Card active state (mobile tap) | `:active` | 100ms | default | `active:bg-[#0a0a0a]` |
| Hero autoplay | mount | 6000ms per slide | embla autoplay default | `stopOnMouseEnter: true`, `stopOnInteraction: false` |
| Hero dot click | `onClick` | embla default | embla default | smooth scroll to slide |
| Drawer open/close | open prop | Vaul default ~300ms | Vaul default | slide up from bottom |
| Dialog open/close | open prop | Radix default ~150ms | Radix default | fade + small scale |
| Overlay content swap | `selectedSlug` change | 150ms | `motion/react` `AnimatePresence mode="wait"` | opacity 0 → 1 |
| CTA hover (sticky overlay) | `:hover` | 150ms | linear | `bg-[#f5f5f0] → bg-[#e5e5e0]` |
| CTA hover (closing-CTA) | `:hover` | 150ms | linear | border → fill swap |
| Hero CTA hover | `:hover` | 150ms | linear | border → fill swap |
| Hover dots (`··`) | `group-hover` | 150ms | linear | opacity 0 → 1 (desktop only) |

**Hard rule:** all transitions ≤ 200ms. Anything longer feels sluggish in the brutalist aesthetic (beats rubric §6).

**Reduced motion:**
- `prefers-reduced-motion: reduce` → glitch overlay hidden (`motion-reduce:hidden`)
- `prefers-reduced-motion: reduce` → hero autoplay pauses (embla autoplay honors media query natively)
- Hover color transitions stay (they're not vestibular triggers)

## 15. Mobile-First Design Layer (why mobile is NOT shrunken desktop)

Mobile is a dedicated design with structural decisions that don't exist on desktop. These are the four mobile-only choices:

| # | Decision | Mobile (≤640px) | Desktop (≥768px) | Reason |
|---|---|---|---|---|
| M-01 | Grid columns | 2 | 3 | 2-col at 375px → 159px tiles → readable name + chip. 3-col at 375px = 104px tiles, too cramped (Codex audit verdict). |
| M-02 | Card aspect | `aspect-square` | `min-h-[220px]` fluid | Square tiles match phone launcher / catalog patterns. Desktop fluid lets the wider tile breathe with the longer prose. |
| M-03 | Detail surface | `<Drawer>` bottom-sheet | `<Dialog>` centered modal | Beats license-modal pattern. Drawer is the iOS-native bottom-sheet idiom — desktop dialog is the desktop-native idiom. |
| M-04 | Short description | `line-clamp-1` (one line max) | `line-clamp-2` (two lines max) | Mobile card has 159px width — even two lines push the chip row off-screen. Desktop has room for two. |
| M-05 | Hero CTA spacing | `px-5 py-2` | `px-6 py-2` | Slightly tighter to fit headline-CTA-dots inside 260px hero height. |
| M-06 | Hover affordance | hidden | "··" dots top-right | Mobile has no hover. The whole tile is tap-active. |
| M-07 | Eyebrow bar copy | `TAP TO VIEW` | `CLICK TO VIEW` | Native idiom per platform. |

**What is NOT a mobile-only decision (shared between mobile + desktop):**
- Color palette
- Typography family + size ratio
- Brutalist geometry (`rounded-none`, 1px borders, no shadows)
- Hero composition (eyebrow + headline + sub + CTA + dots — same anatomy, different sizes)
- 6-tile catalog content
- Detail overlay content order and section labels
- Sticky bottom CTA inside the overlay

The brand DNA (rubric §7) is universal. Layout structure adapts per viewport.

## 21. Inspirations Considered (and why we're not copying them)

| Inspiration | What we took | What we rejected |
|---|---|---|
| **Cal.com event-type grid** | Dense option grid as the primary surface, not an FAQ | Cal's airy padding, brand-pastel palette — wrong for brutalist DNA |
| **Acuity Scheduling service picker** | Service-as-tile with price + duration as primary metadata | Their cards use stock illustrations — we explicitly reject visual decoration that isn't load-bearing |
| **Bay Eight Recording Studios** | Recording-studio surface positioned with confidence (no "book your dream session" boilerplate) | Their hero is photo-heavy — we don't have real photos and won't fake them |
| **`/beats` (own surface)** | Hero carousel mechanics, license-modal Drawer/Dialog primitive, badge/chip pattern, sticky control bar, glitch hover overlay, eyebrow micro-label | None — `/beats` is the canon |
| **Brutalist Framework (brutalistframework.com)** | Geometric discipline (`rounded-none`, no shadows) — already baked into the rubric | Their explicit "ugly on purpose" stance — we're brutalist *and* designed, not anti-design |

**Net direction:** the closest peer is our own `/beats`. The whole spec is a deliberate inheritance from beats with services-specific structural decisions (no filters → eyebrow bar replaces filter-bar; tap-tile → drawer/dialog replaces card-as-link).
