## Decisions Made

- **Eyebrow option count is `enrichedServices.length + 1`** — Custom Request tile renders as a sibling in the grid (Plan 48.4-03) and must be counted to match what the user sees. Empty-state branch shows `0 OPTIONS · CHECK BACK SOON` instead (not `1 OPTIONS · TAP TO VIEW`) because when no DB services exist, the empty-state grid replaces the Custom Request tile with a full-width NO SERVICES LIVE YET tile.
- **Eyebrow + closing CTA stayed inline JSX** — UI-SPEC §5 + §9 document these as "inline in page.tsx". Single caller, locked literal copy, no parameterization needed. Extracting <ServicesEyebrow /> or <ServicesClosingCTA /> for a single use would be over-engineering.
- **Static `<h1>Services</h1>` removed** — UI-SPEC §18 mandates hero slide H2s as the page H1s. The page now has no static heading; the hero owns the page-title surface.
- **`git rm` for service-tabs.tsx** — Single command stages the deletion. Follow-up `git status` confirms `D  src/components/services/service-tabs.tsx`. Per-commit deletion check (HEAD~1 vs HEAD) flags it as the only deletion — expected, documented.
- **Playwright spec uses `SERVICES_URL = http://localhost:3010/services`** — playwright.config.ts baseURL defaults to `http://localhost:3004` (production-like PM2 surface), but CLAUDE.md history + dev workflow run live code on `:3010` via `pnpm dev`. Spec hardcodes `:3010` with env override so verification runs against the surface most likely to have current code, not the production-mirror that often serves stale chunks.
- **No playwright.config.ts edits** — The spec is the only artifact; modifying config is out of scope and would risk regressing other phases' tests. `outputDir` stays as `test-results/`. Operator copying failure artifacts to `.playwright-mcp/48.4/` is the documented workflow (user global preferences).
