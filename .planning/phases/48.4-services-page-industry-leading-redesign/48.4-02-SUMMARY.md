## Files Created/Modified

- `src/components/services/service-card.tsx` (new) — Client component, single `<button>` root with index + name + description + chip row, mobile aspect-square + desktop fluid min-h-[220px], dotted-grid background texture, glitch hover overlay (motion-reduce safe), hover dots desktop only, focus-visible outline

## Decisions Made

- **Whole card is `<button type="button">`, not a `<Link>`** — UI-SPEC §6 click behavior requires overlay open, not route navigation. Keyboard Enter/Space already triggers `onClick` per HTML semantics.
- **Glitch hover overlay kept** — Rubric reserves it for "high-intent cards" and the services grid is exactly that surface. `motion-reduce:hidden` honors accessibility prefs.
- **Helpers exposed via `@/lib/services/...` path** — Plan 04 overlay may reuse `formatPriceChip` for the pricing block (per plan note). Path is server-safe (no `"use client"`), so overlay can call it from either side of the boundary.
- **Decimal-hour preservation** — Used `toFixed(1).replace(/\.0$/, "")` rather than naive Number conversion so `150 / 60 = 2.5` survives but `120 / 60 = 2` is caught by the prior `Number.isInteger` branch.
