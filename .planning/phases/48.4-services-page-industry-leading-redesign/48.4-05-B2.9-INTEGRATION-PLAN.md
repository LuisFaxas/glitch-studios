## 3. Component architecture

### 3.1 ServiceCard → accordion row
- Props: `service` (name, slug, priceLabel, durationMinutes, shortDescription, deliverables, isBookable, ctaText), `icon` (pixelarticons name), `isOpen`, `onToggle(slug)`.
- Closed head: `[icon] NAME … PRICE [caret]`. All mono (`--ico-rest` ≈ `#d8d8d2`, name `#f5f5f0`, price `#8a8a8a`, neutral 1px border + neutral 4px top edge).
- Open: `--accent` (the service hue) drives top-edge, eyebrow, price, chips, button, and `.ico` background-color. Subtle accent radial+linear gradient bg.
- Interior B body: eyebrow (`SERVICE · {category word}`) + `shortDescription` + `WHAT YOU GET` list from **`service.deliverables`** (already 3 items) + chips (`formatPriceChip` + `formatDurationChip`) + full-width button.
- Icon via CSS mask:
  ```
  .ico { background: currentColor (or --accent when open);
         -webkit-mask: var(--icon) center/contain no-repeat;
                 mask: var(--icon) center/contain no-repeat; }
  ```
  `--icon: url('/icons/services/{name}.svg')` — **vendored locally** in `public/icons/services/` (resolved §7.1), no runtime CDN dependency. Fetch the 7 SVGs from iconify once at build-prep and commit them.
- Button label: `ctaText` if set, else `isBookable ? "BOOK THIS SERVICE" : "REQUEST A QUOTE"`. Link target: bookable → `/book?service={slug}`, else `/contact?service={slug}`.
- A11y: `<button aria-expanded={isOpen} aria-controls={bodyId}>` for the head; body `id`. Keyboard Enter/Space toggles. Caret rotates.

### 3.2 ServiceGrid → mono-stack
- `flex flex-col` stack (NOT grid-cols-2). `selectedSlug` state = single-open accordion (click open card again to close).
- **Custom Request** = the 7th row, **same row primitive with `variant="link"`** (resolved §7.5): identical closed-row look; click navigates to `/contact?service=custom` instead of toggling open. No open body.
- **Deep-link**: keep the existing `useEffect` + **`setTimeout(0)` deferral** — load-bearing per CLAUDE.md ranking-filter-safety + RESEARCH Pitfall 6. On load with `#slug`, open that card. On open/close, `history.replaceState`. (Now opens a card, not an overlay — same hash mechanism.)
- Icon mapping (by slug):
  | slug | icon |
  |---|---|
  | recording-session | mic-sharp |
  | mixing-mastering | sliders-2 |
  | sfx-design | audio-waveform-sharp |
  | video-production | video-sharp |
  | photography | camera-sharp |
  | graphic-design | image-sharp |
  | (custom request) | plus-box-sharp |
  Fallback for unknown slug/type → `plus-box-sharp`.

### 3.3 Removals
- `<ServiceDetailOverlay>` + its Drawer/Dialog. Confirm no other importers (`grep -rn ServiceDetailOverlay src`). `useMediaQuery` may become unused here.
- `portfolioByServiceId` / `getPortfolioForService` — interior B has no example-work. **REQUIRED:** remove the prop from `ServiceGrid` and stop fetching in `page.tsx` (saves N queries). Confirmed no other live consumer on this surface.

## 4. Data change — Photography (the risk-gated part)

**Enum propagation (all four spots — REQUIRED, Codex catch):**
1. `src/db/schema.ts` `serviceTypeEnum` — add `"photography"`.
2. `src/actions/admin-services.ts:69` — add to the `type` union.
3. `src/components/admin/service-page-form.tsx` — add to `SERVICE_TYPES`, the zod enum, and the union type.

**Migration (idempotent, mirror Phase-29 `placeholder` pattern):**

- Statement 2+ (separate): insert/seed the Photography row + apply the sortOrder reorder.
- Add the migration to `src/db/migrations/` and wire it into the existing runner (this repo uses manual idempotent SQL migrations alongside drizzle-kit, e.g. `0010_*`, Phase-29).

**Photography row:** name `Photography`, slug `photography`, type `photography`, priceLabel `FROM $250`, shortDescription + description (draft below), isActive true, **not** in `serviceBookingConfig` → `isBookable=false` → button = `REQUEST A QUOTE`.

**`deliverablesFor()` (page.tsx):** add `case "photography": return ["Studio or location shoot", "Lit and retouched", "Press + cover-ready exports"]`.

**sortOrder reorder (REQUIRED — current order is wrong for the design):**
Current seed order is Recording(0) · Mixing(1) · **Video(2) · SFX(3)** · Graphic(4) — Video precedes SFX. Locked design order is audio→visual→custom:

## 5. Brand / safety constraints (carry over)
- Brutalist: `rounded-none`, 1px borders, no drop shadows. Literal hex from the muted palette only.
- Muted palette: `#4eb47a` Rec · `#db8d36` Mix · `#aabd58` SFX · `#4cabbb` Vid · `#c2735a` Pho · `#897dc7` Gfx · `#8a857e` Cst.
- **Ranking-filter safety**: no synchronous React setState inside native input/focus/visibility event paths. Accordion toggle is a click handler (safe); keep the deep-link `setTimeout(0)` deferral.
- `motion-reduce`: color transitions are fine (not vestibular); no animations to gate here (glitch removed).
- Icons sourced (Pixelarticons), not Claude-drawn. No improvised SVG/keyframes.
