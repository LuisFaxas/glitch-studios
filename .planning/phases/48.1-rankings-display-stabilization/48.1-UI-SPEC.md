# Phase 48.1 - UI Design Contract

## Intent

Replace the fragile leaderboard table with a simpler rankings display without changing the page's identity. This is a stabilization UI contract, not a marketing redesign.

## Preserve

- GlitchTech dark monochrome base: `#000`, `#0a0a0a`, `#111`, `#1a1a1a`, `#222`.
- Text colors: primary `#f5f5f0`, muted `#888`, disabled `#444`.
- Font split: mono for structural ranking data and labels, sans only for prose if needed.
- Square/sharp chrome, no rounded card treatment.
- Existing filter bar and mobile filter button.

## Display Contract

The rankings display must show one row/card per filtered product. Each item must include:

- rank number
- product name linked to `/tech/reviews/{reviewSlug}`
- compact metadata chips for sub-category/specs when available
- BPR tier and BPR score when available
- GlitchMark score when available
- benchmark values for the configured benchmark columns, or a clear dash/not-tested state
- year
- price

## Desktop Layout

- Use a dense custom grid/list, not a decorative card wall.
- Keep row height stable; filter changes must not produce large layout jumps.
- Use borders and column labels for scanability.
- Avoid sticky cells, heavy paint containment, transform scale on result rows, and duplicate hidden result trees.
- Product name should truncate safely without overlapping score columns.

## Mobile Layout

- Use the same data source and avoid rendering a second hidden heavy tree on desktop.
- Mobile rows may stack into compact cards.
- The floating filter sheet trigger remains available.
- Touch targets must be at least 44px where practical for controls.

## Interaction Contract

- Clicking product text opens the review link.
- Clicking filter controls updates local filter state.
- Reset clears local filters and restores the full result set.
- Sort interaction remains out of scope unless implemented without TanStack and without route/URL churn.

## Accessibility Contract

- The ranking list has a meaningful `aria-label`.
- Rows/cards expose product links as real anchors.
- Score labels are text, not color-only.
- Dash/not-tested states use `title` or visible text, not portal tooltips.

## Acceptance

- Default `/tech/rankings/laptops` visually resembles the proven `?simple=1` path but is polished enough for production.
- `?simple=1` is not needed for stability.
- No in-app explanatory diagnostic text remains in the default display.
