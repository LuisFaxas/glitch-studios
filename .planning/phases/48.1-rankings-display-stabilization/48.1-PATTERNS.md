## Implementation Patterns

### Stable Display Seed

Use the current simple diagnostic path in `src/components/tech/leaderboard-table.tsx` as the seed:

- local filter state
- `applyFilters`
- `deriveBounds`
- `LeaderboardFilters`
- direct `filteredRows.map(...)`
- `Link prefetch={false}` for product links

Promote this pattern into default runtime and delete the dispatcher.

### Filtering Pattern

Keep the current deferred local state update pattern:

- `setFilters` wraps `setFiltersState` in `window.setTimeout(..., 0)`
- `onResetFilters` sends empty arrays/null values
- filters are not reflected to URL

### Row Identity Pattern

Use stable keys:

- `key={row.reviewId}` for display items
- never array index as key for product rows

### Not-Tested Pattern

Use native `title` for contextual not-tested/dash states. Avoid portal tooltip systems in the ranking display.

### Human Gate Pattern

Mirror the 29.3 real macOS verification style:

- record production URL
- record Safari result
- record Firefox result
- keep failed attempts as historical truth instead of rewriting them
