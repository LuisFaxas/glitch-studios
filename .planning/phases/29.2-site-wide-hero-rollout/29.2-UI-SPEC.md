## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, eyebrow-to-h1 tight stacking |
| sm | 8px | Stat card internal padding, badge-to-label gaps |
| md | 16px | CTA button horizontal padding, card internal padding |
| lg | 24px | Hero vertical center-padding, section padding |
| xl | 32px | Gap between stat cards row and next section |
| 2xl | 48px | Hero bottom → first content section gap |
| 3xl | 64px | Page-level vertical breathing room between major sections |

Exceptions:
- Hero heights follow a named size scale (not spacing tokens): compact = 200px, default = 280px, tall = 400px. These are layout height values, not spacing tokens.
- Touch target minimum: 44px height on all CTAs (WCAG 2.5.5). The existing hero CTA `py-2` (16px × 2 + 12px line height) falls slightly short at mobile; hero CTA must add `min-h-[44px]` on mobile.
- Category tile: `aspect-square` maintained; tile min-width 160px on desktop grid.

## Typography

All type in the GlitchTech brand context is monospaced for structure-level
elements (headings, labels, stat values, code) and sans-serif for body prose.

| Role | Size (mobile / desktop) | Family | Weight | Line Height | Usage |
|------|------------------------|--------|--------|-------------|-------|
| Hero h1 | 24px / 40px | font-mono | 700 (bold) | 1.1 | TechHero title, wrapped in `<GlitchHeading>` |
| Section h2 | 20px / 28px | font-mono | 700 (bold) | 1.2 | Discipline cards section header, medal ladder header, stat card values |
| Body prose | 13px / 15px | font-sans | 400 (regular) | 1.6 | Hero subhead, card description, methodology paragraphs |
| Label / meta | 10px–11px / 10px–11px | font-mono | 700 (bold) | 1.0 | Eyebrow text, count labels, badge text, table column headers |

Size scale is exactly 4 stops: 10–11 (label), 13–15 (body), 20–28 (h2), 24–40 (h1).
Two weights only: 400 regular + 700 bold. No intermediate weights.

Source: `src/components/tech/tech-hero.tsx` (24/40px h1), `methodology-formula.tsx`
(28px h2), `methodology-discipline-table.tsx` (13px body), `category-tile.tsx`
(10px label). Established in Phase 29.1 — carried forward unchanged.

## Color

GlitchTech operates in a monochrome-base palette with two accent tones (cyan,
amber) used as hero background tints only. No saturated accent is used in UI
chrome.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` | Page background, hero radial gradient base |
| Secondary (30%) | `#111111` / `#1a1a1a` | Cards (`bg-card`), tile surfaces, formula block bg, methodology sections |
| Surface border | `#222222` | All card/table/tile borders, section dividers |
| Primary text | `#f5f5f0` | All headings, CTA labels, stat values, badge text on dark |
| Muted text | `#888888` | Hero subhead, table header labels, count labels, disabled tile text |
| Disabled | `#444444` | Opacity-reduced tiles, secondary borders |
| Accent — cyan | `rgba(56,189,248,0.08)` radial tint | Reserved for: TechHero bg on Reviews, Compare, Benchmarks surfaces only |
| Accent — amber | `rgba(245,158,11,0.08)` radial tint | Reserved for: TechHero bg on About, Categories hub, Blog surfaces only |
| Destructive | `#ef4444` (shadcn --destructive) | Not used in this phase (no destructive actions) |

Accent reserved-for list (complete — not "all interactive elements"):
- Cyan tint: `tone="cyan"` TechHero gradient on `/tech/reviews`, `/tech/compare`, `/tech/benchmarks`
- Amber tint: `tone="amber"` TechHero gradient on `/tech/about`, `/tech/categories`, `/tech/blog`
- Cyan/amber do NOT bleed into card backgrounds, stat cards, discipline cards, medal ladder, or formula block

CTA hover state: border `#f5f5f0` + `hover:bg-[#f5f5f0] hover:text-[#000]` (existing hero pattern, carry forward).
Discipline card hover state: `hover:border-[#444] hover:bg-[#1a1a1a]` (existing CategoryTile hover, apply uniformly).
Row hover on rubric/reference table: translate discipline name 2px right + RGB-split via `<GlitchHeading>` wrapper.

BPR medal monochrome tier colors (from `methodology-medal-table.tsx`, locked in Phase 17):
- Platinum: `bg-[#f5f5f0] text-[#000] border-[#f5f5f0]`
- Gold: `bg-[#888] text-[#000] border-[#888]`
- Silver: `bg-transparent text-[#f5f5f0] border-[#555]`
- Bronze: `bg-transparent text-[#888] border-dashed border-[#444]`

Source: Phase 29.1 RESEARCH.md D-08/D-10, `src/styles/globals.css` token block,
`src/components/tech/tech-hero.tsx` gradient constants, `methodology-medal-table.tsx`.

## Component Contracts

### TechHero — size variants (29.2-01)

Extend `TechHeroProps` with `size?: "compact" | "default" | "tall"`. Height map:

| size | Height | When used |
|------|--------|-----------|
| compact | h-[200px] | Future use — no current surface in this phase |
| default | h-[280px] | Reviews, Compare, Benchmarks, Categories hub, Blog |
| tall | h-[400px] | /tech/about only (brand editorial keystone) |

- Default value: `"default"` (no API break on existing callers)
- Eyebrow: `font-mono text-[11px] uppercase tracking-[0.15em] text-[#555]`; rendered outside the gradient box, above it, 8px gap below
- All other props unchanged (title, subhead, ctaLabel, ctaHref, ctaOpensInNewTab, tone)
- h1 always wraps `<GlitchHeading>` — hover-only RGB split, never auto-runs, reduced-motion honored
- CTA button: add `min-h-[44px] flex items-center` on mobile for WCAG touch target

### Stat Cards Row (29.2-02)

Three cards, displayed as a horizontal row on desktop, wrapping single-column on mobile.
Rendered immediately below the TechHero on `/tech/about`.

Per card:
- Background: `bg-[#111] border border-[#222]`
- Stat value: `font-mono text-[40px] font-bold text-[#f5f5f0]` (desktop), `text-[28px]` (mobile)
- Stat label: `font-mono text-[11px] uppercase tracking-[0.1em] text-[#888]`
- Padding: `p-6` (24px all sides)
- No hover animation — static display only
- No accent color on stat values — monochrome only

Values (locked from CONTEXT.md methodology data audit):
- "43" / "TESTS IN RUBRIC V1.1"
- "13" / "DISCIPLINES"
- "7" / "BPR-ELIGIBLE DISCIPLINES"

### Discipline Cards Grid (29.2-03)

Replaces `<MethodologyDisciplineTable>`. Grid of 13 cards (all disciplines, not just
BPR-eligible). Section id preserved: `id="disciplines"`.

Per card:
- Background: `bg-[#111] border border-[#222]` — matches CategoryTile base
- Hover: `hover:border-[#444] hover:bg-[#1a1a1a]` — matches CategoryTile hover
- Discipline name: `font-mono text-[15px] font-bold uppercase tracking-[0.05em] text-[#f5f5f0]`
- Description: `font-sans text-[13px] leading-relaxed text-[#888]`
- BPR-eligible badge (on 7 cards only): `font-mono text-[10px] uppercase tracking-[0.1em] text-[#000] bg-[#f5f5f0] px-2 py-0.5`
- Non-eligible cards: no badge
- Padding: `p-4` (16px)
- Grid: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` — max 3 columns desktop
- Animated underline on hover (discipline name only): `after:block after:h-px after:bg-[#f5f5f0] after:scale-x-0 group-hover:after:scale-x-100 after:transition-transform after:origin-left`
- No auto-running animation; hover-only

### Medal Threshold Ladder (29.2-04)

Replaces `<MethodologyMedalTable>`. Vertical timeline layout. Section id preserved: `id="thresholds"`.

Layout: vertical stack of 4 tier entries + 1 "No medal" entry.

Per tier entry:
- Left column (fixed 80px): medal chip using existing TIER_CHIP color system (carry from `methodology-medal-table.tsx`)
- Center column: score range — `font-mono text-[20px] font-bold text-[#f5f5f0]`
- Right column: description — `font-sans text-[13px] text-[#888]`
- Connecting line between entries: `border-l-2 border-[#222]` on the left gutter
- Tier node (circle on the timeline): `w-3 h-3 rounded-full bg-[#333]`
- No hover animation on the ladder itself

Tier data (source: CONTEXT.md methodology data audit, locked):
- Platinum: 90–100 — medal chip as existing
- Gold: 80–89 — medal chip as existing
- Silver: 70–79 — medal chip as existing
- Bronze: 60–69 — medal chip as existing
- No medal: < 60 or < 5 of 7 disciplines measured

### Terminal Formula Block (29.2-02, enhancement to existing MethodologyFormula)

The existing `<pre>` / `<code>` block gets terminal-prompt styling.

- Container: `bg-[#0a0a0a] border border-[#222] p-6` (unchanged base)
- Add terminal chrome bar above code: `bg-[#111] border-b border-[#222] px-4 py-2` with 3 dot indicators (`w-3 h-3 rounded-full bg-[#444]`) spaced `gap-2`
- Prompt prefix: `text-[#888] select-none` before formula text — value: `$ `
- Blinking cursor on hover: `group-hover:after:content-['▋'] group-hover:after:animate-pulse after:text-[#f5f5f0]` — hover-only, not auto-running
- Formula text color: `text-[#f5f5f0]` unchanged

### Rubric/Reference Table Row Hover (29.2-02)

For the reference-baseline table in `<MethodologyExclusionPolicy>` or wherever a
flat data table exists on `/tech/about`:

- Row hover: `group` on `<tr>`, discipline name cell gets `group-hover:translate-x-[2px] transition-transform`
- RGB split on discipline name text: wrap in `<GlitchHeading>` (existing component)
- Transition: `transition-transform duration-150`
- No color change on row hover — only the positional shift + GlitchHeading effect

### Category Tile Imagery (29.2-07)

Direction B (placeholder phase acceptable per CONTEXT.md):
- Scale lucide icon from current `h-10 w-10` (40px) to `h-20 w-20` (80px)
- Add caption below icon showing content count: `font-mono text-[11px] uppercase tracking-[0.1em] text-[#888]`
  Caption format: `"{N} reviewed · {N} ranked"` or `"Coming soon"` for empty categories
- Remove existing separate `<span>` for countLabel — merge into caption slot
- Category name: `font-mono text-[15px] font-bold uppercase tracking-[0.05em]` (bump from current `text-lg`)
- Structure: `flex-col items-center justify-center gap-4` (bump from current `gap-3`)

Direction A (launch-quality target, deferred to plan-phase decision):
- If thumbnail exists: `<Image>` as full-tile background with `object-cover`, 60% overlay darkening (`bg-gradient-to-t from-[#000]/80`), lucide icon as `h-8 w-8` badge in top-right corner
- Category name and count text remain, positioned at bottom of tile
- Falls back to Direction B if no thumbnail

Plan 29.2-07 should attempt Direction A for Laptops (only category with content) and
fall back to Direction B for Coming-soon tiles.

## Interaction Contracts

### Hover behaviors (exhaustive — no auto-running animations)

| Element | Trigger | Behavior | Reduced-motion |
|---------|---------|----------|----------------|
| TechHero h1 | mouse hover | RGB-split glitch via `<GlitchHeading>` CSS layers | no animation |
| Discipline card | mouse hover | border → `#444`, bg → `#1a1a1a`, discipline name animated underline | no underline animation |
| Discipline card name | mouse hover (via card group) | `<GlitchHeading>` RGB split | no animation |
| CategoryTile (active) | mouse hover | glitch flash overlay `opacity-0 → opacity-100`, existing `animate-glitch-hover` | `motion-reduce:hidden` (already implemented) |
| Terminal formula block | mouse hover | blinking cursor appended to formula text | no blink |
| Rubric table row | mouse hover | discipline name cell `translate-x-[2px]` + RGB split | no translate |
| Hero CTA button | mouse hover | `bg-[#f5f5f0] text-[#000]` fill (existing pattern) | same (color only, no motion) |

All hover effects are hover-only. Zero auto-running animations on any element in this phase.

### Scrolling and layout

- Desktop sidebar must not overflow its container: adding heroes does not increase sidebar height. Sidebar is not affected by page-level hero or content sections.
- `/tech/about` tall hero (400px) renders before the stat cards row; user scrolls to reach section content. No sticky behavior on the hero.
- Discipline card grid scrolls with the page (no fixed/sticky grid).
- Medal ladder scrolls with the page.
