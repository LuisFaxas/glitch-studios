# Phase 07.4: Brand architecture & Glitch Tech sub-brand foundation - Context

**Gathered:** 2026-04-16
**Status:** Ready for planning

<domain>
## Phase Boundary

Build the foundation for a new "Glitch Tech" entity — a product review vertical (computers, components, peripherals; starting with computers) that lives alongside Glitch Studios at `/tech/*` in the same Next.js codebase. This phase delivers the architectural shell (own route group, own layout, own nav config, own mobile tab bar) and the `/tech` landing page with placeholder content. Shared infrastructure (design system, admin, auth, database, payments, root-level providers) stays centralized.

**Out of scope** — product data model, admin input forms, real review content, comparison tool UX, and category detail pages are Phases 7.5 and 7.6.

</domain>

<decisions>
## Implementation Decisions

### Architecture & Routing
- **D-01:** Glitch Tech is its **own entity**, not a subpage. Implemented as a separate route group `(tech)/` at the same level as `(public)/` and `(auth)/`. Each entity has its own `layout.tsx`, own nav config, own pages. Single Next.js codebase, single Vercel deployment, same domain.
- **D-02:** Tech lives at `/tech/*`. Expected routes this phase (some placeholder): `/tech` (landing), `/tech/reviews` (list), `/tech/reviews/[slug]` (detail template), `/tech/categories`, `/tech/compare`, `/tech/benchmarks`.
- **D-03:** Shared state providers (audio player, auth session, cart, theme) live at **app-root** `app/layout.tsx` level — above both route-group layouts. Ensures cross-entity navigation preserves state.
- **D-04:** Glitch Studios retains its existing structure under `(public)/`. Music, video, beats, services, booking, portfolio, artists, blog, contact all stay as-is. The only structural change to `(public)/` is adding a dedicated "Glitch Tech" promotional section on the homepage (D-14).

### Visual Identity
- **D-05:** Glitch Tech uses a **subtle accent variant** of the Cyberpunk Metro aesthetic. Same monochrome base, same tile grid, same glitch effects, same typography. Differentiation = a single accent element (exact color / texture / motif is Claude's Discretion, research-driven in planning) + a "GLITCH TECH" logo wordmark variant paralleling "GLITCH STUDIOS".
- **D-06:** Page transition between entities uses a brief **fade + glitch flicker** effect on route-group crossover to signal the zone shift without breaking flow. Signature Glitch brand language.

### Landing Page (/tech)
- **D-07:** Full homepage built in Phase 7.4 with **placeholder/stub data**. Real data populated in Phase 7.5 (schema + admin input) and 7.6 (review display). Expected sections:
  1. Hero with scroll-collapsible menu — parity with Glitch Studios homepage
  2. Featured Reviews carousel (placeholder cards)
  3. Category tiles (Computers primary; room for Audio, Peripherals)
  4. Spotlight: top-benchmarked product (placeholder)
  5. Compare CTA — "Compare any 2 products →"
  6. Newsletter signup
  7. Shared footer
- **D-08:** Hero + scroll-collapse behavior reuses the existing `hero-section.tsx`, `scroll-section.tsx`, and `homepage-scroll-watcher.tsx` primitives from the Glitch Studios homepage via the shared component library.

### Navigation Architecture
- **D-09:** Glitch Tech has its own tile-nav (`tileNav` pattern) with tech-scoped items: **Reviews / Categories / Compare / Benchmarks / About**. New config file: `src/components/layout/tech-nav-config.ts` paralleling `public-nav-config.ts`.
- **D-10:** Glitch Studios nav (`publicNavItems`, `mobileTabItems`) does **NOT** get a "Tech" item. The two entities' navs stay fully independent. Glitch Studios nav has no Tech items; Glitch Tech nav has no Studios items.
- **D-11:** Glitch Tech has its own mobile bottom tab bar instance — separate from Glitch Studios. Specific items Claude's Discretion in planning (likely Home / Reviews / Categories / Compare or similar).

### Cross-Navigation Between Entities
- **D-12:** Each sidebar includes a prominent "sister site" **logo tile** showing the other entity's wordmark. On Glitch Studios sidebar: "GLITCH TECH" tile → `/tech`. On Glitch Tech sidebar: "GLITCH STUDIOS" tile → `/`. Visually styled to read as a brand-family portal, not a generic nav item.
- **D-13:** Cross-links use Next.js `<Link>` for **seamless in-tab navigation** (NOT `target="_blank"`). Client-side routing + root-level providers (D-03) preserve audio playback, cart, and auth session across the crossover. The transition animation from D-06 communicates the zone shift.
- **D-14:** Glitch Studios homepage gets a **dedicated Glitch Tech promotional section** — a discovery entry point for first-time visitors. Placement and content Claude's Discretion (likely between portfolio and testimonials, or between services and featured beats, TBD in planning).

### Claude's Discretion
- Exact Tech accent color / texture / motif
- Transition animation parameters (duration, easing, effect shape)
- Cross-link tile positioning and visual weight in each sidebar
- Specific mobile tab bar items for Glitch Tech
- Homepage promotional section placement and copy
- Whether to move existing shared providers to app-root `layout.tsx` if they aren't already there

</decisions>

- **Root `app/layout.tsx`** — verify all shared providers (audio, auth, cart, theme) live here. If any live inside `(public)/layout.tsx`, migrate them up per D-03.

- **Sidebar cross-link** — Glitch Studios tile-nav needs a slot for the "GLITCH TECH" logo tile; Glitch Tech tile-nav mirrors with "GLITCH STUDIOS".

<specifics>
## Specific Ideas

- **Wordmark pair:** "GLITCH STUDIOS" on main, "GLITCH TECH" on tech pages. Same typographic treatment (mono, uppercase, tight tracking).
- **Cross-link precedent:** Vox Media / The Verge ↔ Polygon pattern — two sister publications with distinct identities but shared infra. Our cross-link tile should feel equally intentional.
- **Transition feel:** A deliberate zone-shift, not a page change. Glitch flicker matches the brand. Avoid generic fade-through-white.
- **Placeholder content strategy:** Seeded/stub data lets the Tech landing feel complete end of 7.4, so 7.5/7.6 become integration efforts rather than ground-up builds.
- **Reversibility:** If Tech underperforms long term, collapsing to a subpage is a small refactor (merge `(tech)/` back into `(public)/` and delete the cross-links). If Tech takes off, promoting to a subdomain (tech.glitch.com) is a Vercel config + route rewrites — not a rewrite.
- **No subdomain now:** Single-domain same-codebase keeps SEO unified, avoids auth-cookie / CORS complexity, and matches user's operational preference for simple.

</specifics>
