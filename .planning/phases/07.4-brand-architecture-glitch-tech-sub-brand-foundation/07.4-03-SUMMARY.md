### Deviation

**Plan said:** Add a `glitch_tech_promo` renderer entry and insert in fallback block. Admin could optionally add a DB row to place it.

**Deviation:** The Studios homepage uses `useDynamicLayout = homepageSections.length > 0` — DB-driven render path was active (production uses it). The renderer-map entry alone wouldn't render unless admin added a row. I added **auto-injection**: after the `portfolio` section (or as a trailing fallback if no portfolio exists) the promo renders regardless of DB. Admin can still override by inserting an explicit `glitch_tech_promo` row.

**Why:** TECH-03 requires the promo to be on the homepage, period. Leaving it off behind a DB gate would have silently failed the acceptance criterion.
