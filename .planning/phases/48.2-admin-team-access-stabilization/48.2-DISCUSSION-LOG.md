## Invite Model

| Option | Description | Selected |
|--------|-------------|----------|
| Pending invite account | Admin enters email/name/role, invitee receives a link, sets password, then account becomes usable. | yes |
| Active account immediately | Admin creates active account immediately; email only lets invitee set password. | |
| Existing user upgrade only | Invitee must register first, then admin upgrades role. | |

**User's choice:** User wants the easy/admin-driven path: "I want to be able to add people myself and send them a link so they can make their own password."
**Notes:** Existing-user upgrade remains an edge case, not the main path. The current "User must register first" behavior is explicitly not acceptable.

## Producer Role And Permissions

| Option | Description | Selected |
