**Step 2: Staggered entrance animation (D-09)**

Define constants at module level:
```typescript
const EASE_OUT_CUBIC = [0.215, 0.61, 0.355, 1] as const

const ROW_STAGGER = [
  { delay: 0, y: -8 },     // Logo
  { delay: 0.08, y: 12 },  // Nav row
  { delay: 0.16, y: 12 },  // Widget row (Studio Status + Social)
  { delay: 0.24, y: 12 },  // Auth row (Sign In + Close)
] as const
```

Inside the component, call `const shouldReduceMotion = useReducedMotion()`.

Wrap each of the 4 rows (Logo, Nav row div, Widget row div, Auth row div) in a `motion.div` with:
```typescript
<motion.div
  initial={shouldReduceMotion ? false : { opacity: 0, y: ROW_STAGGER[N].y }}
  animate={{ opacity: 1, y: 0 }}
  exit={{ opacity: 0 }}
  transition={{
    duration: shouldReduceMotion ? 0 : 0.18,
    delay: shouldReduceMotion ? 0 : ROW_STAGGER[N].delay,
    ease: EASE_OUT_CUBIC,
  }}
>
```

For the Logo row specifically: duration 0.2 (200ms per UI-SPEC), y: -8 (slides down from above). All other rows: duration 0.18 (180ms), y: 12 (slides up from below).

Exit animation on all rows: `exit={{ opacity: 0 }}` with `transition.exit: { duration: 0.12 }` — all simultaneous, 120ms fade, no stagger on exit.

**Step 3: Swipe-down-to-dismiss gesture (D-04)**

Add `useMotionValue` and `useTransform` for progressive opacity during swipe:
```typescript
const dragY = useMotionValue(0)
const overlayOpacity = useTransform(dragY, [0, 300], [1, 0])
```

On the outermost `motion.div` (the one with `fixed inset-0`), add drag props:
```typescript
<motion.div
  // existing: initial, animate, exit, transition, className
  drag="y"
  dragConstraints={{ top: 0, bottom: 0 }}
  dragElastic={{ top: 0, bottom: 0.4 }}
  dragSnapToOrigin
  dragMomentum={false}
  onDragEnd={(_, info) => {
    if (info.offset.y > 80 && info.velocity.y > 300) {
      onClose()
    }
  }}
  style={{ y: dragY, opacity: overlayOpacity, touchAction: "none" }}
>
```

Key details:
- `dragConstraints={{ top: 0, bottom: 0 }}` prevents upward drag
- `dragElastic={{ top: 0, bottom: 0.4 }}` allows rubber-banding downward
- `dragSnapToOrigin` returns overlay to position if threshold not met
- `dragMomentum={false}` prevents bounce after snap-back
- `style={{ touchAction: "none" }}` is CRITICAL for preventing browser scroll interference on touch devices
- `onDragEnd` checks offset > 80px AND velocity > 300px/s (per UI-SPEC)
- Progressive opacity via `useTransform`: overlay fades as user drags down, formula `1 - (deltaY / 300)`

**Step 4: Exit animation for swipe dismiss (Pitfall 4)**

Update the `exit` variant on the outer motion.div to slide downward:
```typescript
exit={{ opacity: 0, y: 100 }}
```
This ensures that when dismissed via swipe (overlay already translated down), the exit continues downward instead of snapping back to center. Normal close (button/escape) also exits downward, which is acceptable.

**Step 5: Remove the onClick handler on the outer motion.div** that currently closes on background tap (`if (e.target === e.currentTarget) onClose()`). The drag gesture on the same element would conflict with this click handler. The user can close via: swipe down, close button, escape key, or navigating to a page.

Actually — keep the onClick handler but guard it: only fire if the drag didn't happen. Motion's drag + click coexistence should handle this (small movements treated as taps). Leave the onClick as-is; Motion's built-in tap-vs-drag discrimination will prevent accidental closes during drag.
