## Research-Driven Decisions

### D-01: Menu Content — Do NOT Duplicate Tab Bar Items

**Decision:** The menu overlay shows only destinations NOT in the bottom tab bar.

**Rationale:** The iOS "More" tab pattern is the industry standard. Apps with bottom tab bars (Spotify, Apple Music, Instagram, YouTube) use their overflow/hamburger menus for secondary destinations only. The tab bar already provides one-tap access to Beats, Services, Portfolio, Blog. Duplicating them in the menu wastes space and creates navigation confusion.

**Menu nav items (in order):**
1. Book Session (Calendar icon) — primary CTA, most prominent tile
2. Artists (User icon)
3. Contact (Mail icon)

**Retained non-nav tiles:**
- Sign In / Sign Out (LogIn/LogOut icon)
- Studio Status widget
- Social links row (Instagram, YouTube, SoundCloud, X)
- Close button

**Removed from menu:**
- Beats, Services, Portfolio, Blog (already in tab bar)

This reduces the nav grid from 7 items across 3 rows to 3 items in a single row, dramatically improving breathing room.

### D-09: Entrance Animation — Staggered Tile Entrance

**Decision:** Tiles animate in with a staggered row-by-row entrance, top to bottom.

**Parameters:**
- Logo tile: `opacity 0->1, y -8->0`, duration 200ms, delay 0ms
- Nav row: `opacity 0->1, y 12->0`, duration 180ms, delay 80ms
- Widget row 1 (Studio Status + Social): `opacity 0->1, y 12->0`, duration 180ms, delay 160ms
- Widget row 2 (Sign In + Close): `opacity 0->1, y 12->0`, duration 180ms, delay 240ms
- Easing: `[0.215, 0.61, 0.355, 1]` (ease-out-cubic, matches existing tile press curve)
- Exit: all tiles fade simultaneously, duration 120ms (fast exit, no stagger needed)

**Rationale:** Staggered entrance is standard for creative/music sites with tile-based layouts. The 40ms stagger interval keeps total entrance under 420ms, which feels snappy while providing visual rhythm. The current 160ms flat fade is the baseline — this upgrade adds motion polish without slowing interaction.

### D-10: Now Playing Widget — Remove from Menu

**Decision:** Remove the Now Playing widget from the menu overlay entirely.

**Rationale:** Music apps (Spotify, Apple Music, SoundCloud) do NOT duplicate now-playing information inside navigation menus. The global player bar is already visible as a persistent element — when the menu overlay is open, the player bar sits at the bottom of the screen. Duplicating track info, waveform, and play/pause inside the menu is redundant and wastes valuable viewport space. Removing it frees an entire grid row for the remaining tiles to breathe.

### Grid Specification

- Container: `grid grid-rows-[auto_1fr]` with `h-[100dvh]`
- Logo row: auto height, full width
- Content area: `1fr` fills remaining space
- Content grid: `grid grid-cols-12` with `gap-1.5` (6px)
- Tiles use `h-14` (56px) for minimum touch target compliance
- Content grid uses `grid-rows-[auto_auto_auto]` with `gap-1.5` and vertical centering within the 1fr space via `place-content-center`

## Interaction Contract

### Swipe-Down to Dismiss (D-04)

- Touch start tracked on `touchstart` event
- If vertical swipe distance > 80px downward AND velocity > 0.3px/ms, trigger close
- During swipe: overlay translates Y by touch delta with `opacity: 1 - (deltaY / 300)` for progressive fade
- If swipe canceled (released before threshold): spring back to position, duration 200ms
- Implementation: Framer Motion `useDragControls` with `drag="y"` constrained to `dragConstraints={{ top: 0 }}` on the overlay container, OR manual touch handlers

### Tile Tap States

All interactive tiles inherit existing Tile component behavior:
- Hover: `bg-[#1a1a1a]`, `border-[#444444]`
- Active/pressed: `scale(0.97)`, `bg-[#0a0a0a]`, duration 100ms
- Current page: inverted — `bg-[#f5f5f0]`, `text-[#000000]`, `border-[#f5f5f0]`
- Focus visible: `outline-1 outline-[#f5f5f0] outline-offset-2`

### Glitch Flash on Open (D-08)

Preserved from current implementation:
- `animate-glitch-hover` pseudo-layer at `opacity: 0.28` fading to `0` over 240ms with 80ms delay
- Radial glow gradient background layer animating `opacity: 0.6 -> 0.9`

### Focus Management

Preserved from current implementation:
- Focus trap with Tab/Shift+Tab cycling
- Escape key closes overlay
- Initial focus on active nav item or first nav item
- Focus returns to menu trigger button on close

### Route Change Auto-Close

Preserved: pathname change detection auto-closes overlay.
