# Phase 07.3: Mobile Menu Overhaul - Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix the mobile nav overlay to be clean, usable, and industry-quality. The current overlay has the right direction (tile-based, cyberpunk metro aesthetic) but is poorly executed — too many elements crammed into a fixed viewport with no breathing room. This is a redesign of the overlay layout and content, not a change in approach. The tile grid aesthetic stays. Everything must fit like a puzzle.

</domain>

<decisions>
## Implementation Decisions

### Menu Structure & Content
- **D-01:** Whether nav items in the menu should duplicate the tab bar items (Beats, Services, Portfolio, Blog) or only show extras (Book Session, Artists, Contact, Sign In) is a **research-driven decision**. Researcher must investigate how industry-leading music/creative apps handle hamburger/menu overlays when they already have a bottom tab bar. Do Spotify, SoundCloud, BeatStars, Apple Music, etc. repeat tab bar items in their full menu, or only show extras?
- **D-02:** **Studio Status widget stays** in the menu. Visitors want to know if the studio is available — it's a real signal for a production studio.
- **D-03:** **Social links stay** in the menu (Instagram, YouTube, SoundCloud, X). Standard for creative businesses on mobile.
- **D-04:** **Close button stays as a tile** positioned in the bottom-right corner of the grid. Additionally, **swipe-down gesture to dismiss** must be implemented as an alternative close mechanism.
- **D-05:** **Sign In / Sign Out tile stays** in the menu.

### Layout & Scrolling
- **D-06:** **No scrolling** — the menu must fit in a single viewport (100dvh). This forces discipline and feels app-like. No overflow, no scroll.
- **D-07:** **Keep the tile grid approach** — the current 12-column CSS grid with tiles at varying spans is the right direction. It needs to be reworked so everything fits like a puzzle with proper proportions and breathing room. The Cyberpunk Metro tile aesthetic is core to the brand.

### Visual Treatment
- **D-08:** **Keep the glitch flash + radial glow** on menu open. These are signature Glitch Studios brand touches and must be preserved.
- **D-09:** Whether the entrance animation should be more dramatic (e.g., staggered tile entrance, row-by-row animation) is a **research-driven decision**. Researcher must investigate what creative/music industry sites do for mobile menu entrance animations. The current simple fade (0.16s) is the baseline.

### Now Playing Widget
- **D-10:** What the Now Playing widget should look like in the menu (full widget with waveform, simplified version, or removed entirely) is a **research-driven decision**. The player bar at the bottom already has play/pause, track info, time, and waveform. Researcher must check how music apps handle now-playing info in their navigation menus — is it duplicated, simplified, or absent?

### Claude's Discretion
- Exact tile sizes, grid spans, and gap values for the reworked layout
- Specific Framer Motion animation parameters (duration, easing, stagger delay)
- Safe area inset handling details (already partially implemented)
- Whether to use `framer-motion` gesture recognition for swipe-down or a custom touch handler
- CSS approach for making tiles fit like a puzzle (subgrid, named areas, fixed rows, etc.)

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `Tile` component: Supports `size`, `density`, `layout`, `compact` props — already used in the overlay. Can be configured differently without new components.
- `LogoTile` component: Full-width logo with glitch animation — stays at top of overlay.
- `Waveform` component: Canvas-based, configurable height/barWidth/gradient — used in current Now Playing widget.
- `publicNavItems` / `mobileTabItems`: Centralized nav config — overlay reads from `publicNavItems`, tab bar reads from `mobileTabItems`.
- Social icon components: `InstagramIcon`, `YouTubeIcon`, `SoundCloudIcon`, `XIcon` — already imported in overlay.

### Established Patterns
- `AnimatePresence` + `motion.div` for overlay entrance/exit (Framer Motion)
- Focus trap with keyboard navigation (Tab, Escape) — must be preserved
- Body scroll lock via `document.body.style.overflow = "hidden"` on open
- Route change detection to auto-close overlay
- `handleDeferredClose` pattern: `setTimeout(() => onClose(), 0)` to avoid React state conflicts

### Integration Points
- `BottomTabBar` owns the overlay state (`overlayOpen`) and passes `isOpen` / `onClose` / `triggerRef` to overlay
- `useAudioPlayer` hook provides `currentBeat`, `isPlaying`, `toggle`, etc. for Now Playing widget
- `useSession` from auth-client for Sign In / Sign Out logic

</code_context>
