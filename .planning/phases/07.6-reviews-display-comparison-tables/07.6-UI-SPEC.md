# Phase 7.6 — UI Design Contract
## Reviews Display & Comparison Tables (Public Glitch Tech)

> Visual and interaction contract for the **public-facing** Glitch Tech consumption surfaces — the counterpart to 07.5's admin input surfaces. Phase 07.6 turns the data captured in admin (products, reviews, categories, specs, benchmarks) into three public experiences: the **review detail page**, the **reviews list + category browse**, and the **product comparison tool**. No new schema, no admin UI. Generated by gsd-ui-researcher; inherits decisions from 07.4 (brand foundation) and 07.5 (data model).

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | shadcn/ui | `components.json` exists; `style: "base-nova"`, `baseColor: "neutral"`, `iconLibrary: "lucide"` |
| Preset | base-nova | Locked site-wide (inherited 07.4 / 07.5) |
| Component library | Radix UI (via shadcn) | Established Phase 1 |
| Icon library | Lucide React | `components.json` |
| Fonts | JetBrains Mono (headings, labels, data, buttons), Inter (body copy, review long-form, product descriptions) | DESIGN-LANGUAGE.md; inherited 07.4 |
| Chart library | **Recharts 2.x** (new dependency for this phase) | CLAUDE.md stack; roadmap explicitly calls out "comparison tables with Recharts" |
| Video embed | Native `<iframe>` for YouTube (`https://www.youtube.com/embed/...`) | No `react-youtube`/`react-player` dep; iframe is lighter, already in project style |

**Key inheritance note:** Phase 07.6 is **public-facing content**, not admin tooling. Unlike 07.5 (which applied the monochrome palette in a quiet utilitarian form), 07.6 applies the **full Cyberpunk Metro treatment**: glitch hover on tiles, `ScrollSection` entrance animations, sharp-edged brutalist layout, heavy use of wordmarks. The review page and category browse are a content experience — the comparison tool is a data-dense analytical view. Both share the 07.4 visual language.

## Spacing Scale

Declared values (all multiples of 4, inherited from 07.4 / 07.5 / `globals.css`):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon-to-label gaps in rating chips, pros/cons bullet spacing, specs row internal padding |
| sm | 8px | Spec row vertical rhythm, gallery thumb gap, inline chip gap |
| md | 16px | Card inner padding, default text block margin, section-header-to-content gap |
| lg | 24px | Page content gutters (`px-6`), vertical gap between review body paragraphs and callouts |
| xl | 32px | Vertical spacing between review sections (verdict → body → rating → pros/cons → gallery) |
| 2xl | 48px | Review hero vertical padding, comparison table header-to-body gap |
| 3xl | 64px | Page-top padding on review detail (under nav), bottom padding before related reviews |

**Exceptions:**
- Review hero image: aspect ratio `16:9` at full viewport width (mobile) or `max-w-7xl` (desktop) — no padding override
- Comparison table column width: minimum 200px on desktop to avoid cramped spec rows; horizontal scroll triggered below
- Gallery lightbox close-button touch target: 44×44px
- Rating bar visualization: 40px track height (matches 07.5 admin RatingSlider for visual consistency)
- Recharts `ResponsiveContainer` height: 320px (desktop) / 240px (mobile)

## Typography

All typography inherited from 07.4 / 07.5. Public pages use the same 4-size, 2-weight scale — **no additions**.

| Role | Size | Font | Weight | Line Height | Usage in this phase |
|------|------|------|--------|-------------|---------------------|
| Hero title / Display | 64px (desktop) / 44px (mobile) | JetBrains Mono | 700 | 1.0 | Review detail page title over hero image; comparison page title |
| Section heading | 28px | JetBrains Mono | 700 | 1.2 | "Verdict", "Pros & Cons", "Specifications", "Benchmarks", "Gallery", "Who this is for", section headings in comparison tool — `uppercase tracking-[0.05em]` |
| Body | 15px | Inter | 400 | 1.5 | Review body long-form copy (Tiptap-rendered HTML), product descriptions, verdict paragraph, audience paragraphs |
| Small | 11px | JetBrains Mono | 700 | 1.2 | Category breadcrumbs, "Reviewed by", publish date, unit badges next to numbers (kg, GB, score), benchmark metric labels |

**Additional conventions (inherited, reaffirmed):**
- Labels, section headings, buttons, metadata, chart axis labels: `font-mono uppercase tracking-[0.05em]`
- Body copy, review text, product descriptions: `font-sans normal-case`
- Rating score display (numeric overall): `font-mono font-bold`, size 28px (e.g. `8.4`)
- Big benchmark numbers in spotlight/comparison: `font-mono text-2xl font-bold` (matches existing `tech-benchmark-spotlight.tsx` pattern)
- Star icon pairs with rating: Lucide `Star` 12px, `fill-[#f5f5f0] text-[#f5f5f0]` (matches existing `tech-featured-reviews-carousel.tsx`)

**Review body HTML rendering:**
- The `review.body_html` from Tiptap renders inside a wrapper with `prose prose-invert max-w-none font-sans text-[15px] leading-relaxed` — equivalent styling to existing blog post render
- Headings inside body (`<h2>`, `<h3>`) inherit the `font-mono uppercase` treatment via a scoped CSS class `.review-body h2, .review-body h3 { @apply font-mono uppercase tracking-[0.05em]; }`
- Inline images from Tiptap render full-width, bordered `border border-[#222]`

## Color

**Strict monochrome, inherited from 07.4 and 07.5.** No Tech accent introduced in 07.6 — the accent decision was deferred in 07.4 and the 07.5 admin contract explicitly declined to introduce one. 07.6 continues that deferral. **Rationale:** the review page derives visual interest from the hero image, rating bar fills, glitch hover effects, and wordmark typography — adding a color accent would fragment the "Tech = Studios sister brand" visual story and risks collision with future product-category color coding.

| Role | Value | Usage in this phase |
|------|-------|---------------------|
| Dominant (60%) | `#000000` | Page background, review hero backdrop, comparison table background |
| Secondary (30%) | `#111111` | Review card surfaces, spec row backgrounds, verdict callout, pros/cons panel, comparison cell hover, benchmark bars background track |
| Tertiary surface | `#1a1a1a` | Hover lift on review cards, active tab background in comparison product picker, hover on comparison rows |
| Borders (default) | `#222222` | Card borders, table dividers, section separators, gallery thumb borders, spec table rules |
| Borders (emphasis) | `#444444` | Card border on hover, comparison table "winner" column outline |
| Borders (inverted) | `#f5f5f0` | Featured/editor's-choice spotlight outline (`border-2 border-[#f5f5f0]` — matches `tech-benchmark-spotlight.tsx` existing pattern); comparison table "best score" cell outline |
| Text (primary) | `#f5f5f0` | Review title, body text, product names, big numbers, CTA labels |
| Text (label / muted) | `#888888` | Review excerpt, verdict subline, "Reviewed by", date, field labels in specs, axis labels in Recharts |
| Text (tertiary) | `#555555` | Meta text, breadcrumb separators, empty state body, placeholder thumbnail text |
| Text (inverted) | `#000000` | Text on the inverted `#f5f5f0` surfaces (primary CTA button, "winner" badge, active filter pill) |
| Active surface (inverted) | `#f5f5f0` | Primary CTA background ("Read full review", "Compare"), active filter pill, "winner" badge background |
| Destructive | `#dc2626` | Cons bullet marker `−` (NOT the full text — just the marker), "Not recommended" callout if used (optional) |
| Chart fill (bar) | `#f5f5f0` | Recharts bar fill (all bars monochrome white); "winner" bar OR highlighted product gets `#f5f5f0`, others get `#555555` |
| Chart fill (compared) | `#555555` | Non-winning / reference product bars in comparison chart |
| Chart axis/grid | `#222222` | Grid lines, axis ticks |

**Accent reserved for (10% rule):** The inverted `#f5f5f0` surface is this phase's "accent" (same pattern as 07.5). Reserved strictly for:

1. The **primary CTA** on a review detail page (`[Compare to another product]` or similar) — styled as filled button with inverted color
2. The **"Editor's Choice" / "Top Pick" outline** on featured product/review tiles (reuses existing `border-2 border-[#f5f5f0]` pattern from `tech-benchmark-spotlight.tsx`)
3. The **winner column outline** in comparison tables (when a clear "higher is better" metric has a clear winner)
4. The **active filter pill** in category browse (e.g. "Gaming Laptops" selected)
5. The **rating bar fill** on the review page (inverted bar on dark track to visualize the 1-10 rating)

No other element uses the inverted surface. Regular CTAs use bordered-ghost style (`bg-transparent border border-[#f5f5f0] text-[#f5f5f0]` — matches `tech-compare-cta.tsx` existing pattern).

**Winner detection logic (declarative for the executor):**
- **Numeric fields** where spec template has no "direction" metadata: assume higher-is-better for RAM, SSD, battery hours, scores; lower-is-better for weight, price. Encode defaults in a `lib/spec-direction.ts` map keyed by spec field name (lowercased). Falls back to "no winner" if unknown.
- **Boolean fields**: `true` wins over `false` (e.g. "Mechanical Keyboard: true").
- **Enum fields**: no winner (subjective — e.g. "Display: OLED vs IPS"); render both plain.
- **Text fields**: no winner.
- **Benchmark tests**: use `tech_benchmark_tests.direction` field (`higher_is_better` / `lower_is_better`) — explicit in the DB schema, no heuristics.

## Accessibility

- All interactive elements have visible focus rings: `focus-visible:outline-2 outline-offset-2 outline-[#f5f5f0]`

- Reduced motion: all animations documented above respect `prefers-reduced-motion: reduce`
- Breadcrumbs use `<nav aria-label="Breadcrumb">` with `<ol>` + `aria-current="page"` on the last item
