<objective>
Create the canonical rubric-map TypeScript module and the idempotent seed that populates `tech_benchmark_tests` for rubric v1.1 — all 13 disciplines, 39 test rows.

Purpose:
- Phase 16 ingest MUST be able to look up `tech_benchmark_tests.id` from a `(discipline, tool, field)` tuple emitted by the Mac bench harness JSONL. Without `RUBRIC_V1_1` and the seeded rows, ingest has nothing to match against.
- Phase 17 BPR computation uses the `bprEligible = true` flag on test rows to restrict the geometric mean to the 7 eligible disciplines.
- Plan 03 (`getBenchmarkSpotlight` refactor) consumes `RUBRIC_V1_1["cpu:geekbench6:multi"]`.

Output:
- `src/lib/tech/rubric-map.ts` — typed `Record<string, RubricTestSpec>` constant with 39 entries (matrix copied verbatim from RESEARCH.md §Rubric v1.1 Discipline × Test Row Matrix)
- `src/db/seeds/rubric-v1.1.ts` — `pnpm tsx`-runnable, idempotent, append-only, prints insert/skip count and exits 0

Append-only policy (D-14): a mistake in v1.1 requires a new `rubric-v1.2.ts`, NOT an in-place edit. The seed must NEVER use `onConflictDoUpdate`.

Drizzle 0.45.x API note (from checker BLOCKER 3): `onConflictDoNothing()` takes NO parameters in Drizzle 0.45.x — only `onConflictDoUpdate()` accepts `{ target: [...] }`. Passing `{ target: [...] }` to `onConflictDoNothing` type-checks under some Drizzle versions but is ignored / rejected at runtime. We call `.onConflictDoNothing()` with no arguments; Postgres picks the natural-key UNIQUE index automatically because it is the only constraint the rows could violate.
</objective>

  "higher_is_better", "lower_is_better",

export const techBenchmarkTests = pgTable("tech_benchmark_tests", {
  id: uuid("id").defaultRandom().primaryKey(),
  templateId: uuid("template_id").notNull().references(() => techBenchmarkTemplates.id, { onDelete: "cascade" }),
  name: text("name").notNull(),
  unit: text("unit").notNull(),
  direction: techBenchmarkDirectionEnum("direction").notNull().default("higher_is_better"),
  sortOrder: integer("sort_order").notNull().default(0),
  discipline: techBenchmarkDisciplineEnum("discipline"),    // nullable
  mode: techBenchmarkModeEnum("mode").notNull().default("both"),
  bprEligible: boolean("bpr_eligible").notNull().default(false),
  minRamGb: integer("min_ram_gb"),
  createdAt: timestamp("created_at").defaultNow().notNull(),
}, (t) => ({
  naturalKey: uniqueIndex("tech_benchmark_tests_natural_key_uniq")
    .on(t.templateId, t.discipline, t.mode, t.name),
}))
