# Plan 01.3-01: Switch Neon HTTP driver to postgres-js for Supabase compatibility

## Objective

Replace `@neondatabase/serverless` (Neon HTTP driver) with `postgres` (postgres-js) as the Drizzle ORM database driver. The current driver cannot connect to the Supabase pooler connection string, causing the entire site to show a blank page. postgres-js is a universal Postgres driver that works with both Supabase and Vercel serverless.

## Tasks

<task id="1">
<title>Replace Neon driver with postgres-js in db.ts</title>

<read_first>
- src/lib/db.ts
- src/lib/auth.ts
- src/db/schema.ts
- package.json
</read_first>

<action>
1. Install postgres-js and remove Neon driver:
   ```bash
   pnpm add postgres
   pnpm remove @neondatabase/serverless
   ```

2. Rewrite `src/lib/db.ts` to use postgres-js with drizzle-orm/postgres-js adapter:
   ```typescript
   import postgres from "postgres"
   import { drizzle } from "drizzle-orm/postgres-js"
   import * as schema from "@/db/schema"

   const databaseUrl =
     process.env.DATABASE_URL || "postgresql://build:build@localhost/build"
   const client = postgres(databaseUrl)
   export const db = drizzle(client, { schema })
   ```

   Key changes:
   - Import `postgres` from `"postgres"` (not `"@neondatabase/serverless"`)
   - Import `drizzle` from `"drizzle-orm/postgres-js"` (not `"drizzle-orm/neon-http"`)
   - Create client with `postgres(databaseUrl)` (not `neon(databaseUrl)`)
   - Pass `client` to `drizzle()` (same pattern, different driver)

3. Update `.env.example` comment to reflect Supabase:
   - The DATABASE_URL format is already correct for Supabase (postgresql://postgres:...)
   - Just verify the comment says "Supabase Postgres" (already does)

4. No changes needed to:
   - `src/lib/auth.ts` — uses `db` import, doesn't reference driver directly
   - `src/db/schema.ts` — pure Drizzle schema, driver-agnostic
   - `drizzle.config.ts` — uses `dialect: "postgresql"` with `url`, driver-agnostic
   - Any page files — they all import from `@/lib/db`
</action>
